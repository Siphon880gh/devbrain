# Gmail OAuth2 Email Sender — Replication Guide

This guide helps you build a Node.js CLI that sends email from a Gmail account to anyone, using **OAuth2 with a refresh token**.

This guide assumes nothing about your environment beyond Node.js and a Google account. Follow it end-to-end and you will end up with a small project that can:

1. Run a one-time OAuth2 consent flow to mint a long-lived **refresh token** for your Gmail account.
2. Use that refresh token to send email via Gmail's SMTP service from the command line — to any recipient.

---

## How it works (in one paragraph)

You register an **OAuth2 client** in Google Cloud Console. You authorize that client **once**, signing in as the Gmail account that will send mail; Google returns a **refresh token**. You store the client ID, client secret, and refresh token in a `.env` file. From then on, [Nodemailer](https://nodemailer.com/) uses those three values to obtain short-lived access tokens automatically and authenticate to Gmail's SMTP server every time you send mail. Recipients do not need to authorize anything — they just receive normal email from your Gmail address.

---

## Prerequisites

- **Node.js 18+** (the token helper uses the built-in global `fetch`).
- **A Google account** (personal Gmail or Google Workspace) — this is the address mail will be sent **from**.
- **A Google Cloud project** at <https://console.cloud.google.com/> (free to create).

## Step 2 — Set up Google Cloud Console

You need three things from Google:

1. The **Gmail API** enabled on a Cloud project.
2. A configured **OAuth consent screen**.
3. An **OAuth 2.0 Client ID** (with a client secret).

For typical personal or workspace use there is **no separate Gmail API charge** for this pattern; normal Gmail sending limits and policies still apply.

### 2.1 Enable the Gmail API

1. Open <https://console.cloud.google.com/> and select (or create) your project.
2. In the left menu: **APIs & Services** → **Enabled APIs & services**.

   ![[Pasted image 20250607025157.png]]

3. Click **+ ENABLE APIS AND SERVICES**.
4. Search for **Gmail API** and enable it.

   ![[Pasted image 20250607025243.png]]

### 2.2 Configure the OAuth consent screen

You need a usable consent screen before anyone (including you) can complete sign-in.

- Go to **APIs & Services** → **OAuth consent screen**.
- Choose **External** unless you have a Workspace internal-only use case.
- Fill in app name, support email, and developer contact.
- For scopes, you do **not** need to add scopes here — the helper script requests `https://mail.google.com/` directly during consent.

#### Testing vs Production

- While the app is in **Testing**, only **test users** listed on the consent screen can authorize. Add every Google account that will run this CLI (even if that account also owns the GCP project).
- Refresh tokens issued in Testing mode expire after **about 7 days**.
- To remove the test-user restriction and get long-lived refresh tokens, move the app toward **Production** / **published** status per Google's publishing flow.

![[Pasted image 20260501153721.png]]

Skipping this step often leads to "access blocked" or similar errors during sign-in until the account is allowed.

### 2.3 Create the OAuth 2.0 Client ID

- Go to **APIs & Services** → **Credentials** → **+ CREATE CREDENTIALS** → **OAuth client ID**.
- **Application type:** **Desktop app** is the simplest match for this CLI's token helper.
  - **Web application** also works, but then you must add `http://127.0.0.1:53682/oauth2callback` (or whatever you set in `.env`) under **Authorized redirect URIs**.
- Save and copy the **Client ID** and **Client Secret**.

## Step 3 — Create `.env` and `.env.example`

### `.env.example` (commit this)

```env
gmailEmailAddress="<YOUR_GMAIL_ADDRESS>"
gmailClientId="<YOUR_GOOGLE_OAUTH_CLIENT_ID>"
gmailClientSecret="<YOUR_GOOGLE_OAUTH_CLIENT_SECRET>"

# Optional for desktop token helper (defaults to http://127.0.0.1:53682/oauth2callback)
gmailDesktopRedirectUri="http://127.0.0.1:53682/oauth2callback"
```

### `.env` (do NOT commit)

Copy `.env.example` to `.env` and fill in real values. Until you complete the OAuth flow once, leave `gmailRefreshToken` as a placeholder:

```env
gmailEmailAddress="you@gmail.com"
gmailClientId="123456789-abc...apps.googleusercontent.com"
gmailClientSecret="GOCSPX-..."
gmailRefreshToken="MISSING"
gmailDesktopRedirectUri="http://127.0.0.1:53682/oauth2callback"
```

### What is each value?

| Variable | What it is | Where it comes from |
| --- | --- | --- |
| `gmailEmailAddress` | The mailbox that will send mail (the "from" address). | Your Gmail/Workspace account. |
| `gmailClientId` | Identifies the OAuth **app**. | Step 2.3, Google Cloud Console. |
| `gmailClientSecret` | Secret for the OAuth **app**. | Step 2.3, Google Cloud Console. |
| `gmailRefreshToken` | Long-lived token tied to the **user** that signed in. | Step 4 (printed by `get-refresh-token.js`). |
| `gmailDesktopRedirectUri` | Where Google redirects after consent. | Must match the OAuth client's authorized redirect URI. |

### What is being authenticated?

Two different things are involved:

1. **The OAuth client (the "app")** — identified by **client ID** and **client secret**. Those prove which Cloud project / client is requesting access.
2. **The Google account (the "user")** — whoever signs in on Google's OAuth screen. For this CLI that is usually **one** "master" mailbox that will send mail 24/7.

### Refresh token vs access token

After consent, Google issues:

- A **refresh token** — longer-lived (about **7 days** in testing mode, effectively long-lived once the app is published). Stored server-side in `.env`. Treat it like a password.
- **Access tokens** — short-lived (about **1 hour**). Generated automatically from the refresh token whenever you send mail. You never see or store these.

Nodemailer transparently exchanges the refresh token for a fresh access token before each session, so you do not have to write that logic yourself in `app.js`.

## Step 4 — Add `get-refresh-token.js` (the one-time OAuth helper)

This script does three things in order:

1. Builds an **OAuth2 consent URL** scoped to `https://mail.google.com/` (full Gmail send/read access).
2. Starts a **tiny local HTTP server** on `http://127.0.0.1:53682/oauth2callback` to catch Google's redirect.
3. Exchanges the returned `code` for tokens and prints the **refresh token** to your terminal.

### `get-refresh-token.js`

```javascript
require("dotenv").config();
const http = require("http");
const { URL, URLSearchParams } = require("url");

const CLIENT_ID = process.env.gmailClientId || "";
const CLIENT_SECRET = process.env.gmailClientSecret || "";
const REDIRECT_URI =
  process.env.gmailDesktopRedirectUri || "http://127.0.0.1:53682/oauth2callback";
const SCOPE = "https://mail.google.com/";

if (!CLIENT_ID || !CLIENT_SECRET) {
  console.error("Missing gmailClientId or gmailClientSecret in .env");
  process.exit(1);
}

function buildAuthorizationUrl() {
  const url = new URL("https://accounts.google.com/o/oauth2/v2/auth");
  url.searchParams.set("client_id", CLIENT_ID);
  url.searchParams.set("redirect_uri", REDIRECT_URI);
  url.searchParams.set("response_type", "code");
  url.searchParams.set("scope", SCOPE);
  url.searchParams.set("access_type", "offline");
  url.searchParams.set("prompt", "consent");
  return url.toString();
}

async function exchangeCodeForTokens(code) {
  const body = new URLSearchParams({
    code,
    client_id: CLIENT_ID,
    client_secret: CLIENT_SECRET,
    redirect_uri: REDIRECT_URI,
    grant_type: "authorization_code"
  });

  const response = await fetch("https://oauth2.googleapis.com/token", {
    method: "POST",
    headers: { "Content-Type": "application/x-www-form-urlencoded" },
    body
  });

  const data = await response.json();
  if (!response.ok) {
    const reason = data.error_description || data.error || "unknown error";
    throw new Error(`Token exchange failed: ${reason}`);
  }
  return data;
}

function startCallbackServer() {
  const redirectUrl = new URL(REDIRECT_URI);
  const expectedPath = redirectUrl.pathname;
  const port = Number(redirectUrl.port || 80);
  const host = redirectUrl.hostname;

  const server = http.createServer(async (req, res) => {
    try {
      if (!req.url) {
        res.writeHead(400);
        res.end("Missing request URL.");
        return;
      }

      const requestUrl = new URL(req.url, `${redirectUrl.protocol}//${host}:${port}`);
      if (requestUrl.pathname !== expectedPath) {
        res.writeHead(404);
        res.end("Not found.");
        return;
      }

      const error = requestUrl.searchParams.get("error");
      if (error) {
        res.writeHead(400);
        res.end(`OAuth error: ${error}`);
        throw new Error(`OAuth authorization failed: ${error}`);
      }

      const code = requestUrl.searchParams.get("code");
      if (!code) {
        res.writeHead(400);
        res.end("Missing authorization code.");
        throw new Error("Missing authorization code.");
      }

      const tokens = await exchangeCodeForTokens(code);

      res.writeHead(200, { "Content-Type": "text/plain" });
      res.end("Authorization complete. Return to terminal.");

      console.log("\nToken exchange succeeded.");
      console.log("Copy this into gmailRefreshToken:");
      console.log(tokens.refresh_token || "<No refresh_token returned>");

      if (!tokens.refresh_token) {
        console.log(
          "\nNo refresh_token was returned. Remove existing app access and rerun:"
        );
        console.log("https://myaccount.google.com/permissions");
      }

      server.close(() => process.exit(0));
    } catch (error) {
      console.error(error.message || error);
      server.close(() => process.exit(1));
    }
  });

  server.listen(port, host, () => {
    console.log("Open this URL in your browser to authorize:");
    console.log(buildAuthorizationUrl());
    console.log("\nWaiting for callback on:", REDIRECT_URI);
  });
}

startCallbackServer();
```

### Two important OAuth parameters

- `access_type=offline` — tells Google to issue a **refresh token**, not just an access token.
- `prompt=consent` — forces Google to show the consent screen again, which guarantees a **new** refresh token. Without this, Google often returns only an access token on subsequent authorizations.

## Step 5 — Add `app.js` (the email sender)

This script reads `.env`, validates that all required variables are present, builds a Nodemailer transporter using OAuth2 + the refresh token, and sends one message based on CLI arguments.

### `app.js`

```javascript
require("dotenv").config();
const nodemailer = require("nodemailer");

const requiredEnv = [
  "gmailEmailAddress",
  "gmailClientId",
  "gmailClientSecret",
  "gmailRefreshToken"
];
const missingEnv = requiredEnv.filter((name) => !process.env[name]);

if (missingEnv.length > 0) {
  console.error(
    `Missing required environment variables: ${missingEnv.join(", ")}`
  );
  console.error(
    "Run one-time OAuth consent to obtain gmailRefreshToken, then re-run this script."
  );
  process.exit(1);
}

// CLI arguments
const args = process.argv.slice(2);
const defaults = {
  recipient: process.env.gmailEmailAddress,
  subject: "Subject_line",
  message: "Message_body"
};
const [rawRecipient = "", rawSubject = "", rawMessage = ""] = args;
const recipient = rawRecipient.trim() || defaults.recipient;
const subject = rawSubject.trim() || defaults.subject;
const message = rawMessage.trim() || defaults.message;

// Setup Gmail transporter (OAuth2 + refresh token)
const transporter = nodemailer.createTransport({
  service: "gmail",
  auth: {
    type: "OAuth2",
    user: process.env.gmailEmailAddress || "",
    clientId: process.env.gmailClientId || "",
    clientSecret: process.env.gmailClientSecret || "",
    refreshToken: process.env.gmailRefreshToken || ""
  }
});

// Mail options
const mailOptions = {
  from: process.env.gmailEmailAddress,
  to: recipient,
  subject,
  text: message
};

// Send email
transporter.sendMail(mailOptions, (err, info) => {
  if (err) {
    console.error("Error sending email:", err.message || err);
    process.exitCode = 1;
  } else {
    console.log("Email sent:", info.response);
  }
});
```

### How Nodemailer uses the refresh token

By passing `auth.type: "OAuth2"` together with `clientId`, `clientSecret`, and `refreshToken`, Nodemailer:

1. Calls Google's token endpoint with the refresh token to get a fresh access token.
2. Authenticates to Gmail SMTP using `XOAUTH2` (no plain password ever sent).
3. Reuses the access token until it expires, then refreshes again automatically.

You do not need to maintain access tokens yourself.

## Step 6 — Send mail

With `.env` filled in and dependencies installed (`npm install`):

```bash
node app.js "recipient@example.com" "Subject line" "Message body"
```

Or via npm:

```bash
npm start -- "recipient@example.com" "Subject line" "Message body"
```

The sender address is always `gmailEmailAddress` from `.env`. The first CLI argument is the recipient.

Arguments are optional — if any is omitted (or passed as an empty string), defaults from `app.js` are used. So this also works:

```bash
npm start
```

On success you will see something like:

```text
Email sent: 250 2.0.0 OK  1714660000 abc123 - gsmtp
```

## Step 7 — Verify it actually works

OAuth2 only covers the **sending** Google account. Recipients do not need to authorize anything.

### 7.1 Email yourself

Send a message to the same address you authorized:

```bash
node app.js "you@gmail.com" "Self-test" "If you see this, SMTP + OAuth2 works."
```

Confirm it arrives in your own inbox (check Promotions / Spam too).

### 7.2 Email someone else

Send to a different address you control — for example, a separate personal email that never went through OAuth:

```bash
node app.js "other.address@example.com" "External test" "Second test to a non-OAuth recipient."
```

### 7.3 Verify both

Both messages should arrive. If the self-test works but the external one does not, check spam, the recipient address spelling, and any bounces in the shell output.

## Troubleshooting

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| `Missing required environment variables: ...` | `.env` not loaded or values blank. | Confirm `.env` sits next to `app.js` and all four required vars are set. |
| `invalid_grant` | Refresh token expired (testing-mode 7-day limit), revoked, or paired with a different OAuth client. | Re-run `npm run oauth:desktop` and update `gmailRefreshToken`. Consider publishing the consent screen. |
| `unauthorized_client` / `invalid_client` | Wrong client ID/secret, mismatched redirect URI, or consent screen misconfigured. | Re-check Step 2.3. Make sure the redirect URI in Google Cloud exactly matches `gmailDesktopRedirectUri`. |
| `access_denied` during sign-in | Account is not a test user while app is in Testing. | Add the email under **OAuth consent screen → Test users**, or publish the app. |
| No `refresh_token` returned by `get-refresh-token.js` | Google reuses an existing grant. | Revoke the app at <https://myaccount.google.com/permissions> and re-run `npm run oauth:desktop`. |
| Mail sends but never arrives | Spam filtering, recipient typo, or Gmail sending limit reached. | Check Spam/Promotions, recheck recipient address, and review Gmail's daily send limits. |

## Security notes

- **Never commit `.env`.** The refresh token is functionally a long-lived password to your mailbox's send/read access. Add `.env` to `.gitignore`.
- **Do not embed the refresh token in client-side code.** This pattern only makes sense for a server, daemon, or trusted CLI machine.
- **Rotate** if a token is leaked: revoke at <https://myaccount.google.com/permissions> and re-run `npm run oauth:desktop`.
- **Scope minimization:** this guide uses `https://mail.google.com/` (full Gmail). If you only need to send mail and never read, you can swap in `https://www.googleapis.com/auth/gmail.send` in `get-refresh-token.js` and re-authorize.

- A Google Cloud project with the Gmail API enabled and an OAuth 2.0 Desktop client.
