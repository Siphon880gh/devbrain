**OpenCode** is an open-source AI coding agent that helps you write and modify code from the terminal, IDE, or desktop. It is designed around agentic coding, meaning the AI can inspect your project, reason about changes, and assist with real development work instead of only answering questions.

OpenCode is especially known for its terminal-first workflow. The GitHub project describes it as having a strong focus on TUI, or terminal user interface, making it attractive for developers who prefer command-line tools, Neovim-style workflows, or local development environments instead of browser-based coding tools.

OpenCode can be used for tasks such as writing code, editing existing files, exploring a codebase, running agent sessions, and managing multiple AI coding sessions in parallel. Its official site also highlights features like LSP support, multi-session workflows, shareable session links, and the ability to connect with accounts such as GitHub Copilot or ChatGPT Plus/Pro.

A simple way to think about OpenCode:

```
OpenCode = an open-source terminal-first AI coding agent
```

It is useful when you want an AI assistant that feels closer to a command-line developer tool than a full AI IDE. For example, you might open a project in your terminal, start an OpenCode session, ask it to inspect the code, make changes, and iterate with you while staying inside your existing development workflow.

The main difference in positioning is that **Cline is often associated with editor-based workflows like VS Code**, while **OpenCode is strongly associated with terminal-first, TUI-based workflows**. Both are open-source AI coding agents, but they feel different depending on whether you prefer working mainly in an editor or mainly from the terminal.
