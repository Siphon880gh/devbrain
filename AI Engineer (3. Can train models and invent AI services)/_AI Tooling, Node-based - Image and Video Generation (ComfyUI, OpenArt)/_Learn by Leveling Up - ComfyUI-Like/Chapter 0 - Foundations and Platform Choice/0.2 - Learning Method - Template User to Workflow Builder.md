## Learning objectives

- Follow the three-stage progression used in every chapter.
- Know when to follow a template vs rebuild from scratch.
- Use the “node literacy” method: one new node group per session.

---

## Three stages

### Stage 1 — Template user (Chapters 0–1)

- Open an existing workflow.
- Change prompts, seeds, and obvious settings.
- Run and compare outputs.
- **Goal:** Success without understanding every wire.

### Stage 2 — Workflow adjuster (Chapters 2–4)

- Swap checkpoints and LoRAs.
- Add ControlNet or upscale branches.
- Break and fix connections to learn what each port does.
- **Goal:** Predictable control over style, composition, and quality.

### Stage 3 — Workflow builder (Chapters 5–6)

- Assemble core nodes without a template.
- Port JSON between local and RunComfy.
- Install custom nodes only when needed.
- **Goal:** Reusable pipelines for your content niche.

---

## How to study a template (repeat every lesson)

1. **Run unchanged** — confirm the template works on your platform.
2. **Label nodes** — rename groups: “Prompt”, “Sampler”, “Output”.
3. **Change one thing** — e.g. only the positive prompt; run again.
4. **Trace the data** — follow MODEL → CLIP → LATENT → IMAGE.
5. **Screenshot failures** — missing model/node errors teach folder structure.

You do **not** need to recreate from scratch early. Templates are the textbook.

---

## Session length

- **30 min** — one parameter sweep (CFG, steps, seed).
- **60 min** — one new node type (LoRA, ControlNet).
- **90 min** — one full project (character set, product shot).

Stop after one learning objective per session to avoid fatigue.

---

## Node literacy rule

Each lesson introduces at most **one new node family**. Master it before stacking the next.
