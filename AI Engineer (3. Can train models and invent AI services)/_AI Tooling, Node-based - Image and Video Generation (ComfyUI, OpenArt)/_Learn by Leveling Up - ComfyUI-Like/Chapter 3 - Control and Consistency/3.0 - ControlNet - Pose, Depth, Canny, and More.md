## Learning objectives

- Explain ControlNet as **structure guidance** separate from prompt.
- Run workflows with **OpenPose**, **Depth**, and **Canny** preprocessors.
- Balance control strength vs prompt freedom.

---

## What ControlNet adds

Feeds a **conditioning image** (edges, depth map, pose skeleton) into the sampler so composition matches your guide.

Common types:

| Type | Guide source | Best for |
|------|--------------|----------|
| **Canny** | Edge detection | Line art, architecture, products |
| **Depth** | Depth map | Scene layout, 3D-ish consistency |
| **OpenPose** | Body keypoints | Character pose lock |
| **Lineart / Scribble** | User lines | Sketch follow |

---

## Steps

1. Load a **ControlNet txt2img** template from Comfy.org (match SDXL or Flux variant).
2. Download required ControlNet `.safetensors` into `models/controlnet`.
3. Run with default reference image; note preprocessor node output preview.
4. Swap reference: photo → canny edges → compare.
5. Adjust **strength** (0.5–1.0 typical); lower = more prompt freedom.

## Platform routing

- **Local:** OK for single ControlNet + SDXL on decent GPU
- **RunComfy:** Prefer for Flux ControlNet stacks or multiple ControlNets
- **Promptus:** Use simplified pose/edge templates if full graph is overwhelming

---

## Beginner mistakes

- ControlNet model checkpoint family mismatch (SDXL vs SD1.5)
- Strength 1.0 + rigid preprocess → plastic look
- No preview of preprocessor output — always verify edges/pose look right
