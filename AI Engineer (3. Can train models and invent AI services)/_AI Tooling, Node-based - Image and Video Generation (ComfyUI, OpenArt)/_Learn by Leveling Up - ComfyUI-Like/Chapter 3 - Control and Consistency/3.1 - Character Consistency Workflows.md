## Learning objectives

- Combine **character LoRA**, **reference image**, and/or **IP-Adapter** for same-face outputs.
- Produce one character in multiple scenes and outfits.
- Know limits — consistency ≠ identical pixel copy.

---

## Common stacks (2026)

| Technique | Role |
|-----------|------|
| **Character LoRA** | Trained or downloaded face/body prior |
| **IP-Adapter** | Image prompt embedding from reference photo |
| **InstantID / FaceID nodes** | Face-specific conditioning (platform dependent) |
| **Fixed seed + same LoRA** | Cheap consistency for batch variations |

Start simple: **LoRA + locked seed + similar prompt skeleton**.

---

## Steps

1. Choose reference portrait (front-facing, even light works best).
2. Load character LoRA or IP-Adapter template from Comfy.org.
3. Generate **scene A:** `character on city street at night`
4. Keep LoRA strength + core face prompt tokens; change only scene words.
5. Generate **scene B:** cafe, **scene C:** office — same seed optional.
6. If face drifts, lower denoise on img2img pass from best face frame.

---

## Prompt skeleton

```
[CHARACTER TOKENS / LoRA triggers], [outfit], [action],
[environment], [lighting], [camera]
Negative: different person, duplicate face, bad anatomy, ...
```

## Viral use case

Short-form **character series** — same persona in 3 hot takes / 3 backgrounds — reads as branded content.

---

## Beginner mistakes

- Changing face description words every run while using LoRA
- Reference photo profile-only when template expects frontal
- Stack InstantID + strong LoRA + high CFG — uncanny results
