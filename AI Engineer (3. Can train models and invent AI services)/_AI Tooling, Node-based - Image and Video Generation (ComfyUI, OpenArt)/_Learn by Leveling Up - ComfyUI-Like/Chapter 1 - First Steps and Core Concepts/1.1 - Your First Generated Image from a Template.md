**Platform note:** Stable for most computers — baseline M1 MacBook Pro (2021) or Promptus.ai in browser.

**Recommended platform:** Promptus.ai (fastest first win) **or** local ComfyUI “Image Generation” template.

---

## Learning objectives

- Run a pre-built text-to-image workflow end to end.
- Change the positive prompt and seed, then run again.
- Diagnose a missing checkpoint/model error.

---

## Steps

### Option A — Promptus.ai (beginner path)

1. Create an account on Promptus.ai if needed.
2. Open a **text-to-image** or **Flux/SDXL starter** template.
3. Set a simple positive prompt, e.g. `a red apple on a wooden table, soft window light, product photo`.
4. Add a negative prompt: `blurry, deformed, watermark, text`.
5. Run / Generate and wait for completion.
6. Change **one word** in the prompt; run again. Compare.

### Option B — Local ComfyUI

1. Open ComfyUI → **Workflow** → **Browse Templates** → **Basics** → **Image Generation**.
2. Simplify the graph if needed (hide extra nodes) so you see: Checkpoint → CLIP → KSampler → VAE → Save.
3. Enter the same prompts as Option A.
4. Click **Queue Prompt** / Run.
5. If error mentions missing checkpoint, download the named `.safetensors` into your `models/checkpoints` folder (or use ComfyUI Manager model downloader).

### Option C — RunComfy

Use if local fails after one fix attempt. Load the same “Image Generation” starter from the template library on a small GPU tier.

---

## Settings for first run

| Setting | Suggested start |
|---------|-----------------|
| Steps | 20–30 (or template default) |
| CFG | 7–8 |
| Seed | `-1` random first, then lock a good seed |
| Size | 1024×1024 (SDXL/Flux) or template default |

---

## Beginner mistakes

- Empty positive prompt — always set subject + lighting + style hint.
- Ignoring error text — “file not found” almost always means missing model path.
- Changing ten settings at once — change prompt OR seed OR steps, not all three.
