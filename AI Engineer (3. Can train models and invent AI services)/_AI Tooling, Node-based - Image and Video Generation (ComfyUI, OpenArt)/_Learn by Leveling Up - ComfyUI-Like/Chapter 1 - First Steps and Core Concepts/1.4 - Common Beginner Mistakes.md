## Learning objectives

- Recognize the ten most common failure modes in ComfyUI-like workflows.
- Fix missing model/node errors without starting over.
- Know when to switch platform instead of debugging for hours.

---

## Mistakes and fixes

| Mistake | Symptom | Fix |
|---------|---------|-----|
| Missing checkpoint | Red node / file not found | Download exact `.safetensors` name; place in `models/checkpoints` |
| Missing custom node | “Node not found” on load | ComfyUI Manager → Install Missing; or use RunComfy |
| Wrong model for workflow | Mushy or gray output | Match workflow author’s checkpoint (SDXL vs Flux) |
| VRAM overload | CUDA OOM / freeze | Lower resolution, batch=1, use smaller model or cloud GPU |
| Mac + CUDA workflow | Won’t run locally | Move lesson to RunComfy |
| CFG too high | Burnt, crunchy look | Drop CFG to 4–7 (Flux often lower than SDXL) |
| Steps too low | Soft, unfinished detail | Raise steps or use turbo model correctly |
| No negative prompt | Extra fingers, text gibberish | Add standard negative block from [[1.3 - Prompt Writing Basics]] |
| Random setting churn | No learning | Change one variable per run |
| Outdated tutorial paths | Install commands fail | Check workflow date; prefer Comfy.org templates |

---

## Missing model workflow

1. Read full error — note **exact filename**.
2. Search Hugging Face or Civitai for that filename.
3. Place in folder the error specifies (checkpoints, loras, vae, controlnet).
4. Restart ComfyUI or refresh model list.
5. If >3 missing files, consider RunComfy template with models pre-provisioned.

---

## When to stop debugging locally

Switch to **RunComfy** or simplify the template if:

- More than **5** missing nodes after Manager install attempt
- Workflow tagged **video + 2025/2026 custom nodes**
- You spent **>1 hour** on environment fixes for a Chapter 1–2 lesson
