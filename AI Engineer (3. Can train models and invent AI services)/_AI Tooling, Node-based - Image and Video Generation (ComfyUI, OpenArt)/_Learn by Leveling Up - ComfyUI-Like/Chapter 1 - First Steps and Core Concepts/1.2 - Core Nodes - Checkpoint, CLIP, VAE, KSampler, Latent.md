## Learning objectives

- Explain what each core node does in plain language.
- Trace data flow from prompt to pixels.
- Know which nodes swap when changing model families (SDXL vs Flux).

---

## The pipeline

```
Load Checkpoint
    ├── MODEL ──────────────► KSampler
    ├── CLIP ──► CLIP Text Encode (+ / -) ──► KSampler
    └── VAE (optional path) ─────────────────► VAE Decode

Empty Latent Image ──► LATENT ──► KSampler ──► LATENT ──► VAE Decode ──► IMAGE
```

---

## Node reference

### Load Checkpoint

- **Role:** Loads the diffusion **brain** (UNet weights) plus bundled CLIP and often VAE.
- **You change it when:** Switching SDXL ↔ Flux ↔ turbo models.
- **File location (local):** `models/checkpoints/*.safetensors`

### CLIP Text Encode (positive / negative)

- **Role:** Turns your text into numbers the model understands.
- **Positive:** What you want. **Negative:** What to suppress.
- **Tip:** Be specific on subject, lighting, camera; negative kills artifacts.

### Empty Latent Image

- **Role:** Creates invisible canvas (width × height × batch).
- **Not pixels yet** — latent space math until VAE decode.

### KSampler

- **Role:** Denoising engine — turns noise + prompt into latent image.
- **Key knobs:** steps, CFG, sampler name, seed.
- **Deep dive:** sibling track `1.1 - KSampler for diffusion, Latent Image the canvas`

### VAE Decode

- **Role:** Converts latent tensor to viewable RGB image.
- **Some workflows** use external VAE loader for quality tweaks.

### Save Image

- **Role:** Writes PNG/WebP to output folder or preview panel.

## Exercise (30 min)

On your working template:

1. Rename node groups: `Prompt`, `Canvas`, `Engine`, `Output`.
2. Disconnect **negative** CLIP temporarily → run → reconnect. Notice artifact increase.
3. Change latent size only → run. Note composition shift.
4. Lock seed, change only CFG 5 vs 9 → compare prompt adherence.

---

## Beginner mistakes

- Editing CLIP while the wrong checkpoint is loaded (Flux prompt on SDXL node wiring).
- Expecting Empty Latent to show a preview — it never will.
- Setting batch size > 1 without enough VRAM.
