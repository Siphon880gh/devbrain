## Learning objectives

- Design **one reusable pipeline** for your niche (portrait, product, character IP, cinematic b-roll).
- Document models, nodes, presets, and platform.
- Produce a **content batch** proving the workflow end to end.

---

## Capstone requirements

Your signature workflow must include **at least**:

| Layer | Requirement |
|-------|-------------|
| Generation | txt2img and/or img2img base |
| Control | LoRA **or** ControlNet **or** consistency technique |
| Quality | Upscale or face/detail pass |
| Motion (optional) | img2vid branch for one hero piece |
| Portability | Model manifest + JSON runs on backup platform |

---

## Niche examples

### Character IP channel

- Character LoRA + OpenPose ControlNet
- 9:16 still series + img2vid hook
- Face detailer before upscale

### Product / ecommerce

- Studio prompt template + depth ControlNet
- Inpaint for label cleanup
- 1:1 and 4:5 export presets

### Cinematic b-roll

- Flux Dev stills + txt2vid/img2vid b-roll
- Grain/color in editor — minimal nodes in graph

Pick **one** niche for capstone focus.

---

## Deliverables checklist

- [ ] `signature-workflow.json` saved
- [ ] Model manifest (checkpoints, LoRAs, controlnet, custom nodes)
- [ ] Preset card (steps, CFG, samplers, negatives)
- [ ] **5 stills** consistent with niche
- [ ] **1 upscaled hero** image
- [ ] **1 short video** OR 3-slide viral hook set
- [ ] Short write-up: what you would automate next

---

## Graduation criteria

You moved from **template user** → **workflow builder** when:

- You can explain every wire in your signature graph
- You fix missing model/node errors without abandoning the project
- You choose local vs RunComfy vs browse intentionally
- You ship a batch, not a single lucky seed
