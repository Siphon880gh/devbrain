**Platform note:** Stable for most computers — txt2img from blank graph.

---

## Learning objectives

- Assemble core txt2img nodes without a template.
- Save and reload your graph; confirm it runs twice identically.
- Document each connection you make.

---

## Build order (blank canvas)

1. **Add Node → Load Checkpoint** — select your main model
2. **CLIP Text Encode (Prompt)** ×2 — positive and negative
3. Connect checkpoint **CLIP** → both encoders
4. **Empty Latent Image** — set width/height
5. **KSampler** — wire MODEL, positive, negative, latent
6. **VAE Decode** — latent + VAE from checkpoint
7. **Save Image**

Queue. Fix any red ports before adding extras.

---

## Validation tests

| Test | Pass criteria |
|------|---------------|
| Re-run same seed | Identical output |
| Change only negative | Artifact behavior changes predictably |
| Swap checkpoint | You know which nodes to rewire |

---

## When to add branches

Only after base graph runs:

- LoRA → between checkpoint and sampler
- ControlNet → parallel conditioning into sampler
- Upscale → after VAE decode

One branch per session.

---

## Beginner mistakes

- Connecting CLIP to MODEL port
- Forgetting VAE on decode — gray or noise image
- Saving workflow JSON before first successful queue
