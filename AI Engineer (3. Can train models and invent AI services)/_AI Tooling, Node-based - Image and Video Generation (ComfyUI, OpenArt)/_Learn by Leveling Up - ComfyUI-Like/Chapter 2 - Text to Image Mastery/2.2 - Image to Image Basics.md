**Platform note:** Stable for most computers — img2img with SDXL or Flux; needs a source image.

---

## Learning objectives

- Replace Empty Latent with **Load Image → VAE Encode** (or dedicated img2img path).
- Control change strength with **denoise** (KSampler denoise / strength).
- Use img2img for style transfer, sketch-to-render, and photo variation.

---

## Core concept

**Txt2img:** noise → image  
**Img2img:** existing latent + partial noise → modified image

**Denoise strength** (0–1):

| Value | Effect |
|-------|--------|
| 0.2–0.35 | Small tweaks, keep composition |
| 0.45–0.65 | Style and content shift, same layout |
| 0.7–0.9 | Major change; may drift from source |
| 1.0 | Nearly txt2img |

---

## Steps

1. Find an **img2img** template on Comfy.org or your platform.
2. Load a photo or your Chapter 1 best image.
3. VAE Encode to latent → feed KSampler instead of Empty Latent.
4. Run at denoise **0.35** with prompt describing desired change.
5. Sweep denoise: 0.3, 0.5, 0.7 — same seed.

---

## Use cases to try

- **Photo → illustration:** raise denoise, strong style prompt
- **Product color variant:** low denoise, change color words only
- **Sketch cleanup:** medium denoise, “clean line art, flat colors”

---

## Platform notes

- **Promptus:** look for img2img or “image variation” templates.
- **RunComfy:** useful if encode nodes or VAE mismatch causes errors locally.

---

## Beginner mistakes

- Denoise too high on faces — identity drift
- Forgetting to match source aspect ratio on latent
- Using txt2img template without connecting encoded latent
