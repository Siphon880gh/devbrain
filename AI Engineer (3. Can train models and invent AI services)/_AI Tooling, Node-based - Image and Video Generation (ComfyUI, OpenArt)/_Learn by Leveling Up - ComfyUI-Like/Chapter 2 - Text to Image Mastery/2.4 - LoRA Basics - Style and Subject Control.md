**Platform note:** Stable for most computers for SDXL LoRAs; verify Flux-compatible LoRAs before use.

---

## Learning objectives

- Load a LoRA via **Load LoRA** (or platform equivalent) between checkpoint and sampler.
- Tune **strength** (model / clip weights) without destroying base model.
- Stack at most two LoRAs until you understand interactions.

---

## What LoRA does

Small add-on weights that bias the model toward a **style**, **character**, or **object** without full fine-tune.

Files: `models/loras/*.safetensors`

---

## Wiring

```
Load Checkpoint → Load LoRA → KSampler
                      ↑
              (model + clip outputs replace checkpoint outputs)
```

Some workflows chain multiple Load LoRA nodes.

---

## Steps

1. Pick a **style LoRA** (e.g. film grain, anime line, product studio) matched to your checkpoint family.
2. Add Load LoRA node; select file.
3. Start strength **0.6–0.8** (workflow may expose `strength_model` / `strength_clip`).
4. Include trigger words from LoRA page if listed.
5. Run base vs LoRA comparison — same seed.
6. Optional: add second LoRA at **0.4–0.6** (style + character).

---

## Strength guide

| Strength | Effect |
|----------|--------|
| 0.3–0.5 | Subtle |
| 0.6–0.8 | Default sweet spot |
| 0.9–1.2 | Aggressive; may break anatomy |
