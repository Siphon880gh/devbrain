**Platform note:** Stable for most computers — SDXL or Flux Schnell txt2img.

**Recommended:** Local ComfyUI or RunComfy with Comfy.org “Image Generation” or Flux txt2img template.

---

## Learning objectives

- Walk a full txt2img graph node by node with intentional edits.
- Add optional quality nodes (refiner, secondary sampler) only when you understand the base path.
- Export or save a simplified “my txt2img” subgraph.

---

## Base workflow checklist

Confirm your graph has:

- [ ] Load Checkpoint
- [ ] CLIP Text Encode (positive + negative)
- [ ] Empty Latent Image (width/height set)
- [ ] KSampler (model + positive + negative + latent in)
- [ ] VAE Decode
- [ ] Save Image or Preview

---

## Deep-dive exercises

### Exercise 1 — Aspect ratios for use cases

| Use case | Size example |
|----------|--------------|
| Instagram feed | 1080×1080 or 1024×1024 |
| Story / Reels cover | 768×1344 or 9:16 template |
| YouTube thumb | 1280×720 |
| Portrait | 832×1216 |

Run one prompt at three aspect ratios; same seed if possible.

### Exercise 2 — Sampler comparison

Lock seed and prompt. Try two samplers (e.g. `euler` vs `dpmpp_2m`). Note texture and speed.

### Exercise 3 — Optional refiner branch

If template includes SDXL refiner:

- Identify which KSampler is **base** vs **refiner**
- Run without refiner → with refiner → compare skin and micro-detail

Skip refiner on Flux templates unless explicitly present.

---

## Simplifying clutter

Group and collapse nodes you are not editing this week. Save a **minimal** version as “txt2img-learning.json” for repeat drills.

---

## Beginner mistakes

- Duplicating KSampler chains without connecting MODEL correctly
- Refiner at wrong denoise strength (common SDXL gotcha)
- Saving workflow before confirming it runs once
