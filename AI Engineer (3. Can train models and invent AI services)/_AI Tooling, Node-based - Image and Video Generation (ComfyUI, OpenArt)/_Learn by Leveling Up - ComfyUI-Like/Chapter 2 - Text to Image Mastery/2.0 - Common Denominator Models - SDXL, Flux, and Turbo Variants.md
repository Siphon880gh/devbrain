## Learning objectives

- Choose between SDXL, Flux Dev/Schnell, and turbo checkpoints for a given job.
- Know which models appear in Comfy.org, Promptus, and RunComfy templates in 2026.
- Match resolution and step count to the model type.

---

## Model cheat sheet (2026)

| Model family | Best for | Typical steps | Notes |
|--------------|----------|---------------|-------|
| **SDXL** | General stills, LoRA ecosystem | 20–30 | Huge template + LoRA library |
| **Flux Dev** | High-quality stills, realism | 20–28 | Strong prompt following |
| **Flux Schnell** | Fast iteration | 4–8 | Good for brainstorming |
| **Z-Image-Turbo / turbo SDXL** | Speed on lower VRAM | 4–12 | See sibling `1.4 - Z-Image-Turbo Text to Image` |
| **SD 1.5** | Legacy LoRAs, lightweight | 20–30 | Still in old workflows |

**Rule:** Use the checkpoint the template author used until you know how to rewire CLIP/VAE.

---

## When to pick which

- **Learning prompts** → Flux Schnell or turbo variant (fast feedback)
- **Final hero image** → Flux Dev or SDXL + upscale
- **Specific LoRA only on SDXL** → SDXL checkpoint
- **Product / portrait** → Flux Dev or SDXL with good VAE

---

## Steps

1. Open your platform’s model picker or checkpoint node.
2. List what you have installed vs what templates request.
3. Download **one** high-quality and **one** fast model if disk allows.
4. Run the **same prompt** on both; compare time vs quality.
5. Record default steps/CFG the template recommends for each.

---

## Cross-platform tip

Workflow JSON often hardcodes checkpoint filenames. When moving local → RunComfy:

- Either upload the same `.safetensors`
- Or swap to host’s equivalent and re-tune CFG/steps

---

## Beginner mistakes

- Running Flux steps (4) on SDXL or vice versa.
- Mixing SDXL LoRA with Flux checkpoint without conversion workflow.
- Downloading every trending model — pick two and master them.
