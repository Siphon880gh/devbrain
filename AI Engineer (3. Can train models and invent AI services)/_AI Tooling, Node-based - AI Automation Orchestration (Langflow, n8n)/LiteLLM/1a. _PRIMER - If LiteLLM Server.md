## Requirement - Model Access

Before installing LiteLLM, you need **at least one model backend** ready.

That means one of these:
- a **remote AI provider** (like OpenAI)
- or a **local Ollama server**

LiteLLM does not generate responses by itself.  
It sends requests to a real model behind the scenes.

---

## Requirement - uv

When installing a new tool, the docs may tell you to use another package manager.

LiteLLM may recommend using a tool called `uv` to install and run it.

So if you see commands starting with `uv`, first check if it exists:

```bash
uv --version
```

If you see a version → you’re good  
If you see “command not found” → install it first. Refer to [[_PRIMER - uv (Python)]] for uv installation instructions.

**But we're not building Python AI powered app?**

Even though you are setting up LiteLLM as a server, it is still **built in Python**. You are **not writing Python code**, but under the hood:

- LiteLLM runs as a Python application
- `uv` installs and manages the Python environment for you
- The `litellm` command you run is a Python-powered server

👉 This means:

- This is a python based server
- You don’t need to manually manage Python or virtual environments
- But Python is still what powers the server
- And the official instructions recommend using the package manager `uv` which can install cli at the system level or add packages at the project folder level

## What LiteLLM is

LiteLLM is a **middle layer** between your app and AI models.

Instead of connecting your app to multiple providers directly, you connect your app to LiteLLM.

LiteLLM can connect to remote APIs or your own local models (via Ollama)

To summarize, LiteLLM talks to:
- OpenAI
- Anthropic
- Azure
- and others
- Ollama (Your local models)

In addition, LiteLLM can auto route to a specific router based on keywords and other conditions. See [[LiteLLM Auto Routing]]
