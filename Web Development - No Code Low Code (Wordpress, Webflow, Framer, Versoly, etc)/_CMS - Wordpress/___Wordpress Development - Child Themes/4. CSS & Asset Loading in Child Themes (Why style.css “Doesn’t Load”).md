
This is the most common child-theme failure point.

### 1. Folder & naming must be exact

```
/wp-content/themes/hello-elementor-child/
  ├── style.css
  └── functions.php
```

No extra extensions. No capitalization differences.

### 2. style.css header must include Template

```css
/*
 Theme Name: Hello Elementor Child
 Template: hello-elementor
*/
```

If `Template` is missing or wrong:

- WordPress treats it as a standalone theme
    
- Parent styles won’t load
    
- Behavior becomes inconsistent
    

### 3. You must enqueue the child stylesheet

WordPress does **not** automatically output `style.css`.

Minimal, known-good enqueue:

```php
<?php
add_action( 'wp_enqueue_scripts', function() {
    wp_enqueue_style(
        'hello-elementor-child-style',
        get_stylesheet_uri(),
        [],
        wp_get_theme()->get( 'Version' )
    );
});
```

Hello Elementor already enqueues its own parent CSS, so this is sufficient.

### 4. Verify in the browser

In DevTools → Network → CSS, you should see:

```
/wp-content/themes/hello-elementor-child/style.css
```

If it’s missing → enqueue problem  
If it’s there but not applying → selector specificity or Elementor inline styles

### Minimal sanity test

```css
body {
  outline: 5px solid red;
}
```

If you don’t see it, the stylesheet is not loading.
