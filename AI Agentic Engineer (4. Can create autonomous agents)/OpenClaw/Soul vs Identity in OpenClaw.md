# Soul vs Identity in OpenClaw

OpenClaw separates internal behavior from outward presentation:

- **Soul**: internal operating layer (values, decision style, tone, boundaries, uncertainty handling).
- **Identity**: outward presentation layer (name, role, display style, and how the agent introduces itself).

Short version:

- Soul = what the agent embodies.
- Identity = what the user sees.

Why this matters: one agent can have a strict, safety-focused Soul while still presenting with a friendly Identity.
