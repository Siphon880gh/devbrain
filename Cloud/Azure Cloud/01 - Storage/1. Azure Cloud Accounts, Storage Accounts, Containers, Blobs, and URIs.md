
When working with Azure Blob Storage, the terms can feel confusing because they sound similar:

**Azure account → Subscription → Resource group → Storage account → Container → Blob**

Each level has a different job.

---

## 1. Azure Account

Your **Azure account** is your login identity for Microsoft Azure.

Example:

```text
you@example.com
```

This is the account you use to sign in to the Azure Portal.

The Azure account itself does not directly store files. It gives you access to Azure resources, billing, subscriptions, permissions, and services.

---

## 2. Azure Subscription

A **subscription** is the billing and resource boundary.

Think of it like:

```text
Azure account
└── Subscription
```

A subscription is where Azure charges usage. You can have multiple subscriptions under one Azure account.

Example:

```text
Production Subscription
Development Subscription
Client Project Subscription
```

---

## 3. Resource Group

A **resource group** is a folder-like container for Azure resources.

Example:

```text
Azure account
└── Subscription
    └── Resource group
```

You might create a resource group called:

```text
rg-videolistings-prod
```

Inside that resource group, you can place related resources like:

```text
Storage Account
Virtual Machine
App Service
Database
```

---

## 4. Storage Account

A **storage account** is the main Azure Storage resource.

Microsoft says a storage account contains Azure Storage data objects such as blobs, files, queues, and tables. It also gives those objects a unique namespace that can be accessed over HTTP or HTTPS. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/storage/common/storage-account-overview?utm_source=chatgpt.com "Storage Account Overview - Azure ..."))

Example storage account name:

```text
mycompanymedia
```

A storage account can hold different Azure Storage services:

```text
Storage Account
├── Blob Storage
├── File Shares
├── Queues
└── Tables
```

For website assets, images, videos, backups, exports, PDFs, logs, or static files, you are usually talking about **Blob Storage**.

---

## 5. Blob Container

A **container** is inside the storage account.

A container organizes a set of blobs, similar to how a directory organizes files in a file system. A storage account can include many containers, and a container can store many blobs. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/storage/blobs/blob-containers-portal?utm_source=chatgpt.com "Manage blob containers using the Azure portal"))

Example:

```text
Storage Account: mycompanymedia
└── Container: property-videos
```

Common container names:

```text
images
videos
backups
exports
invoices
public-assets
```

Container naming rules include:

- 3 to 63 characters
    
- lowercase letters, numbers, and dashes only
    
- must start with a letter or number
    
- no consecutive dashes ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction?utm_source=chatgpt.com "Introduction to Azure Blob Storage"))
    

Good:

```text
property-videos
client-assets
website-images
```

Bad:

```text
PropertyVideos
client_assets
my--container
```

---

## 6. Blob

A **blob** is the actual file/object stored inside a container.

Example:

```text
Storage Account: mycompanymedia
└── Container: property-videos
    └── Blob: listing-123.mp4
```

A blob can be many types of files:

```text
.jpg
.png
.mp4
.pdf
.csv
.zip
.json
.txt
```

Azure Blob Storage has common blob types, including **block blobs**, **append blobs**, and **page blobs**. For normal file uploads like images, videos, PDFs, and documents, you are usually dealing with **block blobs**. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction?utm_source=chatgpt.com "Introduction to Azure Blob Storage"))

---

# Azure Blob URI Structure

The URI is the web address for the storage account, container, or blob.

## Storage Account URI

The basic storage account endpoint looks like this:

```text
https://{storage-account-name}.blob.core.windows.net
```

Example:

```text
https://mycompanymedia.blob.core.windows.net
```

---

## Container URI

A container URI includes the storage account name and the container name.

Format:

```text
https://{storage-account-name}.blob.core.windows.net/{container-name}
```

Example:

```text
https://mycompanymedia.blob.core.windows.net/property-videos
```

Microsoft gives the same general format for container URIs: `https://myaccount.blob.core.windows.net/mycontainer`. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction?utm_source=chatgpt.com "Introduction to Azure Blob Storage"))

---

## Blob URI

A blob URI includes the storage account, container, and file/blob name.

Format:

```text
https://{storage-account-name}.blob.core.windows.net/{container-name}/{blob-name}
```

Example:

```text
https://mycompanymedia.blob.core.windows.net/property-videos/listing-123.mp4
```

Microsoft’s example blob URI follows the same pattern: `https://myaccount.blob.core.windows.net/mycontainer/myblob`. ([Microsoft Learn](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction?utm_source=chatgpt.com "Introduction to Azure Blob Storage"))

---

## Blob URI with “Folders”

Azure Blob Storage can make blob names look like they are inside folders.

Example:

```text
https://mycompanymedia.blob.core.windows.net/property-videos/2026/los-angeles/listing-123.mp4
```

In this URI:

```text
Storage account: mycompanymedia
Container: property-videos
Blob name: 2026/los-angeles/listing-123.mp4
```

Important: in Blob Storage, these “folders” are usually part of the blob name. They are often called **virtual directories**.

Microsoft also shows this style:

```text
https://myaccount.blob.core.windows.net/mycontainer/myvirtualdirectory/myblob
```

([Microsoft Learn](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction?utm_source=chatgpt.com "Introduction to Azure Blob Storage"))

---

# Full Hierarchy Example

Here is the full structure:

```text
Azure Account
└── Subscription
    └── Resource Group
        └── Storage Account: mycompanymedia
            └── Blob Container: property-videos
                └── Blob: 2026/los-angeles/listing-123.mp4
```

The final blob URI would be:

```text
https://mycompanymedia.blob.core.windows.net/property-videos/2026/los-angeles/listing-123.mp4
```

Breakdown:

|Part|Example|
|---|---|
|Protocol|`https://`|
|Storage account|`mycompanymedia`|
|Blob service domain|`.blob.core.windows.net`|
|Container|`property-videos`|
|Blob path/name|`2026/los-angeles/listing-123.mp4`|

---

# Are Blob URIs Public?

Not always.

A blob can have a URI but still be private.

Example:

```text
https://mycompanymedia.blob.core.windows.net/property-videos/listing-123.mp4
```

That URI may exist, but if the container or blob is private, the browser may return an authorization error.

Access depends on settings such as:

```text
Private access
Public container access
Public blob access
SAS token access
Microsoft Entra ID / Azure RBAC
Storage account keys
```

For secure sharing, Azure commonly uses a **SAS**, or Shared Access Signature. A SAS is a URI/token that grants limited access to Azure Storage resources for a specific time without sharing the storage account key. ([Microsoft Learn](https://learn.microsoft.com/en-us/answers/questions/5596140/azure-stt-cloud-processing-failed-%28invaliduri-auth?utm_source=chatgpt.com "Azure STT Cloud processing failed. [InvalidUri ..."))

A SAS URL may look like this:

```text
https://mycompanymedia.blob.core.windows.net/property-videos/listing-123.mp4?sv=...&se=...&sp=...&sig=...
```

The part after `?` is the SAS token.

---

# Why the Storage Account Name Matters in the URI

Azure storage account names are globally unique. Because of that, the URI can identify the correct storage account across Azure.

Microsoft notes that the URI for a container or blob must be unique. Since every storage account name is unique, two different storage accounts can have containers with the same name, but within one storage account, each container name must be unique. ([Microsoft Learn](https://learn.microsoft.com/en-us/rest/api/storageservices/naming-and-referencing-containers--blobs--and-metadata?utm_source=chatgpt.com "Naming and Referencing Containers, Blobs, and Metadata"))

Example:

```text
https://companyone.blob.core.windows.net/images/logo.png
https://companytwo.blob.core.windows.net/images/logo.png
```

Both can have a container called:

```text
images
```

Both can have a blob called:

```text
logo.png
```

But the storage account names are different:

```text
companyone
companytwo
```

So the full URIs are still unique.

# Simple Mental Model

Think of Azure Blob Storage like this:

```text
Storage Account = the warehouse
Container = a room inside the warehouse
Blob = a file/object stored in that room
URI = the address to reach it
```

Example:

```text
Warehouse:
mycompanymedia

Room:
property-videos

File:
listing-123.mp4

URI:
https://mycompanymedia.blob.core.windows.net/property-videos/listing-123.mp4
```
