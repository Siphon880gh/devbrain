Azure pricing surprises beginners more than any other topic. Once you understand the four-part billing model, you can predict the cost of almost any resource before clicking "Create."

This lesson explains:

```text
- How Azure actually charges
- Free vs free tier vs always-free
- The two services that quietly drain budgets
- The Pricing Calculator
- The Cost Management dashboard
```

---

## 1. The Four-Part Billing Model

Every Azure charge boils down to four ingredients:

```text
Resource type   ×   Region   ×   Tier/SKU   ×   Usage amount
```

Example for a Linux VM:

```text
Standard_B1s   ×   East US   ×   Pay-As-You-Go   ×   730 hours/month   ≈   ~$8/month
```

Example for Blob Storage:

```text
Block Blob   ×   East US   ×   Hot LRS   ×   100 GB stored + 10 GB egress   ≈   pennies
```

The same resource can vary in price by 2–10x depending on **region** and **tier**, so always check both.

## 2. The Three Kinds of "Free"

This is where most beginners get confused.

```text
Term                What it means
-----------------   ----------------------------------------------------
Free trial credit   $200 of credit you can spend on ANYTHING for 30 days
Free tier (12 mo)   Free amounts of popular services for the first year
Always free         Free amounts of certain services FOREVER, no time limit
```

A few examples:

```text
Service                          Always-free amount per month
------------------------------   ---------------------------------------
Azure Functions (Consumption)    1,000,000 executions + 400,000 GB-s
Static Web Apps (Free tier)      100 GB bandwidth, custom domains, SSL
App Service (F1)                 1 hour CPU/day, 1 GB storage, 1 GB RAM
Cosmos DB (free tier)            1000 RU/s + 25 GB on one account
Microsoft Entra ID (Free)        500,000 directory objects
Storage (always free)            Inbound traffic, ARM API calls
```

> When you exceed the free amount, you pay **only for the overage**, not the whole bill.

## 3. How Different Services Bill

Different resources use different units. Knowing the unit tells you what to watch.

| Service                     | Billed Per                                  |
|-----------------------------|---------------------------------------------|
| Virtual Machines            | Per second running (deallocated = $0)       |
| App Service plans           | Per hour the plan exists, regardless of use |
| Azure Functions (Consumption)| Per execution + per GB-second               |
| Storage (Blob/File)         | Per GB stored + per operation + per egress  |
| SQL Database (DTU/vCore)    | Per hour of provisioned capacity            |
| SQL Serverless              | Per vCore-second of compute used            |
| Cosmos DB (provisioned)     | Per RU/s reserved per hour                  |
| Cosmos DB (serverless)      | Per RU consumed                             |
| Application Gateway         | Per hour + per processed GB                 |
| Public IP (Standard)        | Per hour the IP exists                      |
| Egress (outbound bandwidth) | Per GB after first 100 GB free              |

The pattern:

```text
- Pay for what is RESERVED, not just what is USED, on most "managed" services
- Serverless tiers pay for what you actually USE, per request
- Storage pays per GB stored + per operation (read/write) + per egress GB
```

## 4. The "Hidden Hourly" Trap

Some services charge you **even when nothing is happening**:

```text
Resource                              Hourly charge while idle
-----------------------------------   ---------------------------------------
Running VM (Pay-As-You-Go)            ~$0.01 – $5+/hour by SKU
Stopped VM (NOT deallocated)          Same as running — must DEALLOCATE
Application Gateway / Front Door      Hourly base fee
Standard Public IP                    ~$0.005/hour each
Azure Bastion                         ~$0.19/hour
AKS Standard tier                     ~$0.10/hour cluster + node VM cost
Azure Firewall                        ~$1.25/hour
```

Lesson: **stop and deallocate VMs you are not using**, and avoid the heavyweights (Application Gateway, Bastion, Azure Firewall) unless you actually need them.

```bash
# This still bills you — VM is "stopped" but allocated:
az vm stop --resource-group rg-x --name vm-x

# This stops the bill — VM is deallocated:
az vm deallocate --resource-group rg-x --name vm-x
```

## 5. Egress (Outbound) Charges

Inbound traffic is free. Outbound is not.

```text
- First 100 GB outbound per month: FREE
- After that: ~$0.05 – $0.12 per GB depending on region
- Traffic between Azure regions: charged
- Traffic within the same region: usually free
- Traffic to the public internet: charged after 100 GB
```

Beginner traps:

```text
- Hosting a video site that streams a lot of MB
- Running automated download tests
- Mirroring a huge container registry across regions
```

Cap exposure with Azure Front Door + cache, or move heavy files to a CDN.

## 6. Spot, Reserved, and Savings Plans (Discount Options)

For cheaper compute when you commit:

```text
Option                  Discount    Trade-off
---------------------   ---------   -------------------------------------
Pay-As-You-Go           0%          No commitment, full price
Reserved Instances      ~30–60%     1- or 3-year commitment to a VM size
Savings Plan            ~10–35%     1- or 3-year commitment to $/hour spend
Spot VMs                ~60–90%     Azure can evict your VM at any time
```

Beginners should stay on Pay-As-You-Go. Move to Reserved or Savings only when you have steady, predictable usage.

## 7. The Azure Pricing Calculator

Always estimate before you create.

```text
URL: https://azure.microsoft.com/pricing/calculator
```

How to use it:

```text
1. Search for the service (e.g., "Storage Accounts")
2. Pick region, tier, redundancy, expected usage
3. Watch the monthly estimate update on the right
4. Add multiple services to one estimate
5. "Export" to Excel for project planning
```

Example: a small static site with images

```text
- Storage Account: Hot LRS, 5 GB, 100K reads/month   ≈   $0.15
- Static Web Apps: Free tier                         ≈   $0.00
- Estimated total                                    ≈   $0.15/month
```

## 8. Cost Management in the Portal

Inside the portal:

```text
Top search bar → "Cost Management + Billing"

Sections you will use:
- Cost analysis     → daily/monthly spend by service, resource, tag
- Budgets           → email/SMS alerts at thresholds (50%, 80%, 100%)
- Cost alerts       → notifications for budget breaches
- Advisor           → savings recommendations
- Reservations      → manage commitments
```

A basic recommended budget for a learner:

```text
Amount: $5/month
Alerts:
   50% → email me
   80% → email me
   100% → email + text
Filter: by subscription or resource group
```

## 9. The Pattern That Keeps the Bill Low

Repeat this pattern for every experiment:

```text
1. Create one resource group per experiment:
   az group create --name rg-experiment-001 --location eastus

2. Build everything inside that resource group.

3. Test it.

4. When done, delete the whole group:
   az group delete --name rg-experiment-001 --yes --no-wait

5. Confirm in the portal: Resource groups → it should be gone.
```

The single biggest cause of accidental Azure bills is **forgetting to delete a resource**. Resource groups make cleanup a one-liner.

## 10. Estimating Common Workloads

| Workload                                         | Realistic Monthly Cost |
|--------------------------------------------------|------------------------|
| Static portfolio site on Static Web Apps (Free)  | $0                     |
| Personal blog: App Service F1 + Blob (5 GB)      | $0 – $1                |
| Small REST API: Functions Consumption + Cosmos free | $0 – $2             |
| One Linux B1s VM running 24/7                    | ~$8 – $15              |
| Small Postgres Flexible Server (B1ms)            | ~$15 – $25             |
| Two-node AKS with B2s nodes                      | ~$60 – $120            |
| Idle Application Gateway you forgot              | ~$20+                  |
| Idle Bastion you forgot                          | ~$140+                 |

## Quick Recap

| Concept                | Key Point                                                   |
|------------------------|-------------------------------------------------------------|
| Billing formula        | Resource × Region × Tier × Usage                            |
| Free types             | Trial credit ($200/30d) vs Free tier (12mo) vs Always-free  |
| Stopping a VM          | Use `az vm deallocate`, not just stop                       |
| Egress                 | First 100 GB free, then per-GB                              |
| Cost discipline        | One experiment = one resource group = one delete            |
| Estimate first         | Use the Pricing Calculator before creating anything pricey  |
| Catch surprises        | Set a budget alert at 50% / 80% / 100%                      |
