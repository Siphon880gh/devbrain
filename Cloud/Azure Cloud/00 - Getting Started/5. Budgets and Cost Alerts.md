If you remember only one lesson from `00 - Getting Started`, make it this one. A budget alert is **the single most useful safety net** Azure gives you. It takes 3 minutes to set up and saves you from waking up to a $400 bill because you forgot to delete an Application Gateway.

This lesson covers:

```text
- The difference between a spending limit and a budget alert
- Setting a $5/month budget in 3 minutes
- Multiple alert thresholds (50% / 80% / 100%)
- Cost analysis dashboards
- Tagging strategy so you know WHAT cost money
- Stopping the bleed when an alert fires
```

---

## 1. Spending Limit vs. Budget Alert

These sound the same but behave very differently.

```text
Spending limit (free trial only)
  - Hard cap: when free credit is gone, services PAUSE
  - Removed automatically if you upgrade to Pay-As-You-Go
  - You cannot set a custom $ amount; it is tied to the credit

Budget alert
  - Soft alert: emails/SMS you when you cross a threshold
  - Does NOT automatically stop services
  - You set the $ amount and the thresholds
  - Available on every subscription type
```

In short:

```text
Spending limit  = automatic stop, free trial only
Budget alert    = manual heads-up, works on any subscription
```

You cannot rely on the spending limit forever (it disappears when you upgrade). **Always set a budget alert.**

## 2. The 3-Minute Budget Setup

```text
1. Sign in to https://portal.azure.com
2. Top search bar → type "Cost Management + Billing" → click it
3. In the left menu → click "Cost Management"
4. In the new left menu → click "Budgets"
5. Click "+ Add"
6. Fill in:
   - Scope: leave at your subscription (default)
   - Name: budget-personal-monthly
   - Reset period: Monthly
   - Creation date: today
   - Expiration date: 5 years from today
   - Amount: 5  (USD per month)
7. Click "Next"
8. Configure alerts:
   - 50% of budget → email you@example.com
   - 80% of budget → email you@example.com
   - 100% of budget → email you@example.com
9. Click "Create"
```

That is it. You will now get an email the moment your subscription crosses any of those thresholds.

---

## 3. Equivalent in the Azure CLI

If you prefer the terminal:

```bash
az consumption budget create \
  --budget-name budget-personal-monthly \
  --amount 5 \
  --time-grain monthly \
  --start-date 2026-05-01 \
  --end-date   2031-05-01 \
  --category cost \
  --notifications "{
    \"actual_GreaterThan_80_Percent\": {
      \"enabled\": true,
      \"operator\": \"GreaterThan\",
      \"threshold\": 80,
      \"contactEmails\": [\"you@example.com\"]
    }
  }"
```

(Bicep / Terraform versions show up in `06 - DevOps and Tooling`.)

## 4. Cost Analysis — See Where Money Goes

Once you have any spend at all, the **Cost analysis** dashboard shows you what is actually costing money.

```text
Cost Management + Billing → Cost Management → Cost analysis
```

Useful views to switch to:

```text
View                       What it shows
------------------------   ----------------------------------------
Accumulated cost           Running total this month
Daily cost                 Per-day spend (good for catching spikes)
Cost by service            Which Azure service is the biggest line
Cost by resource           Which individual resource costs the most
Cost by resource group     Which "project folder" costs the most
Cost by tag                Custom slicing (see tagging below)
```

The default layout is "Accumulated cost." Change it to **Daily cost** when investigating a sudden bill jump — a chart that shoots up on day 7 makes the cause obvious.

## 5. Action Groups (Send Alerts to Slack, Teams, Webhooks)

A simple budget sends emails. For team setups, use an **action group** to fan out:

```text
Email
SMS
Push notification (Azure mobile app)
Voice call
Webhook (Slack, Teams, PagerDuty)
Logic App
Azure Function
```

Setup:

```text
1. Search bar → "Action groups" → + Create
2. Add the actions you want
3. Back in your budget → set "Action Group" instead of just emails
```

For a one-person learner setup, plain email is fine.

## 6. Tagging Strategy

Tags are key/value labels you put on resources. They do not cost anything, but they make Cost Analysis enormously more useful.

A simple two-tag scheme:

```text
project = blog
env     = dev
```

Apply at create time:

```bash
az group create \
  --name rg-blog-dev \
  --location eastus \
  --tags project=blog env=dev

az storage account create \
  --name blogmediadev \
  --resource-group rg-blog-dev \
  --location eastus \
  --tags project=blog env=dev
```

Now in Cost analysis you can group by tag:

```text
Group by → Tag → project
```

…and instantly see how much "blog" cost across every service.

## 7. Spending Limit Behavior (Free Trial)

While the spending limit is on:

```text
- You cannot create services that have non-trial cost (e.g., a paid VM SKU)
- If you try, the portal blocks you with a clear error
- When credit runs out, services pause (shown as "Disabled")
- You can re-enable by upgrading to Pay-As-You-Go
```

To check or remove the spending limit:

```text
Subscriptions → click your subscription → "Remove spending limit"
```

> Removing the spending limit is a one-way door. After this, the budget alert (lesson section 2) is your only safety net. Set it FIRST.

## 8. Recommended Budget Tiers

Pick the one that matches your comfort level:

```text
Comfort level         Monthly budget        Alerts
-------------------   -------------------   ------------------------
"I am terrified"      $1                    50%, 80%, 100%
"Casual learner"      $5                    50%, 80%, 100%
"Building real apps"  $25 – $50             50%, 80%, 100%
"Small business"      $200+                 50%, 80%, 100%, 110%
```

You can have **multiple budgets** at different scopes — for example, $50 on the whole subscription and $5 per resource group.

## 9. What to Do When an Alert Fires

When you get the email:

```text
1. Open Cost Management → Cost analysis
2. Group by "Service name"  → which service jumped?
3. Group by "Resource"      → which specific resource?
4. Open that resource → consider:
      - Stop / deallocate (if VM)
      - Scale down to a smaller SKU
      - Delete if not needed
5. If you can't tell what's happening, delete the resource group
6. Check "Advisor" → it often shows the same recommendation
```

The fastest emergency stop is always:

```bash
az group delete --name <rg-name> --yes --no-wait
```

…on the resource group containing the runaway resource.

## 10. Other Cost Tools Worth Knowing

```text
Azure Advisor
   - Built-in recommendations
   - Suggests right-sizing, reserved instances, deletions
   - Free, included with every subscription

Cost Management exports
   - Export daily CSV of usage to Blob Storage
   - Useful for chargeback or external dashboards

Azure Pricing Calculator
   - Estimate BEFORE creating
   - https://azure.microsoft.com/pricing/calculator

TCO Calculator
   - Compare Azure cost vs. on-prem cost
   - https://azure.microsoft.com/pricing/tco
```
