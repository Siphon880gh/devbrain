
## All Computers should have passwords

All employee computers must be password protected.

A computer without a password is much easier for a hacker, thief, or unauthorized person to access, especially if they gain physical access or attempt remote access.

For employee workstations, make sure the assigned user has a username and password. If credentials need to be documented, they should be stored securely, not left openly visible.

For developer computers, there may be separate credentials for:

- the regular computer user
    
- the root/admin user
    

These should be kept separate. The regular user account should not automatically have root/admin access. This helps reduce risk if the regular user password is leaked, stolen, or compromised.

The key idea is:

> Protect every computer with a password, and separate normal user access from root/admin access whenever possible.


## Note About Lax Office Practices

In organizations with more relaxed security controls, the IT team may allow employees to keep a Post-it note on the laptop, monitor, or desk with the computer username and password.

This is usually not an official written policy. It is more of an informal or unwritten office practice that some workplaces allow for convenience.

It's not officially written because it can lead to legal and liability issues.

If your company is laxed and that's the standard practice, do your part by not adopting such practice. But if anything, the post-it can have a password hint that only you can decipher.

A better option is to store the password securely on your phone. However, even that has risks if you store it in a notes app, cloud-based app, or remote password app, especially if the note clearly includes the company name, computer name, username, or other identifying details. If you can store it on a local-only app on your phone and your phone is pin protected, then that's better.
