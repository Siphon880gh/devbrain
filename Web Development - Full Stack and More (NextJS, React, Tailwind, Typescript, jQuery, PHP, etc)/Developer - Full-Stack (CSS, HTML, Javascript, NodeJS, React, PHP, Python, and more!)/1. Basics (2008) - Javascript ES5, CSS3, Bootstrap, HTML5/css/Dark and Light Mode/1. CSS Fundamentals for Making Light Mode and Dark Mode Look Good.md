A good light mode and dark mode setup starts with a color system, not random color values scattered across your CSS. If every component hardcodes its own backgrounds, text colors, and borders, theme changes become messy fast. The cleaner approach is to define tokens and let components use those tokens consistently.

A helpful way to think about this is in three layers.

## 1. Palette tokens

Palette tokens are your raw colors. These are the base color values in your design system.

```css
:root {
  --slate-0: #ffffff;
  --slate-50: #f8fafc;
  --slate-500: #64748b;
  --slate-900: #0f172a;
  --blue-600: #2563eb;
  --red-600: #dc2626;
}
```

These tokens are useful as your source colors, but they do not explain where each color should be used.

## 2. Semantic tokens

Semantic tokens describe purpose. Instead of saying “use slate-900,” you say “use primary text.” Instead of saying “use slate-50,” you say “use surface background.”

```css
:root {
  --color-page-bg: var(--slate-0);
  --color-surface: var(--slate-50);
  --color-text-primary: var(--slate-900);
  --color-text-secondary: var(--slate-500);
  --color-border: #dbe3ea;
  --color-action-primary: var(--blue-600);
  --color-danger: var(--red-600);
}
```

This layer makes your CSS easier to maintain because components rely on meaning, not on specific pigment choices.

## 3. Component tokens

Component tokens are optional, but very useful when a certain UI piece needs its own mapping.

```css
:root {
  --button-primary-bg: var(--color-action-primary);
  --button-primary-text: #ffffff;
  --input-bg: var(--color-surface);
  --input-border: var(--color-border);
}
```

This layer helps when buttons, alerts, inputs, or cards need styling that is a little more specific without hardcoding color values inside the component rules.

## Why this helps with dark mode

With this structure, dark mode becomes much easier. You do not rewrite every component. You mainly remap the semantic tokens.

```css
[data-theme="dark"] {
  --color-page-bg: #0f172a;
  --color-surface: #111827;
  --color-text-primary: #f9fafb;
  --color-text-secondary: #9ca3af;
  --color-border: #374151;
  --color-action-primary: #60a5fa;
}
```

Then the components keep working because they are already using the semantic and component tokens.

```css
body {
  background: var(--color-page-bg);
  color: var(--color-text-primary);
}

.card {
  background: var(--color-surface);
  border: 1px solid var(--color-border);
}

.button-primary {
  background: var(--button-primary-bg);
  color: var(--button-primary-text);
}
```

## Final takeaway

The cleanest theme setup uses three layers:

- palette tokens for raw colors
    
- semantic tokens for color roles
    
- component tokens for specific UI elements
    

That structure makes light mode and dark mode easier to manage, easier to scale, and less likely to break as the design grows.
