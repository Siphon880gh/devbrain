The biggest source of surprise Azure charges is forgetting to clean up.

This lesson gives you copy-paste scripts and habits that make cleanup automatic.

---

# 1. The Single Most Effective Habit

Always put all resources for one experiment in one resource group, then delete the group when done.

```bash
# Create a named resource group for each session
az group create --name rg-session-$(date +%Y%m%d) --location eastus

# ... do your work inside this group ...

# Destroy everything at once
az group delete --name rg-session-20260505 --yes --no-wait
```

`--no-wait` returns immediately while deletion runs in the background.

---

# 2. End-of-Session Cleanup Script

Save this as `azure-cleanup.sh`:

```bash
#!/bin/bash
# Run at the end of a learning session
set -e

echo "=== Azure Session Cleanup ==="

# List all resource groups
echo ""
echo "Current resource groups:"
az group list --output table

echo ""
read -p "Enter resource group name to delete (or 'skip' to exit): " RG_NAME

if [ "$RG_NAME" != "skip" ]; then
    echo "Deleting resource group: $RG_NAME"
    az group delete --name "$RG_NAME" --yes --no-wait
    echo "Deletion started in background."
fi

# Check for deallocated (but not deleted) VMs
echo ""
echo "Checking for running VMs in all resource groups..."
az vm list --show-details \
  --output table \
  --query "[].{Name:name, RG:resourceGroup, State:powerState}"

echo ""
echo "Cleanup done. Check Cost Management in 24h to verify $0 ongoing charges."
```

Make it executable:

```bash
chmod +x azure-cleanup.sh
```

# 3. Deallocate All VMs in a Resource Group

```bash
# Deallocate all VMs in one resource group
for vm in $(az vm list -g $RG --query "[].name" -o tsv); do
    echo "Deallocating VM: $vm"
    az vm deallocate -g $RG -n $vm --no-wait
done
```

---

# 4. Stop All Flexible Servers (PostgreSQL / MySQL)

```bash
# Stop all PostgreSQL servers
for server in $(az postgres flexible-server list --query "[].{n:name,g:resourceGroup}" -o tsv); do
    NAME=$(echo $server | cut -f1)
    RG=$(echo $server | cut -f2)
    echo "Stopping PostgreSQL: $NAME in $RG"
    az postgres flexible-server stop -g $RG -n $NAME --no-wait
done
```

# 5. Find and Delete Orphaned Public IPs

Public IP addresses have a small hourly cost even when not attached to anything.

```bash
# Find unattached public IPs
az network public-ip list \
  --query "[?ipConfiguration == null].{Name:name, RG:resourceGroup}" \
  --output table
```

Delete them:

```bash
az network public-ip list \
  --query "[?ipConfiguration == null].[name, resourceGroup]" \
  -o tsv | while read NAME RG; do
    echo "Deleting unattached public IP: $NAME"
    az network public-ip delete --name $NAME --resource-group $RG
done
```

# 6. Find All Running VMs Across Subscription

```bash
az vm list --show-details \
  --query "[?powerState != 'VM deallocated' && powerState != 'VM stopped (deallocated)'].{Name:name,RG:resourceGroup,State:powerState,Size:hardwareProfile.vmSize}" \
  --output table
```

Any VM showing `VM running` is actively being billed.

# 7. Check Your Current Month's Cost

```bash
# Rough cost summary for current month
az consumption usage list \
  --start-date $(date -u +%Y-%m-01) \
  --end-date $(date -u +%Y-%m-%d) \
  --query "[].{Service:instanceName, Cost:pretaxCost, Currency:currency}" \
  --output table 2>/dev/null || \
  echo "Note: consumption API may require a few minutes of delay for recent charges."
```

Or just check the portal:

```text
Portal → Cost Management + Billing → Cost analysis
  → View by: Service name
  → Date range: This month
```

# 8. Set Up a Monthly Budget Alert (One-Time Setup)

```bash
az consumption budget create \
  --budget-name SafetyNet \
  --amount 10 \
  --time-grain Monthly \
  --time-period-start "$(date -u +%Y-%m-01)" \
  --time-period-end "2030-01-01" \
  --category Cost \
  --notification-key alert80 \
  --notification-threshold 80 \
  --notification-operator GreaterThan \
  --notification-contact-emails your@email.com \
  --notification-enabled true
```

At 80% of $10 ($8), you get an email. Most learning stays well under this.

# 10. Daily / Weekly Habits

```text
Daily (if actively working in Azure):
  [ ] Run az vm list --show-details to check for running VMs
  [ ] Deallocate VMs you are done with
  [ ] Delete resource groups from finished experiments

Weekly:
  [ ] Check Cost Management → Cost analysis for unexpected charges
  [ ] Look for unattached public IPs
  [ ] Confirm your budget alert is still configured

Before logging off for the weekend:
  [ ] az group delete --name $RG --yes --no-wait (for any active experiment group)
  [ ] Check for running VMs in all groups
  [ ] Stop flexible-server databases
```

# 11. Full "Nuclear Reset" Script

If you want to delete ALL resource groups in your subscription (except system-managed ones):

> Use with extreme caution. This deletes everything.

```bash
#!/bin/bash
# Delete all user-created resource groups
echo "This will delete ALL resource groups. Type 'DELETE ALL' to confirm."
read CONFIRM

if [ "$CONFIRM" = "DELETE ALL" ]; then
    for rg in $(az group list --query "[?!starts_with(name, 'MC_') && !starts_with(name, 'NetworkWatcher') && !starts_with(name, 'DefaultResource')].name" -o tsv); do
        echo "Deleting: $rg"
        az group delete --name "$rg" --yes --no-wait
    done
    echo "All deletions started in background. Check portal in ~10 minutes."
else
    echo "Aborted."
fi
```

# Quick Recap

| Habit | Command / Action |
|---|---|
| One resource group per experiment | `az group create --name rg-session-...` |
| Delete when done | `az group delete --name $RG --yes --no-wait` |
| Deallocate VMs | `az vm deallocate -g $RG -n $VM` |
| Stop flexible servers | `az postgres flexible-server stop` |
| Find unattached public IPs | `az network public-ip list --query "[?ipConfiguration == null]"` |
| Check costs | Portal → Cost Management → Cost analysis |
| Budget alert | `az consumption budget create ...` at $10/month |

The core rule:

> Build → test → **delete**. Never leave experiments running overnight unless you need to.
