Some things cannot be fully emulated locally:

```text
- Microsoft Entra ID authentication (real OAuth flows)
- Azure Managed Identity
- Azure DNS
- Application Gateway
- Azure Front Door
- Real networking scenarios (VNet peering, private endpoints)
- Service Health and alerts
```

When you need to test these, you have to deploy to Azure. This lesson shows how to minimize cost.

# 1. Use Free Tiers First

Before choosing any paid tier, check if there is a free option.

```text
Service                    Free Option
-------------------------  --------------------------------------------------------

Azure Functions            Consumption — first 1M executions/month free
Static Web Apps            Free tier — $0 for personal/hobby sites
Cosmos DB                  Free tier — 1,000 RU/s + 25 GB (one per subscription)

Blob Storage               5 GB free for 12 months (new accounts)
Container Apps             Scale to zero — $0 when idle

Application Insights       First 5 GB/day included
Azure Monitor              Metrics free; logs ~$2.76/GB after 5 GB/day free
Microsoft Entra ID         Free tier — users, groups, basic SSO, app registration
Key Vault                  $0.03 per 10,000 operations (very cheap)
```

# 2. Use the Smallest Paid SKU

When you do need a paid service, choose the smallest SKU that works for testing:

```text
Service                    Cheapest Paid SKU
-------------------------  --------------------------------------------------------

MySQL Flexible             Burstable B1ms (~$12-15/month) — stop when idle
Redis Cache                Basic C0 (~$15-20/month)
SQL Database               Serverless, 1 vCore, auto-pause 60 min
Container Registry         Basic (~$5/month)
Application Gateway        V2 Standard (~$180/month) — expensive, delete immediately
```

# 3. Stop Resources When Not Using Them

Many services can be stopped or paused to avoid charges.

### Deallocate a VM:

```bash
az vm deallocate --resource-group $RG --name $VM
```

### Stop a PostgreSQL server:

```bash
az postgres flexible-server stop --resource-group $RG --name $SERVER
```

### Stop a MySQL server:

```bash
az mysql flexible-server stop --resource-group $RG --name $SERVER
```

These pause compute billing. Disk storage still costs a small amount (pennies per GB per month).

### App Service (F1) automatically sleeps after 20 minutes of no traffic — no action needed.

### Functions Consumption plan scales to zero automatically — no action needed.

### Container Apps with `--min-replicas 0` scale to zero automatically.

---

# 4. Put Everything in One Resource Group

Always create one resource group per experiment:

```bash
az group create --name rg-experiment-001 --location eastus
```

Deploy everything into that group. When done, delete the entire group:

```bash
az group delete --name rg-experiment-001 --yes --no-wait
```

This deletes the VM, storage account, App Service, database, and any other resources inside — everything at once. You cannot forget to delete a resource if everything is in the group.

# 5. Use Cheap Regions

Some regions cost more than others for the same service. For learning:

```text
Cheapest regions (approximate):
  eastus   (East US)
  eastus2  (East US 2)
  westus   (West US)
  centralus (Central US)
```

Avoid expensive regions unless you need proximity (West Europe, Southeast Asia, etc. can cost more for some services).

# 7. Set a Budget Alert

A $5/month budget alert catches mistakes early.

```bash
az consumption budget create \
  --budget-name LearningBudget \
  --amount 5 \
  --time-grain Monthly \
  --time-period-start "2026-05-01" \
  --time-period-end "2027-05-01" \
  --category Cost \
  --notification-key alert80 \
  --notification-threshold 80 \
  --notification-operator GreaterThan \
  --notification-contact-emails your@email.com
```

Or in the portal:

```text
Portal → Cost Management + Billing → Budgets → Add
  - Amount: $5
  - Alert: 80% threshold → email notification
```

# 9. Practical "Experiments Done" Checklist

```text
[ ] Delete resource group (kills everything inside)
[ ] Check Cost Management for any orphaned resources
[ ] Remove any public IPs you created separately
[ ] Check for any DNS zones you created (tiny cost but remember to clean up)
[ ] Check that no VMs are in "stopped (allocated)" state — they still bill
```

Check for orphaned resources:

```bash
# List all resource groups
az group list --output table

# Check what's running in any suspicious resource group
az resource list --resource-group $RG --output table

# List all VMs and their power state
az vm list --show-details --output table --query "[].{Name:name,RG:resourceGroup,State:powerState}"
```

# 10. Quick Estimate: A Realistic Learning Month

Scenario: one month of active Azure learning — creating, testing, destroying resources daily.

```text
Using good practices (local emulators + free tiers + deleting after):
  App Service F1 × 1:          $0
  Functions Consumption:        $0
  Cosmos DB free tier:          $0
  Blob Storage (a few MB):      $0.01
  SQL Serverless (paused):      $0-2
  Occasional VM (a few hours):  $1-2
  DNS zone (1 month):           $0.50
  Application Insights:         $0 (under 5 GB/day)
  ------------------------------------------
  Total:                        $1.50–$5/month
```

Bad practices (leaving things running):

```text
B1s VM running 24/7 all month:       ~$15
D2s VM left on for a week:           ~$20
Application Gateway left on 3 days:  ~$18
SQL non-serverless all month:        ~$50+
------------------------------------------
Total without discipline:             $50-100+/month easily
```

The difference is 100% habit, not architecture.
