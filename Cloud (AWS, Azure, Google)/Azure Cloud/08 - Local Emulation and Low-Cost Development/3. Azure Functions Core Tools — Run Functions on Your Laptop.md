**Azure Functions Core Tools** is the local runtime for Azure Functions. It lets you write, run, and debug Functions on your laptop without deploying to Azure.

It is the exact same runtime that runs in Azure, so behavior is consistent.

---

# 1. Install

```bash
# macOS
brew tap azure/functions
brew install azure-functions-core-tools@4

# Windows
winget install Microsoft.AzureFunctionsCoreTools

# npm (cross-platform)
npm install -g azure-functions-core-tools@4 --unsafe-perm true
```

Verify:

```bash
func --version
```

---

# 2. Create a New Functions Project

```bash
# Node.js
func init MyFunctions --javascript
cd MyFunctions

# Python
func init MyFunctions --python
cd MyFunctions

# .NET
func init MyFunctions --dotnet
cd MyFunctions

# TypeScript
func init MyFunctions --typescript
cd MyFunctions
```

Project structure (Node.js):

```text
MyFunctions/
├── host.json          ← runtime configuration
├── local.settings.json ← local environment variables (not deployed)
├── package.json
└── (function folders added below)
```

---

# 3. Add a Function

```bash
# HTTP trigger (responds to HTTP requests)
func new --name HttpHello --template "HTTP trigger" --authlevel "anonymous"

# Timer trigger (runs on a schedule)
func new --name TimerJob --template "Timer trigger"

# Blob trigger (runs when a blob is uploaded)
func new --name BlobProcessor --template "Azure Blob Storage trigger"

# Queue trigger (runs when a message arrives)
func new --name QueueProcessor --template "Azure Queue Storage trigger"
```

---

# 4. Run Locally

```bash
func start
```

Output:

```text
Functions:
    HttpHello: [GET,POST] http://localhost:7071/api/HttpHello

For detailed output, run func with --verbose flag.
```

Test it:

```bash
curl "http://localhost:7071/api/HttpHello?name=Azure"
```

# 5. local.settings.json

This file holds local environment variables. It is the local equivalent of App Service Application Settings.

```json
{
    "IsEncrypted": false,
    "Values": {
        "AzureWebJobsStorage": "UseDevelopmentStorage=true",
        "FUNCTIONS_WORKER_RUNTIME": "node",
        "MY_SETTING": "local-value",
        "DATABASE_URL": "postgresql://postgres:password@localhost:5432/mydb"
    }
}
```

`AzureWebJobsStorage` is the storage account used by the Functions runtime for internal state. Locally, point it at Azurite:

```text
"AzureWebJobsStorage": "UseDevelopmentStorage=true"
```

> `local.settings.json` is in `.gitignore` by default and should never be committed.

# 6. Node.js Function Examples

### HTTP trigger

`HttpHello/index.js`:

```javascript
module.exports = async function (context, req) {
    const name = req.query.name || (req.body && req.body.name) || "World";
    context.res = {
        status: 200,
        body: `Hello, ${name}!`,
        headers: { "Content-Type": "text/plain" }
    };
};
```

### Timer trigger

`TimerJob/index.js`:

```javascript
module.exports = async function (context, myTimer) {
    context.log("Timer fired at", new Date().toISOString());
    // do your work here
};
```

`TimerJob/function.json` — adjust the cron schedule:

```json
{
    "bindings": [
        {
            "name": "myTimer",
            "type": "timerTrigger",
            "direction": "in",
            "schedule": "*/30 * * * * *"
        }
    ]
}
```

`*/30 * * * * *` = every 30 seconds (good for local testing).

# 7. Python Function Example

`HttpHello/__init__.py`:

```python
import azure.functions as func

def main(req: func.HttpRequest) -> func.HttpResponse:
    name = req.params.get("name") or "World"
    return func.HttpResponse(f"Hello, {name}!", status_code=200)
```

Run:

```bash
func start
```

# 8. Connect to Azurite for Storage Triggers

To test a Blob Storage trigger locally:

1. Start Azurite:

```bash
azurite --location ./azurite-data --silent
```

2. Set `AzureWebJobsStorage` in `local.settings.json`:

```json
"AzureWebJobsStorage": "UseDevelopmentStorage=true"
```

3. Add a Blob trigger function:

```bash
func new --name BlobProcessor --template "Azure Blob Storage trigger"
```

4. In `BlobProcessor/function.json`, set the container path:

```json
{
    "bindings": [
        {
            "name": "myBlob",
            "type": "blobTrigger",
            "direction": "in",
            "path": "uploads/{name}",
            "connection": "AzureWebJobsStorage"
        }
    ]
}
```

5. Start the function:

```bash
func start
```

6. Upload a file to Azurite's `uploads` container:

```bash
az storage container create --name uploads
az storage blob upload --container-name uploads --name test.txt --file ./test.txt
```

The BlobProcessor function fires immediately.

# 11. host.json — Runtime Configuration

`host.json` controls the Functions runtime behavior:

```json
{
    "version": "2.0",
    "logging": {
        "applicationInsights": {
            "samplingSettings": {
                "isEnabled": true,
                "excludedTypes": "Request"
            }
        }
    },
    "extensions": {
        "queues": {
            "batchSize": 16,
            "maxDequeueCount": 5,
            "visibilityTimeout": "00:01:00"
        }
    },
    "functionTimeout": "00:05:00"
}
```

Common settings:

```text
functionTimeout    → max execution time (default 5 min for Consumption)
logging            → control log verbosity and Application Insights sampling
extensions         → tune behavior of triggers (batch size, retry counts, etc.)
```
