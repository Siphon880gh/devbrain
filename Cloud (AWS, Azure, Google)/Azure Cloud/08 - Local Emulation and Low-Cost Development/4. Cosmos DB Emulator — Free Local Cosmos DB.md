The **Azure Cosmos DB Emulator** is a local version of Cosmos DB for development and testing. It runs on your machine without any Azure account or charges.

---

# 1. What It Emulates

The emulator supports:

```text
- Cosmos DB for NoSQL (JSON documents)
- Cosmos DB for MongoDB
- Cosmos DB for Cassandra
- Cosmos DB for Gremlin (graph)
- Cosmos DB for Table
```

It does not support:

```text
- Multi-region replication
- Serverless billing
- Analytical store
- Some newer features added recently
```

For local development and testing, it is sufficient for all basic operations.

---

# 2. Install

### Linux / macOS (Docker — recommended)

```bash
docker pull mcr.microsoft.com/cosmosdb/linux/azure-cosmos-emulator

docker run -d \
  --name cosmos-emulator \
  -p 8081:8081 \
  -p 10250-10255:10250-10255 \
  -e AZURE_COSMOS_EMULATOR_PARTITION_COUNT=3 \
  -e AZURE_COSMOS_EMULATOR_ENABLE_DATA_PERSISTENCE=true \
  -v cosmosdb-data:/tmp/cosmos \
  mcr.microsoft.com/cosmosdb/linux/azure-cosmos-emulator
```

Wait ~30 seconds for it to start up.

### Windows (native installer)

Download from:

```text
https://aka.ms/cosmosdb-emulator
```

Run the installer. The emulator starts automatically and appears in the system tray.

# 4. Bypass SSL Certificate Warning

The emulator uses a self-signed certificate. Most SDKs will fail SSL verification by default.

### Python:

Set the `COSMOS_EMULATOR_HOST` or disable SSL in the client:

```python
from azure.cosmos import CosmosClient

ENDPOINT = "https://localhost:8081/"
KEY = "C2y6yDjf5/R+ob0N8A7Cgv30VRDJIWEHLM+4QDU5DE2nQ9nDuVTqobD4b8mGGyPMbIZnqyMsEcaGQy67XIw/Jw=="

client = CosmosClient(url=ENDPOINT, credential=KEY, connection_verify=False)
```

### Node.js:

```javascript
import { CosmosClient } from "@azure/cosmos";

process.env.NODE_TLS_REJECT_UNAUTHORIZED = "0"; // disable SSL verification for emulator

const client = new CosmosClient({
    endpoint: "https://localhost:8081/",
    key: "C2y6yDjf5/R+ob0N8A7Cgv30VRDJIWEHLM+4QDU5DE2nQ9nDuVTqobD4b8mGGyPMbIZnqyMsEcaGQy67XIw/Jw=="
});
```

> Only disable SSL verification for local development. Never do this in production.

# 5. Data Explorer (Web UI)

The emulator includes a web-based Data Explorer:

```text
https://localhost:8081/_explorer/index.html
```

From here you can:

```text
- Create databases and containers
- Add and edit items
- Run SQL queries
- Explore the data visually
```

If your browser warns about the self-signed certificate, click "Advanced" → "Proceed anyway".

# 6. Python SDK Example

```bash
pip install azure-cosmos
```

```python
from azure.cosmos import CosmosClient, PartitionKey

ENDPOINT = "https://localhost:8081/"
KEY = "C2y6yDjf5/R+ob0N8A7Cgv30VRDJIWEHLM+4QDU5DE2nQ9nDuVTqobD4b8mGGyPMbIZnqyMsEcaGQy67XIw/Jw=="

client = CosmosClient(url=ENDPOINT, credential=KEY, connection_verify=False)

db = client.create_database_if_not_exists("mydb")
container = db.create_container_if_not_exists(
    id="users",
    partition_key=PartitionKey(path="/userId")
)

container.upsert_item({
    "id": "user-001",
    "userId": "user-001",
    "name": "Alice",
    "email": "alice@example.com"
})

item = container.read_item("user-001", partition_key="user-001")
print(item["name"])

items = container.query_items(
    query="SELECT * FROM c WHERE c.userId = @uid",
    parameters=[{"name": "@uid", "value": "user-001"}],
    enable_cross_partition_query=False
)
for i in items:
    print(i)
```

# 7. Node.js SDK Example

```bash
npm install @azure/cosmos
```

```javascript
import { CosmosClient } from "@azure/cosmos";

process.env.NODE_TLS_REJECT_UNAUTHORIZED = "0";

const client = new CosmosClient({
    endpoint: "https://localhost:8081/",
    key: "C2y6yDjf5/R+ob0N8A7Cgv30VRDJIWEHLM+4QDU5DE2nQ9nDuVTqobD4b8mGGyPMbIZnqyMsEcaGQy67XIw/Jw=="
});

const { database } = await client.databases.createIfNotExists({ id: "mydb" });
const { container } = await database.containers.createIfNotExists({
    id: "users",
    partitionKey: "/userId"
});

await container.items.upsert({
    id: "user-002",
    userId: "user-002",
    name: "Bob",
    email: "bob@example.com"
});

const { resource } = await container.item("user-002", "user-002").read();
console.log(resource.name);
```

# 8. Switch Between Emulator and Azure

Use environment variables to control which endpoint you hit:

`.env` (local):

```text
COSMOS_ENDPOINT=https://localhost:8081/
COSMOS_KEY=C2y6yDjf5/R+ob0N8A7Cgv30VRDJIWEHLM+4QDU5DE2nQ9nDuVTqobD4b8mGGyPMbIZnqyMsEcaGQy67XIw/Jw==
COSMOS_VERIFY_SSL=false
```

Production App Settings:

```text
COSMOS_ENDPOINT=https://mycosmosaccount.documents.azure.com:443/
COSMOS_KEY=<real-account-key>
COSMOS_VERIFY_SSL=true
```

Or skip the key entirely in production and use `DefaultAzureCredential()`.

# 9. Stopping and Resetting

Stop the Docker container:

```bash
docker stop cosmos-emulator
docker start cosmos-emulator    # resume with data
```

Reset (delete all data):

```bash
docker stop cosmos-emulator
docker rm cosmos-emulator
docker run -d --name cosmos-emulator ...    # fresh start
```
