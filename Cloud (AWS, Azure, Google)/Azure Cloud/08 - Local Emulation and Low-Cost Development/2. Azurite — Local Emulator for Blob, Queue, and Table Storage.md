**Azurite** is the official Azure Storage emulator for local development.

It runs on your machine and emulates:

```text
Blob Storage   → port 10000
Queue Storage  → port 10001
Table Storage  → port 10002
```

Your application code talks to Azurite exactly the same way it talks to real Azure Storage — you just change the connection string.

---

# 1. Install Azurite

### Via npm (recommended)

```bash
npm install -g azurite
```

### Via Docker

```bash
docker pull mcr.microsoft.com/azure-storage/azurite
```

### Via VS Code Extension

```text
VS Code → Extensions → search "Azurite" → Install
```

---

# 2. Start Azurite

### npm

```bash
# All services, data persisted in ./azurite-data
azurite --location ./azurite-data --silent

# Blob only
azurite-blob --location ./azurite-data

# Specific host/port
azurite --blobHost 127.0.0.1 --blobPort 10000 \
        --queueHost 127.0.0.1 --queuePort 10001 \
        --tableHost 127.0.0.1 --tablePort 10002
```

### Docker

```bash
docker run -d \
  -p 10000:10000 \
  -p 10001:10001 \
  -p 10002:10002 \
  -v $(pwd)/azurite-data:/data \
  mcr.microsoft.com/azure-storage/azurite \
  azurite --location /data --blobHost 0.0.0.0 --queueHost 0.0.0.0 --tableHost 0.0.0.0
```

### VS Code extension

```text
Ctrl/Cmd + Shift + P → Azurite: Start
```

# 3. Connection String

Azurite uses a well-known published connection string:

```text
DefaultEndpointsProtocol=http;
AccountName=devstoreaccount1;
AccountKey=Eby8vdM02xNOcqFlqUwJPLlmEtlCDXJ1OUzFT50uSRZ6IFsuFq2UVErCz4I6tq/K1SZFPTOtr/KBHBeksoGMGw==;
BlobEndpoint=http://127.0.0.1:10000/devstoreaccount1;
QueueEndpoint=http://127.0.0.1:10001/devstoreaccount1;
TableEndpoint=http://127.0.0.1:10002/devstoreaccount1;
```

Or use the shorthand that most Azure SDKs recognize:

```text
UseDevelopmentStorage=true
```

Set in your `.env`:

```text
AZURE_STORAGE_CONNECTION_STRING=UseDevelopmentStorage=true
```

# 4. Use with Azure Storage Explorer

**Azure Storage Explorer** (free desktop app) can connect to Azurite for a visual interface.

Download:

```text
https://azure.microsoft.com/features/storage-explorer
```

Connect to Azurite:

```text
Storage Explorer → Add Account (plug icon)
  → Local emulator or attached resource
  → Local storage emulator
  → Connect
```

You can now create containers, upload files, browse blobs, and delete data visually.

# 5. Use with the Azure CLI

```bash
export AZURE_STORAGE_CONNECTION_STRING="UseDevelopmentStorage=true"

# Create a container
az storage container create --name mycontainer

# Upload a file
az storage blob upload \
  --container-name mycontainer \
  --name hello.txt \
  --file ./hello.txt

# List blobs
az storage blob list --container-name mycontainer --output table

# Download a blob
az storage blob download \
  --container-name mycontainer \
  --name hello.txt \
  --file ./downloaded.txt
```

# 6. Use with Python SDK

```bash
pip install azure-storage-blob
```

```python
from azure.storage.blob import BlobServiceClient

CONN = "UseDevelopmentStorage=true"
service = BlobServiceClient.from_connection_string(CONN)

container = service.get_container_client("mycontainer")
container.create_container()

# Upload
with open("hello.txt", "rb") as f:
    container.upload_blob(name="hello.txt", data=f, overwrite=True)

# List
for blob in container.list_blobs():
    print(blob.name)

# Download
data = container.download_blob("hello.txt").readall()
print(data.decode())
```

# 7. Use with Node.js SDK

```bash
npm install @azure/storage-blob
```

```javascript
import { BlobServiceClient } from "@azure/storage-blob";

const CONN = "UseDevelopmentStorage=true";
const service = BlobServiceClient.fromConnectionString(CONN);

const container = service.getContainerClient("mycontainer");
await container.createIfNotExists();

const blob = container.getBlockBlobClient("hello.txt");
await blob.uploadData(Buffer.from("hello from azurite"), {
    blobHTTPHeaders: { blobContentType: "text/plain" }
});

for await (const item of container.listBlobsFlat()) {
    console.log(item.name);
}

const download = await blob.download(0);
const text = await streamToString(download.readableStreamBody);
console.log(text);
```

# 8. Queues with Azurite

Azurite also emulates Queue Storage.

Python:

```python
from azure.storage.queue import QueueServiceClient

CONN = "UseDevelopmentStorage=true"
qsvc = QueueServiceClient.from_connection_string(CONN)

queue = qsvc.get_queue_client("myqueue")
queue.create_queue()

queue.send_message("hello from queue")

msgs = queue.receive_messages()
for msg in msgs:
    print(msg.content)
    queue.delete_message(msg)
```

# 9. Tables with Azurite

Azurite also emulates Azure Table Storage.

```bash
pip install azure-data-tables
```

```python
from azure.data.tables import TableServiceClient

CONN = "UseDevelopmentStorage=true"
tsvc = TableServiceClient.from_connection_string(CONN)
table = tsvc.get_table_client("users")

try:
    table.create_table()
except:
    pass

table.upsert_entity({
    "PartitionKey": "us-east",
    "RowKey": "user-001",
    "Name": "Alice",
    "Email": "alice@example.com"
})

entity = table.get_entity(partition_key="us-east", row_key="user-001")
print(entity["Name"])
```

# 10. Switching Between Azurite and Real Azure

The only thing that changes between local dev and production is the connection string.

Local:

```text
AZURE_STORAGE_CONNECTION_STRING=UseDevelopmentStorage=true
```

Production:

```text
AZURE_STORAGE_CONNECTION_STRING=DefaultEndpointsProtocol=https;AccountName=...;AccountKey=...
```

Or use Managed Identity + no connection string at all (production):

```python
from azure.identity import DefaultAzureCredential
from azure.storage.blob import BlobServiceClient

# In production on Azure: uses Managed Identity automatically
# Locally: uses az login credential
service = BlobServiceClient(
    account_url="https://mystorageaccount.blob.core.windows.net",
    credential=DefaultAzureCredential()
)
```

When using `DefaultAzureCredential` locally, you connect to real Azure (your dev storage account) rather than Azurite. Use a development storage account separate from production.
