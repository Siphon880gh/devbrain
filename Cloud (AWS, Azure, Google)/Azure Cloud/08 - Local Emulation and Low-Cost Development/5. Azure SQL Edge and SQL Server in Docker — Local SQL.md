When your app uses Azure SQL Database, you need a local SQL Server-compatible database for development.

Two options:

```text
Azure SQL Edge          — ARM64 compatible (Apple Silicon M1/M2/M3)
SQL Server 2022 Linux   — Intel/AMD x86_64 only
```

Both run in Docker and are fully compatible with Azure SQL Database syntax.

---

# 1. Which Image to Use

| Machine | Use |
|---|---|
| Apple Silicon (M1/M2/M3) | `mcr.microsoft.com/azure-sql-edge` |
| Intel/AMD Mac, Linux, Windows | `mcr.microsoft.com/mssql/server` (SQL Server 2022) |

Check your architecture:

```bash
uname -m
# arm64 = Apple Silicon → use Azure SQL Edge
# x86_64 = Intel/AMD → use SQL Server
```

---

# 2. Start Azure SQL Edge (Apple Silicon / ARM64)

```bash
docker run -d \
  --name sql-local \
  -e "ACCEPT_EULA=1" \
  -e "MSSQL_SA_PASSWORD=YourStr0ng!Passw0rd" \
  -p 1433:1433 \
  mcr.microsoft.com/azure-sql-edge
```

---

# 3. Start SQL Server 2022 (Intel / AMD x86_64)

```bash
docker run -d \
  --name sql-local \
  -e "ACCEPT_EULA=Y" \
  -e "MSSQL_SA_PASSWORD=YourStr0ng!Passw0rd" \
  -p 1433:1433 \
  mcr.microsoft.com/mssql/server:2022-latest
```

---

# 4. Connection Details

```text
Server:   localhost,1433   (note: comma, not colon, for port in SQL Server syntax)
Username: sa
Password: YourStr0ng!Passw0rd
```

Connection string:

```text
Server=localhost,1433;Database=myappdb;User ID=sa;Password=YourStr0ng!Passw0rd;Encrypt=false;TrustServerCertificate=true;
```

# 5. Connect with sqlcmd

```bash
# macOS
brew install sqlcmd

# or use Docker
docker exec -it sql-local /opt/mssql-tools18/bin/sqlcmd \
  -S localhost -U sa -P "YourStr0ng!Passw0rd" -C
```

Create a database and table:

```sql
CREATE DATABASE myappdb;
GO

USE myappdb;
GO

CREATE TABLE users (
    id INT IDENTITY PRIMARY KEY,
    name NVARCHAR(100) NOT NULL,
    email NVARCHAR(200) NOT NULL,
    created_at DATETIME2 DEFAULT GETUTCDATE()
);
GO

INSERT INTO users (name, email) VALUES ('Alice', 'alice@example.com');
INSERT INTO users (name, email) VALUES ('Bob', 'bob@example.com');
GO

SELECT * FROM users;
GO
```

# 6. Connect from Python (pyodbc)

Install the ODBC driver:

```bash
# macOS
brew install msodbcsql18
pip install pyodbc
```

```python
import pyodbc

conn = pyodbc.connect(
    "DRIVER={ODBC Driver 18 for SQL Server};"
    "SERVER=localhost,1433;"
    "DATABASE=myappdb;"
    "UID=sa;"
    "PWD=YourStr0ng!Passw0rd;"
    "TrustServerCertificate=yes;"
    "Encrypt=no;"
)

cursor = conn.cursor()
cursor.execute("SELECT * FROM users")
for row in cursor.fetchall():
    print(row)
```

# 7. Connect from Node.js (mssql)

```bash
npm install mssql
```

```javascript
import sql from "mssql";

const pool = await sql.connect({
    server: "localhost",
    port: 1433,
    database: "myappdb",
    user: "sa",
    password: "YourStr0ng!Passw0rd",
    options: {
        encrypt: false,
        trustServerCertificate: true
    }
});

const result = await pool.request().query("SELECT * FROM users");
console.log(result.recordset);
```

# 9. Using the Same Code Against Azure SQL

When deploying to Azure, change only the connection string.

Local `.env`:

```text
DATABASE_URL=Server=localhost,1433;Database=myappdb;User ID=sa;Password=YourStr0ng!Passw0rd;TrustServerCertificate=True;Encrypt=False;
```

Azure App Service Application Setting:

```text
DATABASE_URL=Server=tcp:mysqlserver.database.windows.net,1433;Database=myappdb;User ID=sqladmin;Password=...;Encrypt=True;
```

Or use Managed Identity in production (no password):

```text
DATABASE_URL=Server=tcp:mysqlserver.database.windows.net,1433;Database=myappdb;Authentication=Active Directory Default;Encrypt=True;
```

# 10. Persist Data Between Restarts

By default, Docker containers lose data when removed. Mount a volume to persist:

```bash
docker run -d \
  --name sql-local \
  -e "ACCEPT_EULA=1" \
  -e "MSSQL_SA_PASSWORD=YourStr0ng!Passw0rd" \
  -p 1433:1433 \
  -v sqldata:/var/opt/mssql \
  mcr.microsoft.com/azure-sql-edge
```

Data in the `sqldata` volume persists across container restarts.

---

# 11. Reset the Database

To start with a fresh database:

```bash
docker stop sql-local
docker rm sql-local
docker volume rm sqldata    # optional: remove data too
docker run -d --name sql-local ...
```

# 12. Common Issues

| Issue | Fix |
|---|---|
| `Connection refused` on port 1433 | Container still starting; wait 30 seconds |
| `Login failed for user 'sa'` | Password does not meet complexity requirements (8+ chars, upper, lower, number, symbol) |
| SSL error | Add `TrustServerCertificate=True` and `Encrypt=False` for local |
| `arm64` image not found | Use `azure-sql-edge` instead of `mssql/server` |
| `ODBC Driver not found` | Install `msodbcsql18`: `brew install msodbcsql18` |

| Topic | Key Point |
|---|---|
| Apple Silicon | Use `mcr.microsoft.com/azure-sql-edge` |
