Application Insights is Azure Monitor's feature for instrumenting your application code.

While Log Analytics gives you infrastructure-level logs, Application Insights gives you:

```text
- Request traces (which endpoint was called, how long it took, did it succeed)
- Dependency traces (database calls, HTTP calls to external APIs)
- Exceptions and stack traces
- Custom events and metrics you define
- User flows and page views (browser-side)
- Live metrics (real-time dashboard)
- Performance profiling
```

---

# 1. Create an Application Insights Resource

```bash
RG=rg-monitoring-demo
LOC=eastus
WS=my-log-workspace
APPINS=my-app-insights

az monitor app-insights component create \
  --app $APPINS \
  --resource-group $RG \
  --location $LOC \
  --workspace $(az monitor log-analytics workspace show -g $RG -n $WS --query id -o tsv)
```

Get the **connection string** (used by your app):

```bash
az monitor app-insights component show \
  --app $APPINS \
  --resource-group $RG \
  --query connectionString \
  --output tsv
```

Output looks like:

```text
InstrumentationKey=abc123...;IngestionEndpoint=https://...;LiveEndpoint=https://...
```

# 2. Instrument a Node.js App

Install the SDK:

```bash
npm install applicationinsights
```

Add at the very top of your entry point (before any other require/import):

```javascript
import appInsights from "applicationinsights";

appInsights
    .setup(process.env.APPLICATIONINSIGHTS_CONNECTION_STRING)
    .setAutoDependencyCorrelation(true)
    .setAutoCollectRequests(true)
    .setAutoCollectExceptions(true)
    .setAutoCollectDependencies(true)
    .setAutoCollectConsole(true)
    .start();
```

Set the environment variable:

```bash
# locally
export APPLICATIONINSIGHTS_CONNECTION_STRING="InstrumentationKey=abc123..."

# App Service
az webapp config appsettings set \
  --resource-group $RG --name $APP \
  --settings APPLICATIONINSIGHTS_CONNECTION_STRING="<your-connection-string>"
```

That is it — no other code changes needed for automatic request, dependency, and exception tracking.

# 3. Instrument a Python App

Install:

```bash
pip install opencensus-ext-azure
```

At app startup:

```python
from opencensus.ext.azure.log_exporter import AzureLogHandler
from opencensus.ext.azure.trace_exporter import AzureExporter
from opencensus.trace.samplers import ProbabilitySampler
from opencensus.trace.tracer import Tracer
import logging

logger = logging.getLogger(__name__)
logger.addHandler(AzureLogHandler(
    connection_string=os.environ["APPLICATIONINSIGHTS_CONNECTION_STRING"]
))

tracer = Tracer(
    exporter=AzureExporter(
        connection_string=os.environ["APPLICATIONINSIGHTS_CONNECTION_STRING"]
    ),
    sampler=ProbabilitySampler(1.0)
)
```

For Django or Flask, there are higher-level integrations:

```bash
pip install opencensus-ext-django
# or
pip install opencensus-ext-flask
```

# 5. Custom Events and Metrics

### Node.js

```javascript
const client = appInsights.defaultClient;

// Track a custom event
client.trackEvent({
    name: "UserSignedUp",
    properties: {
        plan: "free",
        country: "US"
    }
});

// Track a custom metric
client.trackMetric({
    name: "OrderValue",
    value: 49.99
});

// Track an exception manually
try {
    // risky code
} catch (err) {
    client.trackException({ exception: err });
}
```

### Python

```python
from opencensus.ext.azure.trace_exporter import AzureExporter
from applicationinsights import TelemetryClient

client = TelemetryClient(os.environ["APPLICATIONINSIGHTS_CONNECTION_STRING"])

client.track_event("UserSignedUp", { "plan": "free" })
client.track_metric("OrderValue", 49.99)
client.flush()
```

# 7. KQL Queries in Application Insights

The underlying data tables:

```text
requests      → incoming HTTP requests handled by your app
dependencies  → outgoing calls (DB queries, HTTP calls, etc.)
exceptions    → unhandled exceptions
traces        → log messages sent by your app
customEvents  → your trackEvent() calls
pageViews     → browser-side page loads
```

### Average request duration by operation:

```kql
requests
| where timestamp > ago(1h)
| summarize avgDuration = avg(duration), Count = count() by name
| order by avgDuration desc
```

### Failed requests:

```kql
requests
| where timestamp > ago(24h)
| where success == false
| project timestamp, name, resultCode, duration, url
| order by timestamp desc
```

### Slow database calls:

```kql
dependencies
| where timestamp > ago(1h)
| where type == "SQL" or type == "postgresql"
| where duration > 500
| project timestamp, name, duration, data
| order by duration desc
```

### Exception count by type:

```kql
exceptions
| where timestamp > ago(24h)
| summarize Count = count() by type
| order by Count desc
```

### Custom events:

```kql
customEvents
| where timestamp > ago(7d)
| where name == "UserSignedUp"
| summarize Signups = count() by bin(timestamp, 1d)
| render barchart
```

# 8. Availability Tests

Application Insights can ping your app's URL from multiple locations and alert you if it is down.

```text
Portal → Application Insights → Availability → Add test
  - Test name: Homepage Ping
  - URL: https://myapp.azurewebsites.net/health
  - Frequency: every 5 minutes
  - Locations: 5 global locations
  - Failure threshold: 2 of 3 locations fail
```

If the app goes down, you get an alert immediately.

# 9. Smart Detection

Application Insights automatically detects:

```text
- Unusual increase in failed request rate
- Slow page response
- Dependency failures
- Memory leak patterns
- Security vulnerability issues (e.g., potential SQL injection patterns in request URLs)
```

These appear under:

```text
Portal → Application Insights → Smart Detection
```

No configuration needed.

# 10. Pricing

```text
Application Insights is billed through Azure Monitor / Log Analytics:
  - Data ingestion: ~$2.76/GB (same as Log Analytics)
  - First 5 GB/day per workspace: free
  - Sampling: reduce data ingestion by only sending X% of telemetry

For most small apps, you stay in the free 5 GB/day zone.
```

Enable sampling to control volume:

```javascript
appInsights.setup(...)
    .setSendLiveMetrics(true)
    .setDistributedTracingMode(appInsights.DistributedTracingModes.AI_AND_W3C)
    .start();

appInsights.defaultClient.config.samplingPercentage = 25; // send 25% of telemetry
```

|---|---|
| Application Insights | App-level telemetry: requests, dependencies, exceptions, custom events |

| KQL tables | `requests`, `dependencies`, `exceptions`, `traces`, `customEvents` |
