Azure Monitor is the umbrella service for collecting, analyzing, and acting on telemetry from Azure resources.

```text
Azure Monitor
├── Metrics          — numbers over time (CPU %, memory, requests/sec, errors)
├── Logs             — structured log records, stored in Log Analytics Workspace
├── Alerts           — trigger notifications when a metric or log condition is met
└── Insights         — pre-built dashboards per service (VM Insights, Container Insights, etc.)
```

---

# 1. Metrics

**Metrics** are numeric measurements collected at regular intervals.

Examples:

```text
VM:            CPU percentage, network bytes in/out, disk read ops
App Service:   HTTP requests, response times, HTTP 5xx errors, memory working set
Storage:       Transactions, ingress/egress bytes, availability
SQL Database:  DTU percentage, connections, deadlocks
```

View metrics:

```text
Portal → any resource → Metrics (left sidebar)
  → Select a metric (e.g., "CPU Percentage")
  → Select time range (last 1 hour, 24 hours, 7 days)
  → Chart updates automatically
```

Via CLI:

```bash
az monitor metrics list \
  --resource <resource-id> \
  --metric "Percentage CPU" \
  --output table
```

---

# 2. Log Analytics Workspace

A **Log Analytics Workspace** is the destination for logs from Azure resources.

Once you create one and configure your resources to send logs to it, you can query all logs in one place using **KQL** (Kusto Query Language).

Create a workspace:

```bash
RG=rg-monitoring-demo
LOC=eastus
WORKSPACE=my-log-workspace

az group create -n $RG -l $LOC

az monitor log-analytics workspace create \
  --resource-group $RG \
  --workspace-name $WORKSPACE \
  --location $LOC
```

# 3. Enable Diagnostic Settings

Resources do not send logs to Log Analytics automatically. You must configure **Diagnostic Settings** for each resource.

### App Service

```bash
APP_ID=$(az webapp show -g $RG -n $APP --query id -o tsv)
WS_ID=$(az monitor log-analytics workspace show -g $RG -n $WORKSPACE --query id -o tsv)

az monitor diagnostic-settings create \
  --name AppServiceLogs \
  --resource $APP_ID \
  --workspace $WS_ID \
  --logs '[{"category":"AppServiceHTTPLogs","enabled":true},{"category":"AppServiceConsoleLogs","enabled":true}]' \
  --metrics '[{"category":"AllMetrics","enabled":true}]'
```

### Storage Account

```bash
STORAGE_ID=$(az storage account show -g $RG -n $STORAGE --query id -o tsv)

az monitor diagnostic-settings create \
  --name StorageLogs \
  --resource $STORAGE_ID/blobServices/default \
  --workspace $WS_ID \
  --logs '[{"category":"StorageRead","enabled":true},{"category":"StorageWrite","enabled":true},{"category":"StorageDelete","enabled":true}]' \
  --metrics '[{"category":"Transaction","enabled":true}]'
```

### SQL Database

```bash
SQL_ID=$(az sql db show -g $RG --server $SERVER -n $DB --query id -o tsv)

az monitor diagnostic-settings create \
  --name SQLLogs \
  --resource $SQL_ID \
  --workspace $WS_ID \
  --logs '[{"category":"SQLInsights","enabled":true},{"category":"Errors","enabled":true},{"category":"Timeouts","enabled":true}]' \
  --metrics '[{"category":"Basic","enabled":true}]'
```

# 4. KQL — Kusto Query Language

Once logs are flowing to the workspace, you query them with KQL.

Open the Log Analytics query editor:

```text
Portal → Log Analytics workspaces → your workspace → Logs
```

### Basic structure

```kql
TableName
| where TimeGenerated > ago(1h)
| where column == "value"
| project TimeGenerated, column1, column2
| order by TimeGenerated desc
| limit 100
```

---

# 5. Useful KQL Queries

### App Service HTTP errors (last 24 hours):

```kql
AppServiceHTTPLogs
| where TimeGenerated > ago(24h)
| where ScStatus >= 500
| project TimeGenerated, ScStatus, CsUriStem, TimeTaken
| order by TimeGenerated desc
```

### Slowest requests:

```kql
AppServiceHTTPLogs
| where TimeGenerated > ago(1h)
| project TimeGenerated, CsUriStem, TimeTaken
| top 20 by TimeTaken desc
```

### Count requests by status code:

```kql
AppServiceHTTPLogs
| where TimeGenerated > ago(1h)
| summarize Count = count() by ScStatus
| order by Count desc
```

### VM CPU over time:

```kql
Perf
| where TimeGenerated > ago(1h)
| where ObjectName == "Processor" and CounterName == "% Processor Time"
| summarize AvgCPU = avg(CounterValue) by bin(TimeGenerated, 5m), Computer
| render timechart
```

### Find errors in a function app:

```kql
FunctionAppLogs
| where TimeGenerated > ago(6h)
| where Level == "Error"
| project TimeGenerated, FunctionName, Message
| order by TimeGenerated desc
```

### Storage operations:

```kql
StorageBlobLogs
| where TimeGenerated > ago(1h)
| summarize Count = count() by OperationName
| order by Count desc
```

# 6. Alerts

Alerts fire when a condition is met — you get notified by email, SMS, or a webhook.

### Metric alert (e.g., CPU > 80% for 5 minutes):

```bash
APP_ID=$(az webapp show -g $RG -n $APP --query id -o tsv)

az monitor metrics alert create \
  --name HighCPU \
  --resource-group $RG \
  --scopes $APP_ID \
  --condition "avg Percentage CPU > 80" \
  --window-size 5m \
  --evaluation-frequency 1m \
  --action-group my-alert-group \
  --description "App CPU above 80% for 5 minutes"
```

### Log alert (e.g., more than 10 HTTP 500s in 5 minutes):

```bash
WS_ID=$(az monitor log-analytics workspace show -g $RG -n $WORKSPACE --query id -o tsv)

az monitor scheduled-query create \
  --name HTTP500Alert \
  --resource-group $RG \
  --scopes $WS_ID \
  --condition-query "AppServiceHTTPLogs | where ScStatus >= 500 | summarize Count = count() | where Count > 10" \
  --condition-time-aggregation Count \
  --condition-threshold 10 \
  --condition-operator GreaterThan \
  --evaluation-frequency 5m \
  --window-duration 5m \
  --severity 2 \
  --action-group my-alert-group
```

### Create an action group (email notification):

```bash
az monitor action-group create \
  --resource-group $RG \
  --name my-alert-group \
  --short-name myalert \
  --action email admin-email admin@example.com
```

# 7. Dashboards

You can pin metrics charts and log query results to a shared **Azure Dashboard** in the portal.

```text
Portal → any resource → Metrics → chart → Pin to dashboard
```

Or go to:

```text
Portal → Dashboard → Edit → Add tiles
```

Dashboards are shareable within your tenant.

---

# 8. Workbooks

**Workbooks** are interactive reports in Azure Monitor — more flexible than dashboards.

```text
Portal → Azure Monitor → Workbooks → New
```

You can combine metrics charts, KQL query results, text, and parameters into a single interactive page.

# 9. Pricing

```text
Log Analytics:
  - Data ingestion: ~$2.76/GB
  - First 5 GB/day per workspace: free
  - Retention: 31 days free; longer retention has additional cost

Metrics:
  - Basic metrics: free
  - Custom metrics: ~$0.10 per metric-month

Alerts:
  - Metric alerts: ~$0.10/alert/month
  - Log alerts: ~$1.50/alert/month
```

For learning, a small project staying under 5 GB/day ingestion costs nearly nothing.

# Quick Recap

| Concept | Key Point |
|---|---|
| Azure Monitor | Umbrella for metrics, logs, and alerts |
| Metrics | Numeric time-series data; visible in resource → Metrics |
| Log Analytics Workspace | Central store for log data from multiple resources |
| Diagnostic settings | Must be enabled per resource to send logs |
| KQL | Query language for Log Analytics |
| Alerts | Fire notifications on metric or log conditions |
| Workbooks | Interactive reports combining metrics + log queries |
