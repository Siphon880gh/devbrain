The Azure Portal is great for clicking around, but real Azure work happens in the **Azure CLI** (`az`). Once you have it installed, almost every task in this curriculum has a one-line command.

This lesson covers:

```text
- Installing the Azure CLI on macOS, Linux, and Windows
- Logging in and setting your default subscription
- Useful VS Code extensions
- Azure PowerShell (alternative for Windows users)
- Bicep CLI (for infrastructure as code)
- Storage Explorer (visual blob/file browser)
- Optional: Cloud Shell (no-install browser CLI)
```

---

## 1. Why the Azure CLI?

```text
Portal:  point and click, easy to learn, hard to repeat
CLI:     scriptable, copy-pasteable, version-controllable
```

Once a workflow works in the CLI, you can:

```text
- Save it as a .sh / .ps1 file
- Commit it to git
- Re-run it tomorrow exactly the same way
- Hand it to a teammate
```

## 2. Installing the Azure CLI

### macOS (Homebrew)

```bash
brew update
brew install azure-cli
```

### macOS / Linux (one-liner)

```bash
curl -L https://aka.ms/InstallAzureCli | bash
```

### Linux (apt — Debian/Ubuntu)

```bash
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash
```

### Linux (dnf — RHEL/Fedora/CentOS)

```bash
sudo dnf install -y azure-cli
```

### Windows (winget)

```powershell
winget install -e --id Microsoft.AzureCLI
```

### Windows (MSI installer)

```text
Download from: https://aka.ms/installazurecliwindows
```

### Verify

```bash
az --version
```

You should see something like:

```text
azure-cli                         2.65.0
core                              2.65.0
telemetry                         1.1.0
```

## 3. First Login

```bash
az login
```

A browser opens. Pick your Azure account. After approving, the terminal shows your subscriptions:

```text
[
  {
    "cloudName": "AzureCloud",
    "id": "00000000-0000-0000-0000-000000000000",
    "isDefault": true,
    "name": "My Free Trial",
    "state": "Enabled",
    "user": { "name": "you@example.com", "type": "user" }
  }
]
```

If you have more than one subscription, set the default:

```bash
az account list --output table
az account set --subscription "My Free Trial"
```

Logging in inside a server / SSH session without a browser:

```bash
az login --use-device-code
```

## 4. Daily-Use CLI Commands

```bash
az account show                              # current subscription info
az account list -o table                     # all subscriptions
az group create -n rg-demo -l eastus         # create a resource group
az group list -o table                       # list all resource groups
az resource list -g rg-demo -o table         # list resources in a group
az group delete -n rg-demo --yes --no-wait   # delete the whole group
```

The two most common flags:

```text
-o table     pretty table output instead of JSON
-g <name>    target a specific resource group
```

## 5. VS Code Extensions

If you use VS Code, install these:

```text
Extension                              Why
------------------------------------   ---------------------------------------
Azure Account                          Sign in once, share auth across all
                                       Azure VS Code extensions

Azure Resources                        Browse resource groups, resources,
                                       and quick-create from sidebar

Azure Storage                          Manage Storage Accounts, Blobs,
                                       Queues, File Shares from VS Code

Azure Functions                        Create, debug, deploy Functions

Bicep                                  Syntax highlighting + IntelliSense
                                       for .bicep infrastructure files

Azurite                                Run a local Storage emulator
                                       (covered in 08 - Local Emulation)

Azure Tools                            Meta-extension that bundles many
                                       of the above
```

Install from the Extensions sidebar in VS Code, or:

```bash
code --install-extension ms-vscode.azure-account
code --install-extension ms-azuretools.vscode-azureresourcegroups
code --install-extension ms-azuretools.vscode-azurestorage
code --install-extension ms-azuretools.vscode-bicep
code --install-extension Azurite.azurite
```

## 6. Azure PowerShell (Alternative)

If you live in PowerShell:

```powershell
Install-Module -Name Az -Repository PSGallery -Force
Connect-AzAccount

Get-AzSubscription
Set-AzContext -Subscription "My Free Trial"

New-AzResourceGroup -Name rg-demo -Location eastus
Get-AzResourceGroup
Remove-AzResourceGroup -Name rg-demo -Force
```

`az` and `Az` PowerShell can both be installed; pick whichever fits your shell. The rest of this curriculum uses `az`.

## 7. Bicep CLI

Bicep is Microsoft's infrastructure-as-code language. It compiles to ARM JSON. You will not use it in `01 - Storage`, but installing it now means you are ready when you reach `06 - DevOps and Tooling`.

```bash
az bicep install
az bicep version
```

It comes bundled with the Azure CLI and updates itself when needed.

## 8. Storage Explorer (Visual Blob/File Browser)

For visually browsing Storage Accounts, this free desktop tool is excellent:

```text
Download: https://azure.microsoft.com/products/storage/storage-explorer
```

Features:

```text
- Connect to Storage Accounts via key, SAS, or Entra ID
- Drag and drop files in/out of Blob containers
- Browse File Shares, Queues, Tables
- Connect to the local Azurite emulator with one click
- Generate SAS URLs visually
```

## 9. Cloud Shell (No-Install Option)

Don't want to install anything? Use Cloud Shell.

```text
URL: https://shell.azure.com
```

You get:

```text
- A browser-based bash or PowerShell terminal
- Pre-installed: az, kubectl, terraform, helm, git, code editor
- 5 GB of Azure File storage to persist your home directory
- Free to use (you pay for the storage account that backs it: ~pennies)
```

Good for:

```text
- Demos
- Computers where you can't install software
- Quick one-offs from a phone or tablet
```

## 10. Useful CLI Tricks

### Always show output as a table

Add to `~/.azure/config`:

```ini
[core]
output = table
```

### Get JSON for one specific field

```bash
az group show -n rg-demo --query "location" -o tsv
```

### Loop over all your resource groups

```bash
for rg in $(az group list --query "[].name" -o tsv); do
  echo "Deleting $rg"
  az group delete -n "$rg" --yes --no-wait
done
```

(Be careful — this nukes everything.)

### Open the docs for any command

```bash
az storage blob upload --help
az find "az storage"
```
