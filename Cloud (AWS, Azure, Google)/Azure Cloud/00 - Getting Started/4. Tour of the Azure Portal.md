The Azure Portal at `https://portal.azure.com` is the web UI for everything. It can feel overwhelming at first because there are 200+ services. This lesson is a guided tour: where things live, what each panel does, and which shortcuts will save you the most time.

---

## 1. The Layout in Five Zones

```text
+--------------------------------------------------+
|  TOP BAR (search, Cloud Shell, notifications, +) |
+----------+--------------------------------------+
|          |                                      |
|  LEFT    |   MAIN CONTENT                       |
|  MENU    |   (Dashboard / Resource details)     |
|          |                                      |
+----------+--------------------------------------+
|  BOTTOM: Cloud Shell terminal (when toggled)     |
+--------------------------------------------------+
```

The five zones:

```text
1. Top bar      → search, Cloud Shell, notifications, settings, account
2. Left menu   → favorites, Create a resource, All services
3. Main area   → dashboard or whatever resource you opened
4. Cloud Shell  → in-browser bash/PowerShell pinned to the bottom
5. Notification → bell icon shows deployment progress + alerts
```

---

## 2. Top Bar — The Most Used Tool

The **search bar** (center top) is the fastest way around.

```text
Type "Storage" → jumps to Storage Accounts
Type "RG" → jumps to Resource groups
Type a resource name → jumps directly to that resource
Type "Cost" → jumps to Cost Management
Type "Subscriptions" → list of your subscriptions
```

If you remember nothing else from this lesson, remember:

```text
Press / (forward slash) anywhere in the portal to focus the search.
```

Other top-bar icons (right side):

```text
Icon            Purpose
-------------   ----------------------------------------
>_              Cloud Shell (terminal in browser)
🔔              Notifications (deployments, alerts)
⚙️              Portal settings (theme, language, timeouts)
?               Help & support
😀 / your name  Account, switch directory, sign out
```

---

## 3. Left Menu

Default items in the left menu:

```text
+ Create a resource     → service catalog (200+ services)
🏠 Home                 → dashboard
⭐ Favorites            → custom shortcuts you pin
🛠 All services          → searchable service directory
🗂 Subscriptions         → manage billing scopes
📁 Resource groups      → list of all resource groups
🛡 Microsoft Entra ID   → identity and access
📈 Monitor              → metrics, logs, alerts
🛒 Marketplace          → third-party + Microsoft solutions
```

Pin your most-used services to **Favorites**:

```text
1. All services
2. Hover the service you use often (e.g., "Storage accounts")
3. Click the star icon
4. It now appears in the left menu permanently
```

## 4. Subscriptions vs. Resource Groups

These two appear everywhere and are the most-confused pair.

```text
Subscription  = the BILLING boundary (pays for resources)
Resource Group = the FOLDER that holds related resources
```

A subscription contains many resource groups; a resource group contains many resources.

```text
Subscription: My Free Trial
├── Resource Group: rg-blog
│   ├── Storage Account: blogmedia
│   └── App Service: blog-app
├── Resource Group: rg-experiment-001
│   └── Function App: testfn
└── Resource Group: rg-prod
    └── ...
```

Use the search bar → "Subscriptions" to see your billing scope.
Use the search bar → "Resource groups" to see your folders.

## 5. The Resource Page

When you open any single resource (a storage account, a VM, anything), the page has the same layout:

```text
+---------------------------------------------------+
| Top: name, subscription, resource group, location |
| Tabs: Overview, Activity log, Access control, ... |
+----------+----------------------------------------+
| Left     |  Main detail panel for the selected    |
| menu of  |  tab (charts, settings, blade)         |
| sub-     |                                        |
| sections |                                        |
+----------+----------------------------------------+
```

Common left-menu sections you will see on most resources:

```text
Overview              → quick info, charts, "Move", "Delete"
Activity log           → audit log of changes
Access control (IAM)   → who can do what (RBAC)
Tags                   → key/value labels
Diagnostic settings    → send logs to Log Analytics
Properties             → resource ID, location, etc.
Locks                  → prevent accidental delete
Export template        → get the ARM/Bicep for this resource
```

## 6. Cloud Shell — Terminal in the Browser

The `>_` icon at the top opens **Cloud Shell**:

```text
- bash or PowerShell, your choice
- Pre-installed: az, kubectl, terraform, helm, git
- Files persist in a 5 GB Azure File share you create on first run
- Free to use; you pay only the storage backing it (~pennies/month)
```

Try it now:

```bash
az account show -o table
az group list -o table
```

You can also pop Cloud Shell out into a full window:

```text
URL: https://shell.azure.com
```

---

## 7. The Notifications Bell

When you create or modify resources, the bell icon shows progress:

```text
🔔 Deployment in progress: rg-demo  (Microsoft.Storage/storageAccounts)
🔔 Deployment succeeded: rg-demo
🔔 Deployment failed: rg-demo  → click for error details
```

Failed deployments link to the **Activity log**, which shows the JSON error. Most beginner deploy errors are about quotas, name conflicts, or missing role assignments.

For your wallet's safety, pin this:

Sub-pages:

- Budgets           → email/SMS alerts at thresholds

- Advisor           → personalized savings recommendations

## 9. Useful Keyboard Shortcuts

```text
G + N      Go to Notifications
G + H      Go to Home
G + B      Go to Billing
G + D      Go to Dashboard
/          Focus the global search bar
?          Show all keyboard shortcuts
```

(Type the letters in sequence, not at the same time.)

## 10. Themes, Language, and Layout

Top bar → ⚙️ (Settings) → choose:

```text
- Theme: Azure (dark blue) / Light / Dark / High contrast
- Language and region
- "Restore default settings" if the portal looks wrong
- Idle timeout (auto sign-out)
- Default startup page (Dashboard or Home)
```

---

## 11. Multiple Tenants / Directories

If you work for a company AND have a personal account, your one Microsoft identity may be in multiple **directories** (also called tenants).

```text
Top bar → click your avatar → "Switch directory"
```

Each directory has its own subscriptions, users, and Entra ID. Always check the top-right hint that says "Directory: ..." to make sure you are in the right tenant before changing things.
