When you deploy a VM, App Service, or load balancer, Azure assigns a public IP address and often a default hostname. This lesson covers public IPs, how to attach custom domains, and Azure DNS.

---

# 1. Public IP Addresses in Azure

A **public IP** is an internet-facing IP address you can attach to:

```text
- Virtual machines
- Load balancers
- Application Gateway
- Azure Bastion
- VPN Gateways
- NAT Gateways
```

Two SKUs:

| SKU | Notes |
|---|---|
| Basic | Being retired; free but limited features |
| Standard | Current default; supports Availability Zones; small hourly cost |

Standard public IP cost:

```text
~$0.004/hour per IP when associated (~$3/month)
~$0.004/hour per IP when unassociated (dissociated but still exists)
```

Delete public IPs you are not using to avoid charges.

---

# 2. Static vs Dynamic Public IPs

**Dynamic** — IP may change when the resource is deallocated and reallocated.

**Static** — IP stays the same even after deallocate/start cycles.

For production (where DNS points to an IP), always use **static**.

Create a static public IP:

```bash
RG=rg-network-demo
LOC=eastus

az network public-ip create \
  --resource-group $RG \
  --name my-static-ip \
  --allocation-method Static \
  --sku Standard \
  --location $LOC
```

Get the IP address:

```bash
az network public-ip show \
  --resource-group $RG \
  --name my-static-ip \
  --query ipAddress \
  --output tsv
```

# 3. Default Hostnames

Most Azure services get a free default hostname:

```text
App Service:          myapp.azurewebsites.net
Azure Functions:      myfunc.azurewebsites.net
Container Apps:       myapp.nicename-123.eastus.azurecontainerapps.io
Azure Load Balancer:  (none — just the public IP)
VM:                   (just the public IP, or assign a DNS label)
```

These work immediately. You do not need a custom domain for development.

Assign a DNS label to a VM's public IP (gives you a free subdomain):

```bash
az network public-ip update \
  --resource-group $RG \
  --name my-static-ip \
  --dns-name myvm-label
```

Result:

```text
myvm-label.eastus.cloudapp.azure.com
```

# 4. Custom Domains on App Service

To use your own domain (e.g., `api.mycompany.com`) on App Service:

```text
1. Own the domain (buy it from a registrar like Namecheap, GoDaddy, Cloudflare)

2. In your DNS registrar, add a CNAME record:
   Name:   api
   Value:  myapp.azurewebsites.net

   (or an A record pointing to the App Service's public IP for root domains)

3. In the portal:
   App Service → Custom domains → Add custom domain
   Type: api.mycompany.com
   Click "Validate" — Azure checks DNS propagation
   Click "Add"

4. Bind a TLS certificate:
   App Service → TLS/SSL settings
   → Certificates → Managed Certificate → Add
   (Available on Basic plan and above — creates a free certificate)
```

After this, `https://api.mycompany.com` resolves to your App Service.

Via CLI:

```bash
# Verify domain ownership first (adds a TXT record requirement)
az webapp config hostname add \
  --webapp-name $APP \
  --resource-group $RG \
  --hostname api.mycompany.com

# Add managed TLS certificate
az webapp config ssl create \
  --resource-group $RG \
  --name $APP \
  --hostname api.mycompany.com

# Bind it
az webapp config ssl bind \
  --resource-group $RG \
  --name $APP \
  --hostname api.mycompany.com \
  --ssl-type SNI
```

# 5. Azure DNS

**Azure DNS** lets you host your DNS zone in Azure, so you manage DNS records through the Azure Portal or CLI instead of your registrar's control panel.

Benefits:

```text
- Same tools (portal, CLI, ARM/Bicep) as the rest of your infrastructure
- Fast global propagation via Azure's Anycast network
- Integrates with Private DNS Zones for internal resolution
- API-accessible (automation-friendly)
```

### Create a Public DNS Zone

```bash
ZONE=mycompany.com

az network dns zone create \
  --resource-group $RG \
  --name $ZONE
```

Azure gives you 4 name servers:

```bash
az network dns zone show \
  --resource-group $RG \
  --name $ZONE \
  --query nameServers \
  --output table
```

Set these name servers at your domain registrar. After that, Azure DNS manages the zone.

### Add DNS Records

A record (maps hostname to IP):

```bash
az network dns record-set a add-record \
  --resource-group $RG \
  --zone-name $ZONE \
  --record-set-name "www" \
  --ipv4-address 52.x.x.x
```

CNAME record (alias to another hostname):

```bash
az network dns record-set cname set-record \
  --resource-group $RG \
  --zone-name $ZONE \
  --record-set-name "api" \
  --cname myapp.azurewebsites.net
```

MX record (email):

```bash
az network dns record-set mx add-record \
  --resource-group $RG \
  --zone-name $ZONE \
  --record-set-name "@" \
  --exchange mail.mycompany.com \
  --preference 10
```

TXT record (domain verification):

```bash
az network dns record-set txt add-record \
  --resource-group $RG \
  --zone-name $ZONE \
  --record-set-name "@" \
  --value "v=spf1 include:outlook.com ~all"
```

# 6. Private DNS Zones

For resources inside a VNet, you can use **Private DNS Zones** so services resolve to private IPs.

Example: a private endpoint for SQL Database needs a private DNS zone to work properly.

```bash
az network private-dns zone create \
  --resource-group $RG \
  --name privatelink.database.windows.net

az network private-dns link vnet create \
  --resource-group $RG \
  --zone-name privatelink.database.windows.net \
  --name sql-dns-link \
  --virtual-network my-vnet \
  --registration-enabled false
```

After linking, DNS queries for `mysqlserver.database.windows.net` from inside the VNet resolve to the private endpoint IP.

---

# 7. Azure DNS Pricing

```text
Public DNS zone: ~$0.50/month per zone
Queries:         $0.40 per million queries (first 1 billion)
Private DNS zone: ~$0.10/month per zone
```

Very low cost.

---

# 8. Common Patterns

```text
Pattern                          DNS Setup
-------------------------------  -----------------------------------------------
App Service with custom domain   CNAME → myapp.azurewebsites.net
VM with static IP                A record → static public IP
Load balancer with custom domain A record → load balancer public IP
Private endpoint                 Private DNS Zone → private IP
CDN endpoint                     CNAME → myapp.azureedge.net
```

# Quick Recap

| Concept | Key Point |
|---|---|
| Public IP (Standard) | Attach to VMs, LBs, gateways — ~$3/month when associated |
| Static IP | IP does not change after deallocate; use for production |
| DNS label on VM | Free `vmname.region.cloudapp.azure.com` subdomain |
| Custom domain (App Service) | Add CNAME at registrar, then add in portal + bind TLS cert |
| Azure DNS (public zone) | Host your domain's DNS in Azure; ~$0.50/month |
| Private DNS Zone | DNS resolution for private endpoints inside a VNet |
