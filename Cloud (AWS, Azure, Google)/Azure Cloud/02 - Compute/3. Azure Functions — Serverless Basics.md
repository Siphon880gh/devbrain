Azure Functions runs your code in response to a trigger.

You do not provision a server. You do not configure a runtime. You write a function, deploy it, and Azure runs it when the trigger fires.

---

# 1. How Functions Work

```text
Trigger fires
  → Azure wakes up your function (or reuses a warm instance)
    → Your code runs
      → Output bindings send results somewhere (optional)
        → Function exits
```

Common triggers:

```text
HTTP request        — like a tiny API endpoint
Timer               — cron schedule
Blob Storage event  — file uploaded to a container
Queue message       — item added to Storage Queue or Service Bus
Cosmos DB change    — document inserted or updated
Event Hub message   — streaming event arrived
```

# 2. Plans and Cost

## Consumption Plan (default)

```text
- Scales to zero when idle
- You pay only when code runs
- Price: per execution + per GB-second of memory used
- First 1,000,000 executions per month: FREE
- After: ~$0.0000002 per execution (very cheap for most workloads)
- Max execution time: 5 minutes (configurable up to 10)
```

## Flex Consumption Plan (newer, recommended)

```text
- Like Consumption but allows more concurrency control
- Faster cold starts
- Scales to zero
```

## Premium Plan

```text
- Pre-warmed instances (no cold starts)
- Longer execution timeout
- VNet integration
- Always running: costs ~$150+/month minimum
```

## App Service Plan (Dedicated)

```text
- Run Functions on an existing App Service Plan
- Good if you already have a plan and want to avoid waste
- No scale-to-zero (the plan runs 24/7)
```

For learning: use **Consumption** plan.

# 3. Create a Function App (CLI)

A **Function App** is the container that holds one or more functions.

```bash
RG=rg-func-demo
LOC=eastus
STORAGE=funcstoragelearn$RANDOM    # Functions need a storage account for metadata
FUNCAPP=myfuncapp$RANDOM

az group create -n $RG -l $LOC

az storage account create \
  --name $STORAGE \
  --resource-group $RG \
  --location $LOC \
  --sku Standard_LRS

az functionapp create \
  --name $FUNCAPP \
  --resource-group $RG \
  --storage-account $STORAGE \
  --consumption-plan-location $LOC \
  --runtime node \
  --runtime-version 20 \
  --functions-version 4 \
  --os-type Linux
```

Other runtimes: `python`, `dotnet`, `java`, `powershell`.

# 4. Write a Function Locally

Install the Functions Core Tools:

```bash
npm install -g azure-functions-core-tools@4 --unsafe-perm true
```

Create a new project:

```bash
func init MyFunctions --javascript
cd MyFunctions
```

Add an HTTP-triggered function:

```bash
func new --name HttpHello --template "HTTP trigger" --authlevel "anonymous"
```

This creates `HttpHello/index.js`:

```javascript
module.exports = async function (context, req) {
    const name = req.query.name || (req.body && req.body.name) || "World";
    context.res = {
        body: `Hello, ${name}!`
    };
};
```

And `HttpHello/function.json`:

```json
{
    "bindings": [
        {
            "authLevel": "anonymous",
            "type": "httpTrigger",
            "direction": "in",
            "name": "req",
            "methods": ["get", "post"]
        },
        {
            "type": "http",
            "direction": "out",
            "name": "res"
        }
    ]
}
```

Run locally:

```bash
func start
```

Visit:

```text
http://localhost:7071/api/HttpHello?name=Azure
```

Response:

```text
Hello, Azure!
```

# 5. Python Example

Create a Python project:

```bash
func init MyFunctions --python
cd MyFunctions
func new --name HttpHello --template "HTTP trigger" --authlevel "anonymous"
```

`HttpHello/__init__.py`:

```python
import azure.functions as func

def main(req: func.HttpRequest) -> func.HttpResponse:
    name = req.params.get("name") or "World"
    return func.HttpResponse(f"Hello, {name}!")
```

# 6. Timer-Triggered Function

Runs on a schedule using a CRON expression:

```bash
func new --name DailyReport --template "Timer trigger"
```

`DailyReport/function.json`:

```json
{
    "bindings": [
        {
            "name": "myTimer",
            "type": "timerTrigger",
            "direction": "in",
            "schedule": "0 0 9 * * *"
        }
    ]
}
```

CRON format for Functions:

```text
{second} {minute} {hour} {day} {month} {day-of-week}

0 0 9 * * *   = every day at 09:00:00 UTC
0 */5 * * * * = every 5 minutes
0 30 8 * * 1  = every Monday at 08:30:00 UTC
```

# 7. Deploy to Azure

```bash
func azure functionapp publish $FUNCAPP
```

After deploy, the function URL looks like:

```text
https://myfuncapp1234.azurewebsites.net/api/HttpHello
```

For anonymous functions, that URL works directly in a browser or curl.

For protected functions, you need a function key in the URL:

```text
https://myfuncapp1234.azurewebsites.net/api/HttpHello?code=<function-key>
```

Get the function key:

```bash
az functionapp keys list \
  --resource-group $RG \
  --name $FUNCAPP
```

# 8. Environment Variables

Set app settings (environment variables) for your Function App:

```bash
az functionapp config appsettings set \
  --resource-group $RG \
  --name $FUNCAPP \
  --settings MY_SETTING="some-value" DATABASE_URL="postgresql://..."
```

In your function code:

```javascript
const setting = process.env.MY_SETTING;
```

```python
import os
setting = os.environ["MY_SETTING"]
```

# 9. Bindings — Output to Storage

Bindings let functions read from and write to other Azure services without writing SDK code.

Example: write to Blob Storage when the function runs.

`function.json`:

```json
{
    "bindings": [
        {
            "type": "httpTrigger",
            "direction": "in",
            "name": "req",
            "authLevel": "anonymous"
        },
        {
            "type": "blob",
            "direction": "out",
            "name": "outputBlob",
            "path": "outputs/{rand-guid}.txt",
            "connection": "AzureWebJobsStorage"
        }
    ]
}
```

`index.js`:

```javascript
module.exports = async function(context, req) {
    context.bindings.outputBlob = "Content written by the function";
    context.res = { body: "Done" };
};
```

The function writes a file to Blob Storage automatically. No SDK calls needed.

# 10. Common Patterns

```text
Pattern                  Trigger               Use Case
-----------------------  -------------------   ----------------------------------------
HTTP API endpoint        HTTP trigger          REST endpoints without a full web server
Scheduled job            Timer trigger         Daily reports, cleanup, health checks
Process uploaded file    Blob trigger          Resize image, parse CSV after upload
Process queue message    Queue trigger         Async background tasks
React to DB change       Cosmos DB trigger     Sync, audit logs, notifications
```

# 11. Common Errors

| Error | Likely Cause |
|---|---|
| `Function not found` | Wrong function name in URL or failed deploy |
| Cold start delay (first request slow) | Consumption plan spinning up — expected |
| `Cannot find module` | Missing package.json dependencies; run `npm install` before deploy |
| `Function timed out` | Default 5-minute limit exceeded; increase `functionTimeout` in host.json |
| Auth error on HTTP trigger | Function key required; pass `?code=<key>` |

# Quick Recap

| Concept | Key Point |
|---|---|
| Function App | Container that holds your functions |
| Trigger | What starts the function (HTTP, timer, blob, queue, etc.) |
| Binding | Declarative connection to other services (no SDK code) |
| Consumption plan | Pay per execution; first 1M free per month |
| Cold start | First invocation after idle may be slow |
| Deploy | `func azure functionapp publish <name>` |
| Env vars | App settings via portal or `az functionapp config appsettings set` |
