An Azure Virtual Machine (VM) gives you a full operating system — Linux or Windows — running in the cloud.

Unlike App Service or Functions, you are responsible for everything inside the OS: updates, software installs, firewall rules, and disk management. In exchange, you get complete control.

---

# 1. When to Use a VM

Use a VM when:

```text
- You need software that only runs on a specific OS
- You need root/administrator access
- You are running a database server (MySQL, PostgreSQL, MongoDB) that you manage yourself
- You are doing ML model training and need GPU access
- You are hosting legacy software that cannot be containerized
- You are learning Linux system administration
- You need a custom network stack or kernel tuning
```

Do not use a VM when:

```text
- A managed service covers your needs (App Service for web apps, Cosmos DB for databases)
- You want to avoid OS maintenance
- You need scale-to-zero billing (Functions or Container Apps do that)
```

---

# 2. VM Sizes

Azure VM names encode the size family and specs.

Common beginner-friendly sizes:

| Size | vCPU | RAM | Approx. Cost |
|---|---|---|---|
| B1s | 1 | 1 GB | ~$8–$15/month |
| B1ms | 1 | 2 GB | ~$15–$25/month |
| B2s | 2 | 4 GB | ~$30–$50/month |
| B2ms | 2 | 8 GB | ~$65–$95/month |
| D2s_v5 | 2 | 8 GB | ~$80–$120/month |

B-series are **burstable** — good for workloads that are mostly idle with occasional spikes. Ideal for learning and dev/test.

D-series are general-purpose — better for consistent workloads.

# 3. Create a Linux VM (CLI)

```bash
RG=rg-vm-demo
LOC=eastus
VM=my-linux-vm

az group create -n $RG -l $LOC

az vm create \
  --resource-group $RG \
  --name $VM \
  --image Ubuntu2204 \
  --size Standard_B1s \
  --admin-username azureuser \
  --generate-ssh-keys
```

This:

```text
1. Creates the VM
2. Creates a network interface, public IP, NSG, and VNet automatically
3. Generates an SSH key pair (~/.ssh/id_rsa and id_rsa.pub) if you do not have one
4. Prints the public IP of the VM when done
```

Output includes:

```text
{
  "publicIpAddress": "52.x.x.x",
  ...
}
```

---

# 4. Connect via SSH

```bash
ssh azureuser@<publicIpAddress>
```

You are now inside the VM.

# 5. Create a Windows VM (CLI)

```bash
az vm create \
  --resource-group $RG \
  --name my-windows-vm \
  --image Win2022Datacenter \
  --size Standard_B2s \
  --admin-username azureuser \
  --admin-password "YourStr0ng!Pass"
```

Connect via Remote Desktop (RDP):

```bash
az vm open-port --resource-group $RG --name my-windows-vm --port 3389
```

Then open an RDP client and connect to the public IP on port 3389.

# 6. Start, Stop, and Deallocate

**Stop (but still paying):**

```bash
az vm stop --resource-group $RG --name $VM
```

Stopped VMs still charge for compute. Use this only if you need the VM to persist its OS state quickly.

**Deallocate (stop billing for compute):**

```bash
az vm deallocate --resource-group $RG --name $VM
```

A deallocated VM does not charge for compute. Disk storage is still billed (pennies per GB per month). Use deallocate when you are done for the day.

**Start again:**

```bash
az vm start --resource-group $RG --name $VM
```

**Restart:**

```bash
az vm restart --resource-group $RG --name $VM
```

# 7. Open a Port (NSG Rules)

By default, only port 22 (SSH) is open. To open a web server port:

```bash
az vm open-port --resource-group $RG --name $VM --port 80
az vm open-port --resource-group $RG --name $VM --port 443
```

Or open a custom range:

```bash
az network nsg rule create \
  --resource-group $RG \
  --nsg-name ${VM}NSG \
  --name allow-8080 \
  --protocol tcp \
  --priority 1001 \
  --destination-port-range 8080 \
  --access Allow
```

# 8. Attach Additional Disk

The OS disk (typically 30–64 GB) is created automatically. To add a data disk:

```bash
az vm disk attach \
  --resource-group $RG \
  --vm-name $VM \
  --name myDataDisk \
  --new \
  --size-gb 64 \
  --sku Standard_LRS
```

Then inside the VM, partition and mount it:

```bash
sudo fdisk -l                         # find the disk name (e.g., /dev/sdc)
sudo mkfs.ext4 /dev/sdc               # format it
sudo mkdir /mnt/data
sudo mount /dev/sdc /mnt/data         # mount it
echo '/dev/sdc /mnt/data ext4 defaults 0 0' | sudo tee -a /etc/fstab
```

# 9. Run a Command Without SSH (Custom Script Extension)

For automation, run a shell script on the VM without interactive SSH:

```bash
az vm run-command invoke \
  --resource-group $RG \
  --name $VM \
  --command-id RunShellScript \
  --scripts "sudo apt-get update && sudo apt-get install -y nginx && sudo systemctl start nginx"
```

# 10. Common VM Tasks After Connecting

### Install a web server (Ubuntu):

```bash
sudo apt update
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx
```

Visit `http://<publicIp>` — if port 80 is open, you will see the Nginx welcome page.

### Check disk usage:

```bash
df -h
```

### Check memory:

```bash
free -m
```

### Check running processes:

```bash
top
```

### Sudo to root:

```bash
sudo -i
```

# 11. Disk Types

| Type | SKU | Use Case |
|---|---|---|
| Standard HDD | Standard_LRS | Low-cost, infrequent access |
| Standard SSD | StandardSSD_LRS | Better IOPS, moderate workloads |
| Premium SSD | Premium_LRS | Production databases, high IOPS |
| Ultra Disk | UltraSSD_LRS | Extreme IOPS, financial/analytics |

For learning and dev: Standard SSD is sufficient. For production databases: Premium SSD.

---

# 12. Billing Warning

VMs charge by the hour — even when idle.

```text
A B1s VM stopped (not deallocated): still charges ~$8-15/month
A B1s VM deallocated: only charges for disk (~$1-3/month)
```

Always deallocate when not in use:

```bash
az vm deallocate --resource-group $RG --name $VM
```

Or delete the entire resource group when the lesson is done:

```bash
az group delete --name $RG --yes --no-wait
```

# 13. Quick VM Reference

```bash
# Create
az vm create -g $RG -n $VM --image Ubuntu2204 --size Standard_B1s \
  --admin-username azureuser --generate-ssh-keys

# Connect
ssh azureuser@<public-ip>

# Deallocate (stop billing compute)
az vm deallocate -g $RG -n $VM

# Start
az vm start -g $RG -n $VM

# List VMs in resource group
az vm list -g $RG -o table

# Show public IP
az vm show -g $RG -n $VM --show-details --query publicIps -o tsv

# Delete everything
az group delete -n $RG --yes --no-wait
```

# Quick Recap

| Topic | Key Point |
|---|---|
| VM | Full OS in the cloud — you manage the inside |
| B-series | Burstable, cheap, good for learning |
| Deallocate | Stops compute billing; always use instead of just "stop" |
| SSH | `ssh azureuser@<ip>` |
| NSG | Network Security Group — controls which ports are open |
| Disk | OS disk is auto-created; add data disks as needed |
