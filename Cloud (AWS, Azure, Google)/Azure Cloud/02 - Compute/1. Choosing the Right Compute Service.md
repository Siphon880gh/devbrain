Azure has several compute services, and picking the wrong one is one of the most common beginner mistakes. Each service exists for a different workload shape.

The four most important ones for beginners:

```text
1. App Service         — host a web app or API with no infrastructure to manage
2. Azure Functions     — run small pieces of code triggered by events
3. Virtual Machines    — full control over the OS, software, and network
4. Container Apps      — run containerized apps without managing Kubernetes
```

---

# 1. The Quick Decision Table

| Situation | Best Choice |
|---|---|
| "I have a website or REST API" | App Service |
| "I want to run code when something happens (timer, queue, HTTP)" | Azure Functions |
| "I need a full OS, or custom software that needs root access" | Virtual Machine |
| "I already have a Docker image and want to run it" | Container Apps |
| "I need Kubernetes but do not want to manage it" | AKS (advanced — see lesson 5) |

---

# 2. App Service

App Service hosts web apps and APIs in a managed environment.

You deploy code. Azure handles:

```text
- OS updates
- Load balancing
- TLS certificates
- Scaling
```

Best for:

```text
- Standard web apps (React/Vue/Angular + API backend)
- REST APIs
- Background workers that run continuously
- Apps written in Node.js, Python, PHP, .NET, Java, Ruby, Go
```

Example:

```text
You have a Node.js Express API.
You push your code.
App Service runs it.
You never touch a server.
```

Pricing model:

```text
You pay for the App Service Plan, not per request.
Even if no one visits your app, the plan costs money.
Exception: F1 free tier has no charge but has CPU/RAM limits.
```

Free tier:

```text
F1 plan: free, shared CPU, 1 GB RAM, no custom domain SSL
B1 plan: ~$13/month, dedicated, good for dev/test
```

# 3. Azure Functions

Azure Functions runs small code units triggered by events.

Best for:

```text
- Responding to an HTTP request (simple API endpoints)
- Running on a schedule (cron jobs)
- Reacting to a queue message
- Processing a blob when it arrives
- Sending emails / notifications
- Lightweight glue code
```

Key concept:

```text
You only pay when the function runs.
In the Consumption plan:
  - First 1 million executions per month are free
  - After that: fractions of a cent per execution
```

Example:

```text
Every morning at 9 AM, run a function that sends a report email.
Cost: nearly $0.
```

Not ideal for:

```text
- Apps with heavy state (use a database or App Service for that)
- Long-running tasks over 10 minutes (there are workarounds, but it gets complex)
- Apps that need a persistent web server with always-on sockets
```

---

# 4. Virtual Machines

A VM gives you a full operating system — Linux or Windows — in the cloud.

Best for:

```text
- Software that only runs on Windows
- Custom OS configuration
- Legacy apps that need specific system libraries
- Learning Linux administration
- Running your own database server
- GPU workloads (ML training, rendering)
```

You are responsible for:

```text
- OS updates (if you do not enable automatic updates)
- Security patches
- Firewall rules
- Disk management
```

Example sizes and prices (approximate):

```text
B1s  — 1 vCPU, 1 GB RAM  — ~$8-15/month
B2s  — 2 vCPU, 4 GB RAM  — ~$30-50/month
D2s  — 2 vCPU, 8 GB RAM  — ~$80-120/month
```

Important:

```text
A VM that is "stopped" (not deallocated) still charges you for compute.
To stop billing for compute, use:

  az vm deallocate --resource-group <rg> --name <vmName>

"Stop" ≠ "deallocate". Always use deallocate.
```

---

# 5. Azure Container Apps

Container Apps runs your Docker containers without managing Kubernetes.

Best for:

```text
- Microservices
- Apps you have already containerized
- Background workers
- Apps that scale to zero (no traffic = no cost)
```

Example:

```text
You have a Python Flask app in a Docker image.
Push the image to Azure Container Registry.
Deploy to Container Apps.
It scales to zero when idle.
When requests come in, it scales up automatically.
```

Scale to zero is a major advantage:

```text
App Service free tier = limited CPU/RAM, shared, sleep delays
Container Apps = truly scales to zero, only runs (and bills) when active
```

---

# 6. AKS (Azure Kubernetes Service)

AKS runs a full Kubernetes cluster in Azure.

This is an advanced topic beyond this lesson, but here is when you would use it:

```text
- You are already using Kubernetes
- You need fine-grained control over pods, namespaces, ingress, etc.
- You are running many microservices that need coordinated scheduling
- You need node-level GPU access or specialized hardware
```

AKS is expensive to run because you pay for node VMs 24/7.

For most beginners: start with Container Apps and graduate to AKS only when needed.

---

# 7. Side-By-Side Comparison

| Feature | App Service | Functions | VM | Container Apps |
|---|---|---|---|---|
| Manages the OS | Yes | Yes | No | Yes |
| Always on | Yes (paid plans) | No (scales to 0) | Yes | No (scales to 0) |
| Triggered by events | No | Yes | No | Yes (with KEDA) |
| Container support | Yes | Yes | Yes | Yes (primary) |
| Free tier | F1 plan | 1M requests/month | No | Yes (scale to 0) |
| Best for | Web apps, APIs | Event-driven code | Full OS control | Microservices |

---

# 8. Simple Decision Flowchart

```text
Is this a website or always-on API?
  └── Yes → App Service

Does it only run when triggered by an event or schedule?
  └── Yes → Azure Functions

Do you need a full OS or custom software?
  └── Yes → Virtual Machine

Do you have a Docker image and want auto-scaling?
  └── Yes → Container Apps

Are you managing many microservices with Kubernetes needs?
  └── Yes → AKS
```

---

# Quick Recap

| Service | Key Idea | Pay Model |
|---|---|---|
| App Service | Managed web host for code | Per plan (always-on) |
| Functions | Event-driven code | Per execution |
| VM | Full OS in the cloud | Per hour (even idle) |
| Container Apps | Containers without K8s | Per vCPU-second (scales to 0) |
| AKS | Full Kubernetes | Per node VM hour |

The most common beginner path:

```text
Start with App Service (free F1 plan) for web apps.
Use Functions for automation and glue code.
Add a VM only when you truly need it.
```
