Azure DevOps is Microsoft's suite of developer tools:

```text
Azure Repos    — Git repositories (alternative to GitHub)
Azure Pipelines — CI/CD pipelines (alternative to GitHub Actions)
Azure Boards   — project management / issue tracking
Azure Artifacts — package feeds (npm, NuGet, Maven, etc.)
Azure Test Plans — manual and automated testing
```

This lesson focuses on **Azure Pipelines** — the CI/CD piece.

---

# 1. Azure Pipelines vs GitHub Actions

| Feature | Azure Pipelines | GitHub Actions |
|---|---|---|
| Host | Azure DevOps (dev.azure.com) | GitHub |
| Repo | Azure Repos or GitHub or Bitbucket | GitHub |
| Config file | `azure-pipelines.yml` | `.github/workflows/*.yml` |
| Free tier | 1,800 free minutes/month (public), 1 parallel job | 2,000 minutes/month |
| Microsoft-hosted agents | Yes | Yes |
| Self-hosted agents | Yes | Yes |

If your team uses GitHub for code hosting, GitHub Actions is simpler. If your team is already in Azure DevOps (or your company mandates it), use Azure Pipelines.

# 2. Create an Azure DevOps Organization and Project

```text
1. Go to https://dev.azure.com
2. Sign in with your Azure account
3. Click "New organization" → create one
4. Inside the organization, click "New project"
   - Name: my-azure-project
   - Visibility: Private
5. Done — you have an Azure DevOps project
```

# 3. A Minimal Pipeline File

Pipelines are defined in `azure-pipelines.yml` at the root of your repository.

Minimal example — run tests on every push:

```yaml
trigger:
  - main

pool:
  vmImage: ubuntu-latest

steps:
  - task: NodeTool@0
    inputs:
      versionSpec: '20.x'
    displayName: 'Install Node.js'

  - script: npm ci
    displayName: 'Install dependencies'

  - script: npm test
    displayName: 'Run tests'
```

---

# 4. Key Concepts

```text
Trigger     → what event starts the pipeline (push, pull request, schedule)
Pool        → which agent (hosted Ubuntu/Windows/macOS, or self-hosted)
Stage       → a group of jobs (e.g., Build, Test, Deploy)
Job         → a unit of work that runs on one agent
Step        → individual actions inside a job (tasks or scripts)
Task        → a pre-built action (like NodeTool, DotNetCoreCLI, AzureWebApp)
Artifact    → output from a job that can be used by later jobs/stages
```

# 5. Connect Azure DevOps to Azure

To deploy resources, the pipeline needs an Azure connection.

```text
Azure DevOps project → Project settings (bottom-left)
  → Service connections → New service connection → Azure Resource Manager
  → Workload Identity Federation (recommended) or Service principal
  → Select your subscription and resource group
  → Name the connection: my-azure-connection
```

Use this connection name in pipeline tasks.

# 6. Build and Deploy to App Service

```yaml
trigger:
  - main

variables:
  azureSubscription: 'my-azure-connection'
  appName: 'my-appservice-name'
  resourceGroup: 'rg-myapp'

stages:
  - stage: Build
    displayName: Build
    jobs:
      - job: BuildJob
        pool:
          vmImage: ubuntu-latest
        steps:
          - task: NodeTool@0
            inputs:
              versionSpec: '20.x'

          - script: npm ci
            displayName: Install dependencies

          - script: npm run build --if-present
            displayName: Build

          - task: ArchiveFiles@2
            inputs:
              rootFolderOrFile: '$(System.DefaultWorkingDirectory)'
              includeRootFolder: false
              archiveType: zip
              archiveFile: '$(Build.ArtifactStagingDirectory)/app.zip'

          - publish: '$(Build.ArtifactStagingDirectory)/app.zip'
            artifact: drop

  - stage: Deploy
    displayName: Deploy to App Service
    dependsOn: Build
    jobs:
      - deployment: DeployApp
        environment: production
        pool:
          vmImage: ubuntu-latest
        strategy:
          runOnce:
            deploy:
              steps:
                - task: AzureWebApp@1
                  inputs:
                    azureSubscription: '$(azureSubscription)'
                    appType: webAppLinux
                    appName: '$(appName)'
                    package: '$(Pipeline.Workspace)/drop/app.zip'
```

# 7. Deploy to Azure Functions

```yaml
trigger:
  - main

variables:
  azureSubscription: 'my-azure-connection'
  functionAppName: 'my-function-app'

pool:
  vmImage: ubuntu-latest

steps:
  - task: NodeTool@0
    inputs:
      versionSpec: '20.x'

  - script: npm ci

  - task: ArchiveFiles@2
    inputs:
      rootFolderOrFile: '$(System.DefaultWorkingDirectory)'
      includeRootFolder: false
      archiveType: zip
      archiveFile: '$(Build.ArtifactStagingDirectory)/func.zip'

  - task: AzureFunctionApp@2
    inputs:
      azureSubscription: '$(azureSubscription)'
      appType: functionAppLinux
      appName: '$(functionAppName)'
      package: '$(Build.ArtifactStagingDirectory)/func.zip'
```

# 8. Run CLI Commands in a Pipeline

```yaml
- task: AzureCLI@2
  inputs:
    azureSubscription: 'my-azure-connection'
    scriptType: bash
    scriptLocation: inlineScript
    inlineScript: |
      az group list --output table
      az storage account list --output table
```

# 9. Environments and Approvals

An **environment** in Azure DevOps tracks deployment history and supports manual approval gates.

Create an environment:

```text
Azure DevOps → Pipelines → Environments → New environment
  - Name: production
```

Add an approval rule:

```text
Environments → production → Approvals and checks → + → Approvals
  → Add approvers (yourself or your team lead)
```

When the pipeline reaches the deploy-to-production stage, it pauses and waits for a human to approve before proceeding.

# 10. Scheduled Pipelines

```yaml
schedules:
  - cron: "0 2 * * *"        # every day at 2:00 AM UTC
    displayName: Nightly build
    branches:
      include:
        - main
    always: true              # run even if nothing changed

trigger: none                 # disable push trigger
```

# 11. Variable Groups (Secrets)

Store secrets in a **variable group** and reference them in pipelines.

```text
Azure DevOps → Pipelines → Library → Variable groups → New variable group
  - Name: myapp-secrets
  - Add variable: DATABASE_URL = postgresql://...  (lock the value)
```

Reference in the pipeline:

```yaml
variables:
  - group: myapp-secrets

steps:
  - script: echo "$(DATABASE_URL)"
    displayName: Use secret
```

You can also link a variable group to Azure Key Vault so secrets are pulled automatically.

---

# 12. Azure DevOps vs GitHub Actions Summary

If you are starting fresh on Azure with a GitHub repo, **GitHub Actions** (previous lesson) is simpler and requires fewer setup steps.

Use Azure Pipelines when:

```text
- Your organization mandates Azure DevOps
- You are already using Azure Repos or Azure Boards
- You need multi-stage pipelines with approval gates and complex artifact management
- You need self-hosted agents for on-premises resources
```
