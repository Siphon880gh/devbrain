GitHub Actions lets you automate the build and deploy of your app to Azure on every push to main (or any trigger you choose).

This lesson shows the two most common patterns: deploying to **App Service** and deploying to **Azure Functions**, using a service principal for auth.

---

# 1. How It Works

```text
Push to GitHub (main branch)
  → GitHub Actions workflow starts
    → Build the app
    → Authenticate to Azure
    → Deploy to App Service / Functions / Container Registry / etc.
      → App is live on Azure
```

---

# 2. Create a Service Principal for GitHub Actions

GitHub Actions needs permission to deploy to Azure. The cleanest way is a **service principal** — an identity for the workflow.

```bash
SUB_ID=$(az account show --query id -o tsv)
RG=rg-myapp

az ad sp create-for-rbac \
  --name "github-actions-myapp" \
  --role Contributor \
  --scopes /subscriptions/$SUB_ID/resourceGroups/$RG \
  --json-auth
```

This prints a JSON blob:

```json
{
    "clientId": "...",
    "clientSecret": "...",
    "subscriptionId": "...",
    "tenantId": "...",
    ...
}
```

Copy the entire JSON output.

---

# 3. Store the Credentials in GitHub Secrets

```text
GitHub repo → Settings → Secrets and variables → Actions → New repository secret

Name:  AZURE_CREDENTIALS
Value: <paste the full JSON from above>
```

Also add:

```text
AZURE_WEBAPP_NAME    = your-app-service-name
AZURE_RESOURCE_GROUP = rg-myapp
```

# 4. Deploy to App Service — Node.js Example

`.github/workflows/deploy-appservice.yml`:

```yaml
name: Deploy to Azure App Service

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build --if-present

      - name: Log in to Azure
        uses: azure/login@v2
        with:
          creds: ${{ secrets.AZURE_CREDENTIALS }}

      - name: Deploy to App Service
        uses: azure/webapps-deploy@v3
        with:
          app-name: ${{ secrets.AZURE_WEBAPP_NAME }}
          package: .
```

# 6. Deploy to Azure Functions

```yaml
name: Deploy to Azure Functions

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Log in to Azure
        uses: azure/login@v2
        with:
          creds: ${{ secrets.AZURE_CREDENTIALS }}

      - name: Deploy to Azure Functions
        uses: Azure/functions-action@v1
        with:
          app-name: ${{ secrets.AZURE_FUNCAPP_NAME }}
          package: .
```

# 7. Deploy a Container Image

Build and push to Azure Container Registry, then deploy to Container Apps:

```yaml
name: Build and Deploy Container

on:
  push:
    branches: [main]

env:
  REGISTRY: myregistry.azurecr.io
  IMAGE: my-app

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Log in to Azure
        uses: azure/login@v2
        with:
          creds: ${{ secrets.AZURE_CREDENTIALS }}

      - name: Log in to ACR
        run: az acr login --name myregistry

      - name: Build and push image
        run: |
          docker build -t $REGISTRY/$IMAGE:${{ github.sha }} .
          docker push $REGISTRY/$IMAGE:${{ github.sha }}

      - name: Deploy to Container Apps
        run: |
          az containerapp update \
            --name my-container-app \
            --resource-group ${{ secrets.AZURE_RESOURCE_GROUP }} \
            --image $REGISTRY/$IMAGE:${{ github.sha }}
```

# 8. OpenID Connect (OIDC) — No Stored Credentials

A more secure approach avoids storing the service principal JSON as a secret entirely, using **federated credentials** (OIDC).

Create a federated identity credential:

```bash
APP_ID=$(az ad app create --display-name "github-actions-oidc" --query appId -o tsv)
SP_ID=$(az ad sp create --id $APP_ID --query id -o tsv)

az ad app federated-credential create --id $APP_ID --parameters '{
  "name": "github-main",
  "issuer": "https://token.actions.githubusercontent.com",
  "subject": "repo:yourorg/yourrepo:ref:refs/heads/main",
  "audiences": ["api://AzureADTokenExchange"]
}'

az role assignment create \
  --assignee $SP_ID \
  --role Contributor \
  --scope /subscriptions/$SUB_ID/resourceGroups/$RG
```

Add GitHub secrets (no clientSecret needed):

```text
AZURE_CLIENT_ID     = <APP_ID>
AZURE_TENANT_ID     = <TENANT_ID>
AZURE_SUBSCRIPTION_ID = <SUB_ID>
```

Workflow login step:

```yaml
- name: Log in to Azure (OIDC)
  uses: azure/login@v2
  with:
    client-id: ${{ secrets.AZURE_CLIENT_ID }}
    tenant-id: ${{ secrets.AZURE_TENANT_ID }}
    subscription-id: ${{ secrets.AZURE_SUBSCRIPTION_ID }}
```

No secret rotation needed — credentials are short-lived tokens issued per job.

---

# 9. Run CLI Commands in a Workflow

After the login step, `az` commands work directly:

```yaml
- name: Log in to Azure
  uses: azure/login@v2
  with:
    creds: ${{ secrets.AZURE_CREDENTIALS }}

- name: Delete old blobs
  run: |
    az storage blob delete-batch \
      --account-name mystorageaccount \
      --source backups \
      --pattern "*.log"
```

# 10. Auto-Generated Workflows from App Service

If you link GitHub to App Service via the portal's Deployment Center, Azure generates a workflow file for you:

```text
App Service → Deployment Center → GitHub → authorize → select repo and branch → Save
```

Azure creates `.github/workflows/main_yourappname.yml` in your repo. This is a quick way to get started — you can then customize it.

# Quick Recap

| Step | Key Point |
|---|---|
| Service principal | `az ad sp create-for-rbac --json-auth` |
| GitHub secret | Store the JSON as `AZURE_CREDENTIALS` |
| Login action | `azure/login@v2` with `creds: ${{ secrets.AZURE_CREDENTIALS }}` |
| App Service deploy | `azure/webapps-deploy@v3` |
| Functions deploy | `Azure/functions-action@v1` |
| Container deploy | `az acr login` → `docker build/push` → `az containerapp update` |
| OIDC (no stored secret) | Federated credentials — more secure for production |
