Azure SQL Database is a fully managed relational database based on SQL Server.

You write standard SQL. Azure handles backups, high availability, patching, and scaling.

---

# 1. Key Concepts

```text
SQL Server (logical server)
└── Azure SQL Database (one or more databases on the server)
```

A **logical server** is the management endpoint — it holds the admin credentials and firewall rules. It is not a real VM; it is just a namespace.

A **database** is the actual database where your tables, views, and data live.

---

# 2. Create a SQL Server and Database (CLI)

```bash
RG=rg-sql-demo
LOC=eastus
SERVER=mysqlserver$RANDOM       # globally unique
DB=myappdb
ADMIN=sqladmin
PASSWORD="YourStr0ng!Pass123"

az group create -n $RG -l $LOC

az sql server create \
  --name $SERVER \
  --resource-group $RG \
  --location $LOC \
  --admin-user $ADMIN \
  --admin-password $PASSWORD

az sql db create \
  --resource-group $RG \
  --server $SERVER \
  --name $DB \
  --edition GeneralPurpose \
  --compute-model Serverless \
  --family Gen5 \
  --capacity 1 \
  --auto-pause-delay 60
```

Key flags:

```text
--edition GeneralPurpose      The vCore-based tier
--compute-model Serverless    Scales to zero when idle
--capacity 1                  1 vCore (minimum for serverless)
--auto-pause-delay 60         Pause after 60 minutes of inactivity
```

The serverless tier pauses compute when no connections are active, saving money for dev/test.

Free offer alternative (one per subscription):

```bash
az sql db create \
  --resource-group $RG \
  --server $SERVER \
  --name $DB \
  --free-limit-exhaustion-behavior AutoPause \
  --use-free-limit
```

# 3. Open the Firewall

By default, the SQL server rejects all connections.

Allow your current IP:

```bash
MY_IP=$(curl -s https://ifconfig.me)

az sql server firewall-rule create \
  --resource-group $RG \
  --server $SERVER \
  --name AllowMyIP \
  --start-ip-address $MY_IP \
  --end-ip-address $MY_IP
```

Allow Azure services (so App Service, Functions, etc. can connect):

```bash
az sql server firewall-rule create \
  --resource-group $RG \
  --server $SERVER \
  --name AllowAzureServices \
  --start-ip-address 0.0.0.0 \
  --end-ip-address 0.0.0.0
```

> The `0.0.0.0 – 0.0.0.0` rule is a special Azure shorthand for "allow all Azure services." It does not mean open to the entire internet.

# 4. Connect with sqlcmd

Install sqlcmd:

```bash
# macOS
brew install sqlcmd

# Windows
winget install Microsoft.Sqlcmd

# Docker (no install needed)
docker run -it mcr.microsoft.com/mssql-tools /bin/bash
```

Connect:

```bash
sqlcmd -S ${SERVER}.database.windows.net \
  -d $DB \
  -U $ADMIN \
  -P $PASSWORD \
  -N yes
```

Once connected, run SQL:

```sql
CREATE TABLE users (
    id INT IDENTITY PRIMARY KEY,
    name NVARCHAR(100) NOT NULL,
    email NVARCHAR(200) NOT NULL,
    created_at DATETIME2 DEFAULT GETUTCDATE()
);

INSERT INTO users (name, email) VALUES ('Alice', 'alice@example.com');
INSERT INTO users (name, email) VALUES ('Bob', 'bob@example.com');

SELECT * FROM users;
GO
```

# 5. Connection String

Your app needs the connection string to connect.

Get it:

```bash
az sql db show-connection-string \
  --server $SERVER \
  --name $DB \
  --client ado.net
```

Returns something like:

```text
Server=tcp:mysqlserver1234.database.windows.net,1433;
Initial Catalog=myappdb;
Persist Security Info=False;
User ID=sqladmin;
Password={your_password};
MultipleActiveResultSets=False;
Encrypt=True;
TrustServerCertificate=False;
Connection Timeout=30;
```

For Python (pyodbc):

```python
import pyodbc

conn = pyodbc.connect(
    "DRIVER={ODBC Driver 18 for SQL Server};"
    f"SERVER={server}.database.windows.net;"
    f"DATABASE={db};"
    f"UID={admin};"
    f"PWD={password};"
    "Encrypt=yes;"
)

cursor = conn.cursor()
cursor.execute("SELECT * FROM users")
for row in cursor:
    print(row)
```

For Node.js (mssql):

```javascript
import sql from "mssql";

const pool = await sql.connect({
    server: "mysqlserver1234.database.windows.net",
    database: "myappdb",
    user: "sqladmin",
    password: "YourStr0ng!Pass123",
    options: { encrypt: true }
});

const result = await pool.request().query("SELECT * FROM users");
console.log(result.recordset);
```

# 6. Managed Identity (No Password in Code)

Storing passwords in connection strings is a risk. A safer pattern uses **Managed Identity** — the App Service or Function authenticates to SQL without a password.

Set up (covered in detail in `05 - Identity and Security / 3`):

```bash
# Enable managed identity on your App Service
az webapp identity assign --resource-group $RG --name $APP

# Get the identity's object ID
OID=$(az webapp identity show --resource-group $RG --name $APP --query principalId -o tsv)

# Grant the identity db_datareader + db_datawriter roles
# (run this SQL inside the database as the admin)
```

```sql
CREATE USER [mywebapp] FROM EXTERNAL PROVIDER;
ALTER ROLE db_datareader ADD MEMBER [mywebapp];
ALTER ROLE db_datawriter ADD MEMBER [mywebapp];
```

Then your connection string does not need a password — it uses the identity token automatically.

# 7. Backup and Restore

Azure SQL Database creates automated backups:

```text
- Full backup: weekly
- Differential backup: every 12–24 hours
- Transaction log backup: every 5–12 minutes
```

Point-in-time restore:

```bash
az sql db restore \
  --dest-name myappdb-restored \
  --edition GeneralPurpose \
  --resource-group $RG \
  --server $SERVER \
  --name $DB \
  --time "2026-05-01T10:00:00"
```

# 8. Pricing Reminder

| Model | Notes |
|---|---|
| Serverless (vCore) | Pauses when idle; good for dev/test |
| Provisioned (vCore) | Always-on compute; predictable but more expensive |
| DTU-based | Simpler bundles; Basic (~$5/month), Standard, Premium |
| Free offer | 100,000 vCore-seconds + 32 GB/month; one per subscription |

For learning: use the free offer or the serverless tier with `--auto-pause-delay 60`.

---

# 9. Common Errors

| Error | Likely Cause |
|---|---|
| `Cannot open server ... firewall` | Your IP is not in the firewall rules |
| `Login failed for user` | Wrong username or password |
| `Database is paused` | Serverless auto-pause; first connection resumes it (takes ~30 sec) |
| `SSL error` | Missing `Encrypt=true` in connection string |
| `Timeout expired` | Serverless resuming from pause; retry with a longer timeout |

# Quick Recap

| Topic | Key Point |
|---|---|
| Logical server | Management endpoint, holds firewall and admin credentials |
| SQL Database | The actual database on the server |
| Serverless tier | Pauses compute when idle — best for dev/test |
| Firewall | Must add your IP or `0.0.0.0` for Azure services to connect |
| Connection string | Includes server, database, user, password, `Encrypt=true` |
| Managed Identity | No-password auth — preferred for production |
| Backups | Automated, up to 35-day retention, point-in-time restore |
