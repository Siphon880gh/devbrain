Azure has many database options. Picking the wrong one wastes money and creates pain later.

This lesson maps the most common database choices to use cases, so you can pick confidently.

---

# 1. The Main Options

```text
Azure SQL Database        — managed SQL Server (relational)
Azure Database for PostgreSQL — managed PostgreSQL (relational)
Azure Database for MySQL  — managed MySQL (relational)
Azure Cosmos DB           — globally distributed NoSQL (JSON documents, key-value, etc.)
Azure Table Storage       — simple key-value, very cheap
Azure Cache for Redis     — in-memory cache, session store, pub/sub
```

For most beginner projects, the choice is between **Azure SQL Database**, **PostgreSQL**, and **Cosmos DB**.

---

# 2. Quick Decision Table

| Question | Recommendation |
|---|---|
| "I already know SQL and want a familiar relational DB" | Azure SQL Database or PostgreSQL |
| "My app currently uses PostgreSQL locally" | Azure Database for PostgreSQL |
| "My app currently uses MySQL locally" | Azure Database for MySQL |
| "I need to store flexible JSON documents (no fixed schema)" | Cosmos DB |
| "I need global read/write in multiple regions" | Cosmos DB |
| "I need very low-cost storage for simple key-value data" | Azure Table Storage |
| "I need a cache layer or session store" | Azure Cache for Redis |

---

# 3. Azure SQL Database

A fully managed SQL Server database in Azure.

Best for:

```text
- Apps that already use SQL Server or T-SQL
- Complex queries, joins, transactions
- .NET applications that use Entity Framework
- Reporting and analytics
- Anything that benefits from stored procedures, views, triggers
```

You do not manage:

```text
- OS patching
- SQL Server version upgrades
- High availability (built in with 99.99% SLA)
- Backups (automated)
```

You still manage:

```text
- Schema design
- Query optimization
- Firewall rules (which IPs can connect)
```

Pricing models:

```text
DTU-based:      Simpler, bundled compute + storage
vCore-based:    Flexible, separate compute and storage billing
Serverless:     Scales to zero compute (pauses when idle, best for dev/test)
Free offer:     100,000 vCore-seconds + 32 GB / month (one per subscription)
```

For learning: use the **serverless tier** or the **free offer**.

---

# 4. Azure Database for PostgreSQL

A fully managed PostgreSQL service.

Best for:

```text
- Apps that already use PostgreSQL
- Open-source stacks (Django, Rails, FastAPI, Node + Sequelize, etc.)
- GIS / spatial data (PostGIS)
- Apps using PostgreSQL-specific features (JSON operators, window functions, etc.)
```

Two deployment options:

```text
Flexible Server (recommended):
  - More control over configuration
  - Better cost management
  - Stop/start support (deallocate when idle)

Single Server (legacy):
  - Being retired — do not start new projects on this
```

Pricing starts around $12-15/month for the smallest Burstable tier. Stop the server when not in use to avoid charges:

```bash
az postgres flexible-server stop \
  --resource-group $RG \
  --name $SERVER
```

---

# 5. Azure Database for MySQL

A fully managed MySQL service.

Best for:

```text
- Apps that already use MySQL
- WordPress, Drupal, Laravel, and similar PHP stacks
- Apps using MySQL-specific syntax
```

Works similarly to PostgreSQL Flexible Server. Use the **Burstable** tier for dev/test.

---

# 6. Azure Cosmos DB

Cosmos DB is a globally distributed, multi-model NoSQL database.

Data model options:

```text
NoSQL (JSON documents)  — most common
MongoDB API             — use if your app already targets MongoDB
Cassandra API           — for Cassandra workloads
Gremlin API             — graph database
Table API               — compatible with Azure Table Storage
```

Best for:

```text
- Document storage with flexible schemas
- Apps that need low-latency reads globally (data replicated to multiple regions)
- Event sourcing, IoT, gaming leaderboards, user profiles
- Apps already using MongoDB
- High-volume workloads where you pay per operation (RU/s)
```

Not ideal for:

```text
- Complex joins or ad-hoc aggregation queries
- Apps that need full SQL compliance
- Small apps where Cosmos overhead is not worth it
```

Free tier:

```text
1,000 RU/s + 25 GB storage — free forever (one per Azure account)
```

For many small apps, the free tier is sufficient.

---

# 7. Azure Table Storage

Simple key-value / wide-column storage.

Best for:

```text
- Very cheap storage of structured data with simple lookups
- Audit logs, telemetry, user-data rows
- When you do not need joins or complex queries
```

Not a replacement for a real database. Querying is limited to partition key and row key lookups. No secondary indexes.

Cost:

```text
~$0.07 per GB / month
Nearly free for small amounts of data
```

---

# 8. Azure Cache for Redis

An in-memory data store based on Redis.

Best for:

```text
- Caching database query results
- Session storage for web apps
- Rate limiting counters
- Pub/sub messaging
- Leaderboards (sorted sets)
```

Not a primary database. Redis data can be lost if the cache is restarted (unless persistence is configured).

# 9. Cost Comparison (Approximate)

| Service | Cheapest Dev Option | Approximate Cost |
|---|---|---|
| Azure SQL Database | Serverless or Free offer | $0–$5/month when idle |
| PostgreSQL Flexible | Burstable B1ms, stopped when idle | $0–$15/month |
| MySQL Flexible | Burstable B1ms, stopped when idle | $0–$15/month |
| Cosmos DB | Free tier (1,000 RU/s + 25 GB) | $0 |
| Table Storage | Pay per GB | ~$0.07/GB |
| Redis Cache | Basic C0 (250 MB) | ~$15–$20/month |

# 10. Simple Mental Model

```text
Need SQL and already know it?
  → Azure SQL Database (familiar, powerful, good .NET support)

Using an open-source stack (Django, Rails, FastAPI)?
  → PostgreSQL Flexible Server

Using a PHP stack (WordPress, Laravel)?
  → MySQL Flexible Server

Need flexible JSON storage, global distribution, or MongoDB API?
  → Cosmos DB

Need a cache?
  → Redis Cache

Need dirt-cheap key-value?
  → Table Storage
```
