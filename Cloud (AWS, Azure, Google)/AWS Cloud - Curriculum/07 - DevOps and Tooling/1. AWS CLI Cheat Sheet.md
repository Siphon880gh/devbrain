# AWS CLI Cheat Sheet

The AWS Command Line Interface (CLI) lets you control AWS services directly from your terminal. Instead of clicking through the Console, you type commands — which is faster, scriptable, and repeatable. This lesson covers the commands you'll use most often.

## What You'll Learn

- How to install and configure the AWS CLI
- Essential commands for S3, EC2, Lambda, IAM, and DynamoDB
- How to format and filter output with `--query` and `--output`
- How to work with named profiles for multiple AWS accounts

## Installation and Configuration

### Installing the AWS CLI

On macOS (using Homebrew):

```bash
brew install awscli
```

On Linux:

```bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
```

On Windows, download the MSI installer from https://aws.amazon.com/cli/ and run it.

Verify the installation:

```bash
aws --version
# aws-cli/2.x.x Python/3.x.x ...
```

### Initial Configuration

Run the configure command to set up your credentials:

```bash
aws configure
```

It prompts you for four values:

```
AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: us-east-1
Default output format [None]: json
```

This creates two files in `~/.aws/`:
- `credentials` — your access key and secret key
- `config` — your default region and output format

### Named Profiles

If you work with multiple AWS accounts (personal, work, staging), set up named profiles:

```bash
aws configure --profile staging
```

Then use that profile in any command:

```bash
aws s3 ls --profile staging
```

Or set it as an environment variable for your entire terminal session:

```bash
export AWS_PROFILE=staging
```

### Verify Your Identity

Always know which account and user you're operating as:

```bash
aws sts get-caller-identity
```

```json
{
    "UserId": "AIDAEXAMPLEID",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/wendy"
}
```

## Common Commands by Service

### S3 (Simple Storage Service)

List all your buckets:

```bash
aws s3 ls
```

List objects in a bucket:

```bash
aws s3 ls s3://my-bucket/
aws s3 ls s3://my-bucket/images/ --recursive
```

Create a new bucket:

```bash
aws s3 mb s3://my-new-bucket-12345
```

Remove an empty bucket:

```bash
aws s3 rb s3://my-new-bucket-12345
```

Remove a bucket and all its contents:

```bash
aws s3 rb s3://my-new-bucket-12345 --force
```

Copy a file to/from S3:

```bash
# Upload a file
aws s3 cp myfile.txt s3://my-bucket/

# Download a file
aws s3 cp s3://my-bucket/myfile.txt ./downloaded.txt

# Copy between buckets
aws s3 cp s3://bucket-a/data.csv s3://bucket-b/data.csv
```

Sync a local directory with S3 (only uploads changes):

```bash
aws s3 sync ./my-folder s3://my-bucket/my-folder
aws s3 sync s3://my-bucket/my-folder ./my-folder
```

Generate a pre-signed URL (temporary download link):

```bash
aws s3 presign s3://my-bucket/secret-report.pdf --expires-in 3600
```

This outputs a URL that anyone can use to download the file for the next hour, no AWS credentials needed.

### EC2 (Elastic Compute Cloud)

List all your instances:

```bash
aws ec2 describe-instances
```

List only running instances with their IDs and public IPs:

```bash
aws ec2 describe-instances \
  --filters "Name=instance-state-name,Values=running" \
  --query "Reservations[].Instances[].[InstanceId, PublicIpAddress, Tags[?Key=='Name'].Value|[0]]" \
  --output table
```

Launch a new instance:

```bash
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --instance-type t2.micro \
  --key-name my-keypair \
  --security-group-ids sg-0123456789abcdef0 \
  --count 1
```

Stop, start, or terminate instances:

```bash
aws ec2 stop-instances --instance-ids i-0abc123def456
aws ec2 start-instances --instance-ids i-0abc123def456
aws ec2 terminate-instances --instance-ids i-0abc123def456
```

List security groups:

```bash
aws ec2 describe-security-groups
```

### Lambda

Invoke a function directly:

```bash
aws lambda invoke \
  --function-name my-function \
  --payload '{"key": "value"}' \
  output.json

cat output.json
```

List all functions:

```bash
aws lambda list-functions --query "Functions[].FunctionName"
```

Create a new function from a zip file:

```bash
aws lambda create-function \
  --function-name my-new-function \
  --runtime python3.12 \
  --role arn:aws:iam::123456789012:role/lambda-role \
  --handler lambda_function.lambda_handler \
  --zip-file fileb://function.zip
```

Update a function's code:

```bash
aws lambda update-function-code \
  --function-name my-new-function \
  --zip-file fileb://function.zip
```

### IAM (Identity and Access Management)

List all users:

```bash
aws iam list-users --query "Users[].UserName"
```

Create a new user:

```bash
aws iam create-user --user-name developer-jane
```

Create a role:

```bash
aws iam create-role \
  --role-name lambda-s3-role \
  --assume-role-policy-document file://trust-policy.json
```

The trust policy file (`trust-policy.json`) defines who can assume the role:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": { "Service": "lambda.amazonaws.com" },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

Attach a managed policy to a role:

```bash
aws iam attach-role-policy \
  --role-name lambda-s3-role \
  --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess
```

### DynamoDB

Create a table:

```bash
aws dynamodb create-table \
  --table-name Users \
  --attribute-definitions AttributeName=userId,AttributeType=S \
  --key-schema AttributeName=userId,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST
```

Put an item:

```bash
aws dynamodb put-item \
  --table-name Users \
  --item '{"userId": {"S": "u001"}, "name": {"S": "Alice"}, "email": {"S": "alice@example.com"}}'
```

Get an item by key:

```bash
aws dynamodb get-item \
  --table-name Users \
  --key '{"userId": {"S": "u001"}}'
```

Query items (requires a key condition):

```bash
aws dynamodb query \
  --table-name Users \
  --key-condition-expression "userId = :uid" \
  --expression-attribute-values '{":uid": {"S": "u001"}}'
```

## Output Formats and Filtering with --query

### Output Formats

Every AWS CLI command supports three output formats:

```bash
# JSON (default) — full detail, good for scripting
aws ec2 describe-instances --output json

# Table — human-readable, good for quick checks
aws ec2 describe-instances --output table

# Text — tab-separated, good for piping to other commands
aws ec2 describe-instances --output text
```

Set your default in `~/.aws/config`:

```
[default]
output = json
```

### Filtering with --query

The `--query` parameter uses JMESPath syntax to filter and reshape output. This is extremely useful for pulling out exactly the data you need.

Get just instance IDs:

```bash
aws ec2 describe-instances \
  --query "Reservations[].Instances[].InstanceId"
```

Get instance ID and state as a table:

```bash
aws ec2 describe-instances \
  --query "Reservations[].Instances[].[InstanceId, State.Name]" \
  --output table
```

Get all S3 bucket names:

```bash
aws s3api list-buckets --query "Buckets[].Name"
```

Filter Lambda functions by runtime:

```bash
aws lambda list-functions \
  --query "Functions[?Runtime=='python3.12'].FunctionName"
```

### Using --filter vs --query

Some commands have a `--filters` parameter that filters on the server side (faster for large result sets). Use `--query` for client-side filtering and reshaping.

```bash
# Server-side filter: only running instances
aws ec2 describe-instances --filters "Name=instance-state-name,Values=running"

# Client-side query: reshape the output
aws ec2 describe-instances \
  --filters "Name=instance-state-name,Values=running" \
  --query "Reservations[].Instances[].[InstanceId, PublicIpAddress]"
```

## Tips and Aliases

### Useful Shell Aliases

Add these to your `~/.bashrc` or `~/.zshrc`:

```bash
alias awswho="aws sts get-caller-identity"
alias s3ls="aws s3 ls"
alias ec2ls="aws ec2 describe-instances --query 'Reservations[].Instances[].[InstanceId,State.Name,PublicIpAddress,Tags[?Key==\`Name\`].Value|[0]]' --output table"
```

### Dry Run

Many EC2 commands support `--dry-run` to test if you have the right permissions without actually executing:

```bash
aws ec2 run-instances --dry-run --image-id ami-0abcdef1234567890 --instance-type t2.micro
```

### Wait Commands

Some operations take time. Use `wait` to block until a resource reaches a desired state:

```bash
aws ec2 wait instance-running --instance-ids i-0abc123def456
echo "Instance is now running!"
```

### Auto-Prompt Mode

Enable interactive command completion:

```bash
aws --cli-auto-prompt
```

This gives you a menu-driven interface to build commands.
