# Installing the AWS CLI, AWS Toolkit for VS Code, and SDKs

Installing and configuring the essential tools for working with AWS locally.

## What You'll Learn

- How to install AWS CLI v2 on macOS, Linux, and Windows
- How to configure credentials using `aws configure` and named profiles
- The difference between access keys and IAM Identity Center (SSO)
- How to set up the AWS Toolkit for VS Code
- An overview of AWS SDKs for Python and Node.js
- How to verify everything is working

## Installing the AWS CLI

The AWS CLI (Command Line Interface) lets you manage AWS services from your terminal. Version 2 is the current release and the only one you should install.

### macOS

The recommended method uses the official `.pkg` installer:

```bash
# Download the installer
curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"

# Run the installer
sudo installer -pkg AWSCLIV2.pkg -target /

# Clean up the downloaded file
rm AWSCLIV2.pkg
```

Alternatively, if you use Homebrew:

```bash
brew install awscli
```

### Linux (x86_64)

```bash
# Download and unzip
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip

# Run the installer
sudo ./aws/install

# Clean up
rm -rf aws awscliv2.zip
```

For ARM-based Linux (e.g., Graviton instances or Raspberry Pi):

```bash
curl "https://awscli.amazonaws.com/awscli-exe-linux-aarch64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
rm -rf aws awscliv2.zip
```

### Windows

Download and run the MSI installer from:
[https://awscli.amazonaws.com/AWSCLIV2.msi](https://awscli.amazonaws.com/AWSCLIV2.msi)

Or use winget from PowerShell:

```powershell
winget install Amazon.AWSCLI
```

After installation, close and reopen your terminal (or PowerShell) so the `aws` command is available on your PATH.

### Verify the Installation

Run this on any platform:

```bash
aws --version
```

You should see output like:

```
aws-cli/2.17.x Python/3.11.x Darwin/23.x.x source/arm64
```

The exact version numbers will differ, but as long as it starts with `aws-cli/2` you're on v2.

## Configuring Credentials and Profiles

### Method 1: Access Keys (Simplest for Learning)

If you followed Lesson 1, you created an IAM user with access keys. Use them now:

```bash
aws configure
```

The CLI prompts you for four values:

```
AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: us-east-1
Default output format [None]: json
```

- **Access Key ID:** Starts with `AKIA`. This is not secret.
- **Secret Access Key:** This IS secret. Never commit it to Git or share it.
- **Region:** `us-east-1` (N. Virginia) is the most common and cheapest. Pick the region closest to you if latency matters.
- **Output format:** `json` is the most useful. Other options are `table` (human-readable) and `text` (tab-delimited).

This creates two files in `~/.aws/`:

- `~/.aws/credentials` — stores your access key and secret
- `~/.aws/config` — stores your region and output preferences

### Named Profiles

If you work with multiple AWS accounts (e.g., personal and work), use named profiles:

```bash
aws configure --profile personal
aws configure --profile work
```

Use a profile by adding `--profile` to any command:

```bash
aws s3 ls --profile personal
```

Or set it as the default for your terminal session:

```bash
export AWS_PROFILE=personal
```

### Method 2: IAM Identity Center (SSO)

For organizations and more advanced setups, IAM Identity Center (formerly AWS SSO) is the recommended approach. It uses temporary credentials that rotate automatically, which is more secure than long-lived access keys.

To configure SSO:

```bash
aws configure sso
```

The CLI walks you through:
1. Enter your SSO start URL (provided by your organization, e.g., `https://my-org.awsapps.com/start`)
2. Enter the SSO region
3. A browser window opens for authentication
4. Select the AWS account and role
5. Choose a profile name

To log in and refresh your credentials:

```bash
aws sso login --profile my-sso-profile
```

> **For beginners:** Start with access keys (Method 1). Switch to IAM Identity Center when you join a team or want to follow production best practices.

### Security Best Practices for Credentials

- Never hardcode access keys in source code
- Never commit `~/.aws/credentials` to Git — add it to `.gitignore`
- Rotate access keys every 90 days
- Use IAM Identity Center or IAM Roles for production workloads
- If a key is compromised, deactivate it immediately in the IAM console

## AWS Toolkit for VS Code

The AWS Toolkit extension adds AWS integration directly into VS Code, making it easier to browse and manage resources without leaving your editor.

### Installation

1. Open VS Code
2. Go to the Extensions panel (`Cmd+Shift+X` on macOS, `Ctrl+Shift+X` on Windows/Linux)
3. Search for **"AWS Toolkit"**
4. Click **Install** on the extension by Amazon Web Services

### What It Gives You

- **AWS Explorer** panel in the sidebar to browse S3 buckets, Lambda functions, CloudFormation stacks, and more
- **Credentials management** — connect using the same profiles you configured with `aws configure`
- **Lambda local testing** — invoke Lambda functions locally with SAM
- **CloudWatch Logs** — view and search log groups from the editor
- **S3 file browser** — upload, download, and browse S3 objects
- **Step Functions visualization** — render state machine diagrams

### Connecting to Your Account

1. Open the AWS Toolkit panel (click the AWS icon in the left sidebar)
2. Click **Select a connection** or **Connect to AWS**
3. Choose the profile you configured earlier (e.g., `default` or `personal`)
4. The Explorer panel populates with your AWS resources

## AWS SDKs Overview

The AWS CLI is great for interactive use and scripts, but when building applications you'll use an AWS SDK (Software Development Kit) in your programming language.

### Python — boto3

boto3 is the official AWS SDK for Python and one of the most popular ways to interact with AWS programmatically.

```bash
pip install boto3
```

Quick example — listing your S3 buckets:

```python
import boto3

s3 = boto3.client('s3')
response = s3.list_buckets()

for bucket in response['Buckets']:
    print(bucket['Name'])
```

boto3 automatically uses the credentials from `~/.aws/credentials` (or environment variables, or an IAM role if running on AWS).

### Node.js — @aws-sdk

AWS SDK v3 for JavaScript uses modular packages — you install only the services you need:

```bash
npm install @aws-sdk/client-s3
```

Quick example — listing your S3 buckets:

```javascript
import { S3Client, ListBucketsCommand } from "@aws-sdk/client-s3";

const client = new S3Client({ region: "us-east-1" });
const response = await client.send(new ListBucketsCommand({}));

for (const bucket of response.Buckets) {
  console.log(bucket.Name);
}
```

### Other Supported Languages

AWS provides official SDKs for Java, .NET, Go, Ruby, PHP, Rust, and more. They all follow similar patterns: create a client, call methods, handle responses. The [AWS SDK documentation](https://aws.amazon.com/tools/) has getting-started guides for each.

### How SDKs Find Credentials

All AWS SDKs use the same credential resolution chain (checked in this order):

1. **Environment variables** — `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY`
2. **Shared credentials file** — `~/.aws/credentials`
3. **AWS config file** — `~/.aws/config` (with profiles)
4. **Container credentials** — for ECS tasks
5. **Instance profile** — for EC2 instances with an IAM role attached

This means code that works locally with your `~/.aws/credentials` file will also work on EC2 or Lambda without changes — as long as the appropriate IAM role is attached.

## Hands-On: Your First CLI Command

Let's verify everything works by running a few commands.

### Check Your Identity

This confirms your credentials are configured correctly and shows which account and user you're authenticated as:

```bash
aws sts get-caller-identity
```

Expected output:

```json
{
    "UserId": "AIDAEXAMPLEUSERID",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/admin"
}
```

### List Your S3 Buckets

```bash
aws s3 ls
```

If your account is new, this returns empty output (no buckets yet). That's fine — no error means it's working.

### Check Your Configured Region

```bash
aws configure get region
```

Should output your default region (e.g., `us-east-1`).

### List Available Regions

```bash
aws ec2 describe-regions --query 'Regions[*].RegionName' --output table
```

This lists all AWS regions available to your account.

### Troubleshooting

**"Unable to locate credentials"** — Run `aws configure` again. Check that `~/.aws/credentials` exists and contains your keys.

**"An error occurred (InvalidClientTokenId)"** — Your access key is wrong or has been deactivated. Generate a new key pair in the IAM console.

**"An error occurred (SignatureDoesNotMatch)"** — Your secret key is wrong. Regenerate it. Watch for extra spaces when copying.

**"Could not connect to the endpoint URL"** — Check your internet connection. Also verify your default region is valid (`aws configure get region`).

## Key Takeaways

- Install AWS CLI v2 — it's available on macOS, Linux, and Windows
- Run `aws configure` to store your access key, secret key, default region, and output format
- Use named profiles (`--profile`) when working with multiple AWS accounts
- Access keys are the simplest way to start; switch to IAM Identity Center for team environments
- The AWS Toolkit for VS Code lets you browse resources and logs without leaving your editor
- boto3 (Python) and @aws-sdk (Node.js) are the main SDKs you'll use for building applications
- `aws sts get-caller-identity` is your go-to command for checking if credentials are working
