# Signing Up for an AWS Account

Free Tier — 12 Months + Always-Free Services.

## What You'll Learn

- How to create an AWS account from scratch
- What information and payment method you need
- The difference between Always-Free and 12-Month Free Tier services
- The specific Free Tier limits you can rely on while learning
- Why you should never use the root account for daily work

## Creating Your AWS Account

Go to [https://aws.amazon.com](https://aws.amazon.com) and click **Create an AWS Account** (top-right corner).

### Step 1 — Root User Email and Account Name

Enter an email address and an account name. The email becomes your **root user** login — it has unrestricted access to everything in the account.

> **Tip:** Use a dedicated email for AWS (e.g., `yourname+aws@gmail.com`). Gmail's plus-addressing lets you filter AWS emails easily while using your existing inbox.

The account name is just a label (e.g., "My Learning Account"). You can change it later.

### Step 2 — Verify Your Email

AWS sends a verification code to your email. Enter it on the next screen. Check your spam folder if it doesn't arrive within a minute.

### Step 3 — Set a Root Password

Choose a strong, unique password for the root user. This password protects the most powerful credentials in your account — treat it like a bank password.

Requirements:
- At least 8 characters
- Mix of uppercase, lowercase, numbers, and symbols

Store this password in a password manager. You will rarely use it after initial setup.

### Step 4 — Choose Account Type

Select **Personal** for learning purposes. Business accounts are functionally identical — the only difference is the contact information fields AWS collects.

### Step 5 — Contact Information

Fill in your name, phone number, and address. This information is used for billing and account recovery.

## Verifying Your Identity and Payment Method

### Phone Verification

AWS will call or text your phone number with a verification code. This confirms you're a real person and helps with account recovery.

### Credit or Debit Card

You **must** provide a valid credit or debit card, even if you plan to stay within Free Tier. AWS performs a temporary $1 USD authorization (sometimes shown as a pending charge) to verify the card. This is reversed within 3–5 days.

> **Important:** AWS charges are post-paid. If you accidentally use paid services, the charge appears on your card at the end of the billing cycle. Set up billing alerts immediately (covered in Lesson 5).

### Choose a Support Plan

Select **Basic Support (Free)**. It includes:
- 24/7 access to documentation, whitepapers, and support forums
- AWS Trusted Advisor (limited checks)
- AWS Personal Health Dashboard

The paid plans (Developer at $29/mo, Business at $100/mo+) add direct access to AWS support engineers. You don't need these while learning.

After completing these steps, AWS activates your account. This usually takes a few minutes but can take up to 24 hours. You'll receive a confirmation email when it's ready.

## Understanding the Free Tier

The AWS Free Tier has three categories. Knowing which category a service falls into prevents surprise bills.

### Always Free (Never Expires)

These services remain free forever, regardless of when you created your account:

| Service | Free Allowance |
|---------|---------------|
| **Lambda** | 1 million requests + 400,000 GB-seconds of compute per month |
| **DynamoDB** | 25 GB storage + 25 read/write capacity units |
| **SNS** | 1 million publishes per month |
| **SQS** | 1 million requests per month |
| **CloudWatch** | 10 custom metrics + 10 alarms |
| **API Gateway** | 1 million REST API calls per month (first 12 months) |
| **Step Functions** | 4,000 state transitions per month |
| **CodeBuild** | 100 build minutes per month |

### 12-Month Free (Starts from Account Creation Date)

These are free for 12 months after you create your account. The clock starts immediately — not from first use:

| Service | Free Allowance |
|---------|---------------|
| **EC2** | 750 hours/month of t2.micro (or t3.micro in regions where t2.micro is unavailable) |
| **S3** | 5 GB standard storage + 20,000 GET + 2,000 PUT requests |
| **RDS** | 750 hours/month of db.t2.micro (Single-AZ) + 20 GB storage |
| **CloudFront** | 1 TB data transfer out per month |
| **EBS** | 30 GB of General Purpose (SSD) storage |
| **Elastic Load Balancer** | 750 hours/month of Classic or Application Load Balancer |

> **750 hours/month = 1 instance running 24/7.** If you run two t2.micro instances, you'll use 1,500 hours and be billed for the second 750. Shut down what you're not using.

### Short-Term Trials

Some services offer limited free trials (e.g., Amazon Redshift gives 2 months free, Amazon Inspector offers a 15-day trial). These vary and change over time — always check the [Free Tier page](https://aws.amazon.com/free/) before launching a new service.

## Securing Your Root Account

The root account has unrestricted access to your entire AWS environment, including billing. Follow these steps immediately after account creation.

### Enable Multi-Factor Authentication (MFA) on Root

1. Sign in to the AWS Console as root
2. Click your account name (top-right) → **Security credentials**
3. Under **Multi-factor authentication (MFA)**, click **Assign MFA device**
4. Choose **Authenticator app** (use Google Authenticator, Authy, or 1Password)
5. Scan the QR code and enter two consecutive codes to confirm

### Create an IAM User for Daily Work

Never use the root account for day-to-day tasks. Create an IAM user instead:

1. Go to the **IAM** service in the console (search "IAM" in the top search bar)
2. Click **Users** → **Create user**
3. Enter a username (e.g., `admin`)
4. Check **Provide user access to the AWS Management Console**
5. On the permissions page, click **Attach policies directly**
6. Search for and attach `AdministratorAccess` (for learning purposes — you'd use more restrictive policies in production)
7. Complete the wizard and save the sign-in URL, username, and temporary password

From now on, sign in using the **IAM user sign-in URL** (which looks like `https://123456789012.signin.aws.amazon.com/console`) instead of the root sign-in page.

### Create Access Keys for the CLI (on Your IAM User)

You'll need these in Lesson 3 when setting up the AWS CLI:

1. Go to **IAM** → **Users** → click your IAM user
2. Go to the **Security credentials** tab
3. Under **Access keys**, click **Create access key**
4. Select **Command Line Interface (CLI)** as the use case
5. Download or copy the **Access Key ID** and **Secret Access Key**

> **Warning:** The secret access key is shown only once. Save it in a password manager immediately. If you lose it, you'll need to create a new key pair.

## Key Takeaways

- Creating an AWS account requires an email, phone number, and credit/debit card
- Choose the **Basic (Free)** support plan — it's all you need for learning
- The Free Tier has three categories: Always Free, 12-Month Free, and Trials
- **750 hours/month** is the magic number for EC2 and RDS — that's one instance running continuously
- Enable MFA on root immediately after account creation
- Create an IAM user and use it for everything — lock the root credentials away
- Your 12-month free tier clock starts the day you create the account, not when you first use a service
