## Why Cleanup Matters More Than You Think

The most common cause of surprise AWS bills is forgotten resources. Here's what happens:

```
Tuesday:  Spin up RDS instance to test a query         ($0.02/hour)
Tuesday:  Create a NAT Gateway for VPC testing         ($0.045/hour)
Tuesday:  Allocate an Elastic IP                       ($0.005/hour)
Tuesday:  Finish learning, close laptop

Wednesday-Sunday: Resources sit idle

Monday:   Check email — AWS bill alert: $15.60
```

The biggest offenders for forgotten charges:

| Resource | Cost if Forgotten |
|----------|------------------|
| NAT Gateway | ~$32/month |
| RDS db.t3.micro (running 24/7) | ~$12-15/month |
| Elastic IP (unattached) | ~$3.60/month |
| EC2 t3.micro (running 24/7) | ~$7.50/month |
| EBS volumes (orphaned) | ~$0.10/GB/month |
| Elastic Load Balancer | ~$16/month |
| VPN Connection | ~$36/month |

A single forgotten NAT Gateway for one month costs more than most learners' entire AWS experiments should.

## Manual Cleanup — Console Walkthrough

When in doubt, check these services in the AWS Console:

1. **EC2 Dashboard** → Running instances → Terminate any dev instances
2. **EC2 → Elastic IPs** → Release any unattached IPs
3. **EC2 → Volumes** → Delete any "available" (unattached) EBS volumes
4. **EC2 → NAT Gateways** → Delete any NAT Gateways
5. **EC2 → Load Balancers** → Delete unused ALBs/NLBs
6. **RDS → Databases** → Delete instances (skip final snapshot for dev)
7. **S3 → Buckets** → Empty and delete test buckets
8. **VPC → NAT Gateways** → Delete (most expensive forgotten resource)
9. **VPC → Elastic IPs** → Release unused
10. **CloudFormation → Stacks** → Delete old stacks (cascading delete)

**Pro tip:** Switch between regions! Resources in `us-west-2` don't show up when you're looking at `us-east-1`. Check every region you've used.

## AWS CLI Cleanup Commands

**Find and terminate running EC2 instances:**

```bash
# List all running instances
aws ec2 describe-instances \
  --filters "Name=instance-state-name,Values=running" \
  --query "Reservations[].Instances[].[InstanceId,InstanceType,LaunchTime,Tags[?Key=='Name'].Value|[0]]" \
  --output table

# Terminate specific instances
aws ec2 terminate-instances --instance-ids i-0abc123def456 i-0xyz789ghi012
```

**Release unattached Elastic IPs:**

```bash
# Find Elastic IPs not associated with an instance
aws ec2 describe-addresses \
  --query "Addresses[?AssociationId==null].[AllocationId,PublicIp]" \
  --output table

# Release them
aws ec2 release-address --allocation-id eipalloc-0abc123
```

**Delete orphaned EBS volumes:**

```bash
# Find volumes in "available" state (not attached to anything)
aws ec2 describe-volumes \
  --filters "Name=status,Values=available" \
  --query "Volumes[].[VolumeId,Size,CreateTime]" \
  --output table

# Delete them
aws ec2 delete-volume --volume-id vol-0abc123
```

**Delete NAT Gateways:**

```bash
# List NAT Gateways
aws ec2 describe-nat-gateways \
  --filter "Name=state,Values=available" \
  --query "NatGateways[].[NatGatewayId,SubnetId,CreateTime]" \
  --output table

# Delete
aws ec2 delete-nat-gateway --nat-gateway-id nat-0abc123
```

**Delete RDS instances:**

```bash
# List RDS instances
aws rds describe-db-instances \
  --query "DBInstances[].[DBInstanceIdentifier,DBInstanceClass,DBInstanceStatus]" \
  --output table

# Delete (skip final snapshot for dev instances)
aws rds delete-db-instance \
  --db-instance-identifier my-dev-db \
  --skip-final-snapshot
```

**Empty and delete S3 buckets:**

```bash
# List buckets
aws s3 ls

# Empty a bucket (required before deletion)
aws s3 rm s3://my-test-bucket --recursive

# Delete the bucket
aws s3 rb s3://my-test-bucket
```

**Delete CloudFormation stacks (cascading resource cleanup):**

```bash
# List stacks
aws cloudformation list-stacks \
  --stack-status-filter CREATE_COMPLETE UPDATE_COMPLETE \
  --query "StackSummaries[].[StackName,CreationTime]" \
  --output table

# Delete (removes all resources in the stack)
aws cloudformation delete-stack --stack-name my-dev-stack
```

## aws-nuke and cloud-nuke Tools

For comprehensive account cleanup, use dedicated nuke tools.

**cloud-nuke (by Gruntwork) — simpler, fewer dependencies:**

```bash
# Install
brew install gruntwork-io/tap/cloud-nuke

# Dry run — see what would be deleted (ALWAYS do this first)
cloud-nuke aws --dry-run

# Nuke everything older than 24 hours
cloud-nuke aws --older-than 24h

# Nuke specific resource types only
cloud-nuke aws --resource-type ec2 --resource-type ebs

# Exclude production regions
cloud-nuke aws --exclude-region us-east-1 --exclude-region eu-west-1
```

**aws-nuke (by rebuy) — more configurable, supports filters:**

```bash
# Install
wget https://github.com/rebuy-de/aws-nuke/releases/latest/download/aws-nuke-linux-amd64.tar.gz
tar -xzf aws-nuke-linux-amd64.tar.gz

# Create config file
cat > nuke-config.yml << 'EOF'
regions:
  - us-east-1
  - us-west-2
  - global

account-blocklist:
  - "999999999999"  # production account - NEVER nuke

accounts:
  "123456789012":   # your dev account
    filters:
      IAMUser:
        - "admin"   # don't delete your admin user
      IAMRole:
        - "OrganizationAccountAccessRole"
EOF

# Dry run
aws-nuke -c nuke-config.yml --profile dev-account

# Actually nuke (requires typing account alias to confirm)
aws-nuke -c nuke-config.yml --profile dev-account --no-dry-run
```

**CRITICAL SAFETY RULES:**
- NEVER run nuke tools on production accounts
- ALWAYS use `--dry-run` first
- ALWAYS use account blocklists for production
- Use a dedicated "sandbox" account for learning if possible
- Double-check the account ID before confirming

## Writing a Personal Cleanup Script

Create a reusable script for your regular cleanup:

```bash
#!/bin/bash
# cleanup-dev.sh — Run weekly to catch forgotten resources

set -e

echo "=== AWS Dev Account Cleanup ==="
echo "Account: $(aws sts get-caller-identity --query Account --output text)"
echo "Region: ${AWS_DEFAULT_REGION:-us-east-1}"
echo ""

# 1. Check for running EC2 instances
echo "--- Running EC2 Instances ---"
INSTANCES=$(aws ec2 describe-instances \
  --filters "Name=instance-state-name,Values=running" \
  --query "Reservations[].Instances[].InstanceId" --output text)

if [ -n "$INSTANCES" ]; then
  echo "WARNING: Found running instances: $INSTANCES"
  echo "  Terminate with: aws ec2 terminate-instances --instance-ids $INSTANCES"
else
  echo "  None found. OK."
fi

# 2. Check for unattached Elastic IPs
echo ""
echo "--- Unattached Elastic IPs ---"
EIPS=$(aws ec2 describe-addresses \
  --query "Addresses[?AssociationId==null].AllocationId" --output text)

if [ -n "$EIPS" ]; then
  echo "WARNING: Unattached EIPs found. Releasing..."
  for eip in $EIPS; do
    aws ec2 release-address --allocation-id "$eip"
    echo "  Released: $eip"
  done
else
  echo "  None found. OK."
fi

# 3. Check for NAT Gateways
echo ""
echo "--- NAT Gateways ---"
NATS=$(aws ec2 describe-nat-gateways \
  --filter "Name=state,Values=available" \
  --query "NatGateways[].NatGatewayId" --output text)

if [ -n "$NATS" ]; then
  echo "WARNING: NAT Gateways found ($$$ cost). Deleting..."
  for nat in $NATS; do
    aws ec2 delete-nat-gateway --nat-gateway-id "$nat"
    echo "  Deleted: $nat"
  done
else
  echo "  None found. OK."
fi

# 4. Check for orphaned EBS volumes
echo ""
echo "--- Orphaned EBS Volumes ---"
VOLS=$(aws ec2 describe-volumes \
  --filters "Name=status,Values=available" \
  --query "Volumes[].VolumeId" --output text)

if [ -n "$VOLS" ]; then
  echo "WARNING: Unattached volumes found. Deleting..."
  for vol in $VOLS; do
    aws ec2 delete-volume --volume-id "$vol"
    echo "  Deleted: $vol"
  done
else
  echo "  None found. OK."
fi

# 5. Check for RDS instances
echo ""
echo "--- RDS Instances ---"
RDS=$(aws rds describe-db-instances \
  --query "DBInstances[].DBInstanceIdentifier" --output text)

if [ -n "$RDS" ]; then
  echo "WARNING: RDS instances found: $RDS"
  echo "  Delete with: aws rds delete-db-instance --db-instance-identifier <name> --skip-final-snapshot"
else
  echo "  None found. OK."
fi

# 6. Check estimated charges
echo ""
echo "--- Current Month Estimated Charges ---"
aws ce get-cost-and-usage \
  --time-period Start=$(date -u +%Y-%m-01),End=$(date -u +%Y-%m-%d) \
  --granularity MONTHLY \
  --metrics BlendedCost \
  --query "ResultsByTime[0].Total.BlendedCost.Amount" \
  --output text 2>/dev/null || echo "  (Cost Explorer not enabled or no data yet)"

echo ""
echo "=== Cleanup check complete ==="
```

Make it executable and run weekly:

```bash
chmod +x cleanup-dev.sh
./cleanup-dev.sh
```

## Tag-Based Resource Discovery and Deletion

If you consistently tag resources, cleanup becomes much easier:

```bash
# Find all resources tagged with Project=learning
aws resourcegroupstaggingapi get-resources \
  --tag-filters Key=Project,Values=learning \
  --query "ResourceTagMappingList[].[ResourceARN]" \
  --output table

# Find resources tagged DeleteAfter with a past date
aws resourcegroupstaggingapi get-resources \
  --tag-filters Key=DeleteAfter \
  --query "ResourceTagMappingList[].[ResourceARN,Tags[?Key=='DeleteAfter'].Value|[0]]" \
  --output table
```

When creating resources, always add a `DeleteAfter` tag:

```bash
aws ec2 run-instances \
  --instance-type t3.micro \
  --tag-specifications \
    'ResourceType=instance,Tags=[{Key=DeleteAfter,Value=2026-05-07},{Key=Project,Value=learning}]' \
  ...
```

## Daily and Weekly Habits

**Daily (takes 30 seconds):**
- Glance at the AWS Billing Dashboard → is the number still $0?
- Before closing your laptop: did you leave anything running?

**Weekly (takes 5 minutes):**
- Run your cleanup script
- Check Cost Explorer for any non-zero line items
- Review each region you've used

**After every learning session:**
- Terminate EC2 instances you started
- Delete CloudFormation stacks you created
- Stop RDS instances (or delete if you won't use them again)
- Release Elastic IPs

**Monthly:**
- Review the full bill line by line
- Check for any services you don't recognize
- Verify free tier usage hasn't exceeded limits

## Automating Cleanup with EventBridge + Lambda

For extra safety, set up an automated cleanup that runs nightly:

```python
import boto3
from datetime import datetime, timezone

def lambda_handler(event, context):
    ec2 = boto3.client('ec2')
    
    # Find instances tagged with DeleteAfter in the past
    today = datetime.now(timezone.utc).strftime('%Y-%m-%d')
    
    response = ec2.describe_instances(
        Filters=[
            {'Name': 'tag-key', 'Values': ['DeleteAfter']},
            {'Name': 'instance-state-name', 'Values': ['running', 'stopped']}
        ]
    )
    
    terminated = []
    for reservation in response['Reservations']:
        for instance in reservation['Instances']:
            tags = {t['Key']: t['Value'] for t in instance.get('Tags', [])}
            delete_after = tags.get('DeleteAfter', '9999-12-31')
            
            if delete_after <= today:
                instance_id = instance['InstanceId']
                ec2.terminate_instances(InstanceIds=[instance_id])
                terminated.append(instance_id)
    
    if terminated:
        print(f"Terminated expired instances: {terminated}")
    
    return {'terminated': terminated}
```

Schedule it with EventBridge:

```yaml
# SAM template
Resources:
  CleanupFunction:
    Type: AWS::Serverless::Function
    Properties:
      CodeUri: cleanup/
      Handler: app.lambda_handler
      Runtime: python3.11
      Timeout: 60
      Policies:
        - Statement:
          - Effect: Allow
            Action:
              - ec2:DescribeInstances
              - ec2:TerminateInstances
            Resource: "*"
      Events:
        NightlyCleanup:
          Type: Schedule
          Properties:
            Schedule: cron(0 2 * * ? *)  # 2 AM UTC daily
```

## Final Checklist Before Walking Away

Run through this every time you finish a learning session:

```
□ EC2 instances terminated (not just stopped)
□ RDS instances deleted (or stopped if you'll use them soon)
□ NAT Gateways deleted
□ Elastic IPs released
□ Load Balancers deleted
□ EBS volumes checked for orphans
□ S3 buckets emptied and deleted (if test data)
□ CloudFormation stacks deleted
□ VPC endpoints removed (if not gateway type)
□ Check Cost Explorer — is today's cost $0?
```

**One final check:**

```bash
# Quick cost check for today
aws ce get-cost-and-usage \
  --time-period Start=$(date -u +%Y-%m-%d),End=$(date -u -d tomorrow +%Y-%m-%d) \
  --granularity DAILY \
  --metrics BlendedCost \
  --query "ResultsByTime[0].Total.BlendedCost" \
  --output table
```

If it shows anything above $0.01, investigate immediately.

## Key Takeaways

- Forgotten resources are the #1 cause of surprise AWS bills for learners
- NAT Gateways, RDS, and Elastic IPs are the most commonly forgotten cost sources
- Write a cleanup script and run it weekly (or daily if you're actively learning)
- Tag every resource with `Project` and `DeleteAfter` for easy discovery
- Use `cloud-nuke` for comprehensive account cleanup (always dry-run first!)
- Automate expiration-based cleanup with EventBridge + Lambda
- Check your billing dashboard daily — it takes 30 seconds and saves you money
- When in doubt, delete it. You can always recreate resources for learning.
