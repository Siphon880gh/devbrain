# LocalStack Paid Plans and Limits (What the Free Hobby Tier Cannot Emulate)

Learn which AWS services require a LocalStack Pro or Team license, and understand the boundaries of the free Community Edition.

## What You'll Learn

- What the free Hobby tier includes and what it doesn't
- The differences between Starter, Pro, and Team plans
- Which services require paid plans
- When you actually need to upgrade
- Free alternatives for services not in the Hobby tier
- Cost comparison: LocalStack paid vs. real AWS

## Community vs. Pro vs. Team — Plan Comparison

| Feature | Hobby (Free) | Starter (~$15/mo) | Pro (~$35/mo) | Team (~$70/mo/seat) |
|---------|-------------|-------------------|---------------|---------------------|
| Core services (S3, Lambda, DynamoDB, SQS, SNS) | Yes | Yes | Yes | Yes |
| API Gateway | Yes | Yes | Yes | Yes |
| CloudFormation | Yes | Yes | Yes | Yes |
| IAM (basic, not enforced) | Yes | Yes | Yes | Yes |
| Kinesis, Step Functions | Yes | Yes | Yes | Yes |
| **IAM enforcement** | No | Yes | Yes | Yes |
| **Cognito** | No | No | Yes | Yes |
| **RDS / Aurora** | No | No | Yes | Yes |
| **ElastiCache (Redis)** | No | No | Yes | Yes |
| **ECS / EKS** | No | No | Yes | Yes |
| **MSK (Kafka)** | No | No | Yes | Yes |
| **CloudFront** | No | No | Yes | Yes |
| **Athena** | No | No | Yes | Yes |
| Cloud pods (state snapshots) | No | Limited | Yes | Yes |
| CI/CD integration | Limited | Yes | Yes | Yes |
| Team collaboration | No | No | No | Yes |
| Shared environments | No | No | No | Yes |
| Priority support | No | No | Yes | Yes |

Prices are approximate and change over time — check [localstack.cloud/pricing](https://localstack.cloud/pricing) for current rates.

## Services Missing from the Free Tier

The Hobby tier lacks these commonly-needed services:

**Authentication & Identity:**
- Cognito User Pools and Identity Pools
- Full IAM policy enforcement (Hobby tier accepts any permissions)

**Databases (beyond DynamoDB):**
- RDS (PostgreSQL, MySQL, MariaDB)
- Aurora
- ElastiCache (Redis, Memcached)
- Neptune (graph)
- Redshift

**Container Services:**
- ECS (Elastic Container Service)
- EKS (Elastic Kubernetes Service)
- ECR (Elastic Container Registry) — partial support in Hobby

**Streaming & Analytics:**
- MSK (Managed Streaming for Apache Kafka)
- Athena
- Glue
- EMR

**Networking & Content Delivery:**
- CloudFront
- Route 53 (advanced features)
- App Mesh

**Advanced Compute:**
- Batch
- App Runner

## Fidelity Gaps — Where Emulation Differs from AWS

Even for supported services, LocalStack doesn't perfectly replicate AWS:

**S3:**
- Object versioning works but lifecycle policies may not execute
- CORS and bucket policies are partially enforced
- Transfer acceleration doesn't apply (already local)

**Lambda:**
- Cold starts behave differently (Docker container startup vs. AWS microVM)
- Layers work but have size limit differences
- Concurrency limits aren't enforced

**API Gateway:**
- Custom authorizers work but may skip some validation
- WebSocket APIs have limited support in Hobby
- Usage plans and throttling are not enforced

**DynamoDB:**
- On-demand vs. provisioned makes no difference locally (no throttling)
- Global tables not available
- DAX (accelerator) not emulated

**CloudFormation:**
- Most resources deploy, but drift detection doesn't work
- Stack rollback behavior differs
- Custom resources may not work identically

## IAM Enforcement and Advanced Features

By default, LocalStack's free tier **does not enforce IAM policies**. This means:

```bash
# This works in free LocalStack even without proper permissions
aws --endpoint-url=http://localhost:4566 s3 ls
```

In real AWS, this would fail without `s3:ListAllMyBuckets` permission. The Starter and Pro tiers offer IAM enforcement mode where policies are actually checked — useful for testing that your application has correct permissions before deploying.

To enable IAM enforcement (paid plans):

```bash
LOCALSTACK_AUTH_TOKEN=your-token \
ENFORCE_IAM=1 \
localstack start
```

## Is the Pro Plan Worth It for Learners?

**You probably don't need Pro if:**
- You're learning core AWS services (S3, Lambda, DynamoDB, SQS)
- You're building serverless applications
- Your app doesn't use Cognito, RDS, or ECS
- You're comfortable testing IAM in a real AWS free tier account

**Consider Pro if:**
- You're building a full-stack app with Cognito authentication
- Your architecture requires RDS or ElastiCache
- You want to run realistic ECS/EKS workloads locally
- You need IAM enforcement for security testing
- Your team wants shared development environments

For most beginners, the free Hobby tier covers everything you need for the first 6-12 months of learning.

## Alternatives When a Service Is Not Emulated

You don't need LocalStack Pro for everything. Use standalone alternatives:

| Service | Free Alternative | How |
|---------|-----------------|-----|
| DynamoDB | DynamoDB Local | `docker run amazon/dynamodb-local` |
| S3 | MinIO | `docker run minio/minio server /data` |
| Redis/ElastiCache | Redis in Docker | `docker run redis:7` |
| PostgreSQL/RDS | PostgreSQL in Docker | `docker run postgres:16` |
| MySQL/RDS | MySQL in Docker | `docker run mysql:8` |
| SQS | ElasticMQ | `docker run softwaremill/elasticmq` |
| Cognito | cognito-local | `npm install -g cognito-local` |
| Kafka/MSK | Kafka in Docker | Use Confluent's Docker images |
| Elasticsearch | OpenSearch in Docker | `docker run opensearchproject/opensearch` |

**Example: Running Redis locally instead of paying for LocalStack Pro:**

```yaml
# docker-compose.yml
version: "3.8"
services:
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
  
  localstack:
    image: localstack/localstack
    ports:
      - "4566:4566"
    environment:
      - LOCALSTACK_AUTH_TOKEN=${LOCALSTACK_AUTH_TOKEN}
```

Your app connects to Redis at `localhost:6379` and AWS services at `localhost:4566`. This gives you the same development experience without paying for Pro.

**Example: cognito-local for authentication testing:**

```bash
npm install -g cognito-local

# Start the Cognito emulator
cognito-local
# Runs on http://localhost:9229
```

Configure your app:

```javascript
const cognitoClient = new CognitoIdentityProviderClient({
  endpoint: "http://localhost:9229",
  region: "us-east-1",
  credentials: { accessKeyId: "test", secretAccessKey: "test" },
});
```

## Cost Comparison: LocalStack Pro vs. Real AWS

| Scenario | LocalStack Pro Cost | Real AWS Cost (estimate) |
|----------|--------------------|-----------------------|
| Monthly subscription | ~$35/month | $0 (free tier) to $50+/month |
| Running Cognito + RDS + Redis | $35/month flat | $0 free tier (first year) then $40-80/month |
| Team of 5 developers | ~$350/month (Team) | $0 (shared dev account) |
| CI/CD pipeline (100 runs/mo) | Included in plan | $5-20/month in Lambda/resource time |

**Verdict for learners:**
1. Start with the free Hobby tier + standalone Docker alternatives
2. Use real AWS free tier for services that can't be emulated
3. Only consider Pro if you're building production apps with Cognito/RDS/ECS and the free alternatives don't cut it

## Making the Decision

```
Do I need Cognito, RDS, ECS, or ElastiCache?
├── No → Free Hobby tier is enough
└── Yes → Can I use standalone Docker alternatives?
    ├── Yes → Use Docker (Redis, PostgreSQL, cognito-local)
    └── No → Do I need high-fidelity AWS behavior?
        ├── No → Docker alternatives are fine
        └── Yes → Consider LocalStack Pro or use real AWS free tier
```

## Key Takeaways

- The free Hobby tier covers S3, Lambda, DynamoDB, SQS, SNS, API Gateway, Step Functions, and more — plenty for learning
- Paid plans add Cognito, RDS, ElastiCache, ECS/EKS, IAM enforcement, and team features
- You can replace most missing services with standalone Docker containers at no cost
- IAM enforcement (testing permissions) requires at least the Starter plan
- For most learners, Hobby tier + DynamoDB Local + Redis Docker + PostgreSQL Docker covers 95% of use cases
- Only upgrade when standalone alternatives genuinely don't work for your architecture
