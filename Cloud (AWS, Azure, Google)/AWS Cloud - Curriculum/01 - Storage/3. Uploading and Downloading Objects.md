# Uploading and Downloading Objects

Using the Console, CLI, and SDK to move data in and out of S3.

## What You'll Learn

- How to upload files through the AWS Console
- Essential CLI commands: `cp`, `sync`, `mv`, `ls`, and `rm`
- How to use Python (boto3) and Node.js SDKs for programmatic access
- When and how multipart uploads work for large files
- Low-level operations with `aws s3api`

## Uploading via the Console

The simplest way to upload a file is through the AWS Management Console.

**Step-by-step:**

1. Open the [S3 Console](https://s3.console.aws.amazon.com/)
2. Click on your bucket name
3. Click the **Upload** button
4. Drag files into the upload area or click **Add files** / **Add folder**
5. (Optional) Expand **Properties** to set:
   - Storage class (Standard, IA, Glacier, etc.)
   - Server-side encryption settings
6. (Optional) Expand **Permissions** — leave defaults unless you know you need to change them
7. Click **Upload**

The console shows a progress bar and confirms when the upload completes. This works well for a handful of files, but for bulk uploads or automation, you need the CLI or SDK.

> **Tip:** The console has a 160 GB limit per file upload. For files larger than a few GB, use the CLI — it handles multipart uploads automatically.

## Uploading via the CLI

The AWS CLI provides two levels of S3 commands:

- **`aws s3`** — High-level commands (cp, sync, mv, ls, rm). These handle multipart uploads, retries, and parallelism automatically.
- **`aws s3api`** — Low-level API calls that map directly to S3 REST operations. More control, more verbose.

### aws s3 cp — Copy Files

```bash
# Upload a single file
aws s3 cp myfile.txt s3://my-bucket/myfile.txt

# Upload with a different name
aws s3 cp myfile.txt s3://my-bucket/documents/renamed-file.txt

# Upload all files in a directory (recursive)
aws s3 cp ./local-folder/ s3://my-bucket/remote-folder/ --recursive

# Download a file
aws s3 cp s3://my-bucket/myfile.txt ./downloaded-file.txt

# Copy between buckets
aws s3 cp s3://source-bucket/file.txt s3://destination-bucket/file.txt
```

### aws s3 sync — Sync Directories

`sync` compares source and destination, then only copies files that are new or changed. This is the most efficient way to keep a local directory and an S3 prefix in sync.

```bash
# Sync a local directory to S3
aws s3 sync ./my-website/ s3://my-website-bucket/

# Sync from S3 to local
aws s3 sync s3://my-bucket/backups/ ./local-backups/

# Sync and delete files in destination that don't exist in source
aws s3 sync ./my-website/ s3://my-website-bucket/ --delete

# Sync only certain file types
aws s3 sync ./assets/ s3://my-bucket/assets/ --exclude "*" --include "*.jpg" --include "*.png"
```

> **Important:** `--delete` will remove files from the destination that don't exist in the source. Use with caution — test with `--dryrun` first.

```bash
# Preview what sync would do without actually doing it
aws s3 sync ./my-website/ s3://my-website-bucket/ --delete --dryrun
```

### aws s3 mv — Move or Rename

```bash
# Move a local file to S3 (deletes the local copy)
aws s3 mv myfile.txt s3://my-bucket/myfile.txt

# Rename an object in S3 (copies then deletes the original)
aws s3 mv s3://my-bucket/old-name.txt s3://my-bucket/new-name.txt

# Move all objects to a new prefix
aws s3 mv s3://my-bucket/old-prefix/ s3://my-bucket/new-prefix/ --recursive
```

### aws s3 ls — List Objects

```bash
# List all buckets
aws s3 ls

# List objects in a bucket
aws s3 ls s3://my-bucket/

# List objects under a specific prefix
aws s3 ls s3://my-bucket/images/

# List ALL objects recursively (flat list, no "folder" grouping)
aws s3 ls s3://my-bucket/ --recursive

# Show human-readable file sizes
aws s3 ls s3://my-bucket/ --recursive --human-readable --summarize
```

The `--summarize` flag at the end shows total objects and total size.

### aws s3 rm — Delete Objects

```bash
# Delete a single object
aws s3 rm s3://my-bucket/myfile.txt

# Delete all objects under a prefix
aws s3 rm s3://my-bucket/temp-data/ --recursive

# Delete everything in a bucket
aws s3 rm s3://my-bucket/ --recursive
```

## Multipart Uploads

For files larger than **100 MB**, the CLI automatically splits the upload into multiple parts and uploads them in parallel. This is called a **multipart upload**.

**Why multipart matters:**

- **Faster** — Multiple parts upload simultaneously
- **Resilient** — If one part fails, only that part retries (not the entire file)
- **Required** — Objects over 5 GB _must_ use multipart upload

The CLI handles this transparently. You run `aws s3 cp` and it decides whether to use single-part or multipart based on file size.

### Configuring Multipart Thresholds

```bash
# Set the threshold and chunk size via CLI config
aws configure set s3.multipart_threshold 64MB
aws configure set s3.multipart_chunksize 16MB
```

### Manual Multipart Upload with s3api

For full control (useful in scripts or recovery scenarios):

```bash
# Step 1: Initiate the upload
aws s3api create-multipart-upload \
  --bucket my-bucket \
  --key large-file.zip

# Returns an UploadId — save it

# Step 2: Upload each part
aws s3api upload-part \
  --bucket my-bucket \
  --key large-file.zip \
  --part-number 1 \
  --body part1.bin \
  --upload-id "YOUR_UPLOAD_ID"

# Step 3: Complete the upload (after all parts are uploaded)
aws s3api complete-multipart-upload \
  --bucket my-bucket \
  --key large-file.zip \
  --upload-id "YOUR_UPLOAD_ID" \
  --multipart-upload '{"Parts":[{"ETag":"etag1","PartNumber":1},{"ETag":"etag2","PartNumber":2}]}'
```

> **Tip:** Incomplete multipart uploads consume storage and cost money. Set a lifecycle rule to abort them after a few days (covered in Lesson 4).

## Downloading Objects

### CLI Downloads

```bash
# Download a single file
aws s3 cp s3://my-bucket/data/report.csv ./report.csv

# Download an entire prefix (directory) recursively
aws s3 cp s3://my-bucket/backups/2024-01/ ./local-backups/ --recursive

# Sync only changes from S3 to local
aws s3 sync s3://my-bucket/data/ ./local-data/
```

### Python SDK (boto3)

```python
import boto3

s3 = boto3.client('s3')

# Upload a file
s3.upload_file('local-file.txt', 'my-bucket', 'remote-key.txt')

# Download a file
s3.download_file('my-bucket', 'remote-key.txt', 'downloaded-file.txt')

# Upload with metadata
s3.upload_file(
    'report.pdf',
    'my-bucket',
    'reports/2024/q1-report.pdf',
    ExtraArgs={
        'ContentType': 'application/pdf',
        'ServerSideEncryption': 'aws:kms'
    }
)

# List objects in a bucket
response = s3.list_objects_v2(Bucket='my-bucket', Prefix='reports/')
for obj in response.get('Contents', []):
    print(f"{obj['Key']}  ({obj['Size']} bytes)")

# Read a file directly into memory (no local download)
response = s3.get_object(Bucket='my-bucket', Key='config.json')
content = response['Body'].read().decode('utf-8')
print(content)
```

### Node.js SDK (v3)

```javascript
const { S3Client, PutObjectCommand, GetObjectCommand } = require('@aws-sdk/client-s3');
const fs = require('fs');

const s3 = new S3Client({ region: 'us-east-1' });

// Upload a file
async function uploadFile() {
  const fileContent = fs.readFileSync('local-file.txt');
  await s3.send(new PutObjectCommand({
    Bucket: 'my-bucket',
    Key: 'remote-key.txt',
    Body: fileContent,
    ContentType: 'text/plain'
  }));
  console.log('Upload complete');
}

// Download a file
async function downloadFile() {
  const response = await s3.send(new GetObjectCommand({
    Bucket: 'my-bucket',
    Key: 'remote-key.txt'
  }));
  const content = await response.Body.transformToString();
  console.log(content);
}
```

### Low-Level Operations with aws s3api

`s3api` commands give you direct access to every S3 API operation:

```bash
# Get detailed object metadata without downloading
aws s3api head-object --bucket my-bucket --key myfile.txt

# Get an object and save to a file
aws s3api get-object --bucket my-bucket --key myfile.txt output.txt

# Put an object with specific content type and encryption
aws s3api put-object \
  --bucket my-bucket \
  --key config.json \
  --body ./config.json \
  --content-type application/json \
  --server-side-encryption aws:kms

# List objects with details (including storage class)
aws s3api list-objects-v2 --bucket my-bucket --prefix "data/" --max-items 10
```

## Hands-On: Upload and Retrieve a File Three Ways

Practice uploading and downloading using the console, CLI, and Python:

**1. Console:** Navigate to your bucket in the S3 Console, upload a small text file using the GUI.

**2. CLI:**

```bash
echo "Uploaded via CLI" > cli-test.txt
aws s3 cp cli-test.txt s3://your-bucket/cli-test.txt
aws s3 cp s3://your-bucket/cli-test.txt cli-downloaded.txt
cat cli-downloaded.txt
```

**3. Python:**

```python
import boto3

s3 = boto3.client('s3')
s3.put_object(Bucket='your-bucket', Key='sdk-test.txt', Body=b'Uploaded via SDK')
response = s3.get_object(Bucket='your-bucket', Key='sdk-test.txt')
print(response['Body'].read().decode())
```

**Clean up all three:**

```bash
aws s3 rm s3://your-bucket/cli-test.txt
aws s3 rm s3://your-bucket/sdk-test.txt
# Delete the console-uploaded file too
```

## Key Takeaways

- The **Console** is good for quick, small uploads; the **CLI** and **SDKs** are for automation and bulk operations
- `aws s3 cp` copies files; `aws s3 sync` efficiently mirrors directories; `aws s3 mv` moves/renames
- **Multipart upload** is automatic for large files via the CLI and improves speed and reliability
- `aws s3` is high-level and easy; `aws s3api` is low-level and gives you full API control
- **boto3** (Python) and **@aws-sdk/client-s3** (Node.js) are the most common SDKs for programmatic access
- Always use `--dryrun` with `sync --delete` before running it for real
