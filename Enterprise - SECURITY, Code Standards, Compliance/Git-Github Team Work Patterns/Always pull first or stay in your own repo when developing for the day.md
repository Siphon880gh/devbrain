## Startup Bootstrapped (not so professional)

When developing at localhost for the day, pull in case changes are made to your repo. Actually, you may want to fetch then pull so you can review the changes. 

If everyone's working on the same branch, you may want to schedule who's on to make changes to the branch.

## More Professional Team

Each developer works on their own branch. No need to pull before working on code for the day at localhost.
