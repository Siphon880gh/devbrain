**Tailscale** is a tool that lets your devices securely connect to each other **as if they’re on the same private network**, no matter where they are.

Their website:
https://tailscale.com/

They offer a free tier. Higher plans have security and enterprise controls.

## Simple way to think about it

Instead of:

* opening ports
* dealing with firewalls
* setting up a traditional VPN server
* doing a bunch of network configuration

Tailscale usually makes it much easier through its **simple CLI and app-based setup**.

You typically just:

* install it
* sign in
* run a basic command like `tailscale up`

That is a big reason people like it: **no complicated VPN-style configuring** for most normal setups.

## What it actually does

At its core, Tailscale is a **mesh VPN** built on **WireGuard**.

That means:

* each device gets a private IP
* devices can talk directly to each other
* traffic is encrypted end-to-end

## What people use it for

### 1. Access your servers from anywhere

* SSH into your VPS without exposing port 22
* access internal dashboards remotely

### 2. Connect home lab or local machines

* laptop ↔ desktop ↔ NAS
* works even across different networks

### 3. Private internal APIs

* run services without exposing them publicly
* keep tools internal to only your trusted devices

### 4. Replace traditional VPN

* no OpenVPN-style config files
* no separate VPN server to maintain

### 5. Expose local AI and dev services privately

This is especially useful for tools like:

* **LiteLLM**
* **Ollama**

For example, if one machine is running:

```bash
Ollama  -> http://localhost:11434
LiteLLM -> http://localhost:4000
```

After joining that machine to Tailscale, another one of your Tailscale-connected devices can often reach them through that machine’s Tailscale IP, such as:

```bash
http://100.x.x.x:11434
http://100.x.x.x:4000
```

So you can use your local AI services across your own devices **without publicly exposing those ports to the internet**.

## Why this is helpful

Normally, exposing services to other machines can mean:

* opening firewall ports
* setting up reverse proxies
* handling DNS
* worrying about bot traffic
* managing more networking rules

Tailscale reduces a lot of that friction.

Its **CLI-first, low-config approach** makes it feel much more like:

* install
* log in
* connect
* use the service

instead of:

* build and manage a whole VPN/network stack by hand

## Quick mental model

> Tailscale is your own private network between your devices, with a simple CLI and much less setup pain than a traditional VPN.
