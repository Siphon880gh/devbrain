If you’re running apps with PM2, logs can quietly grow over time, especially if you have loops or frequent `console.log` calls. By default, PM2 writes logs to `$HOME/.pm2/logs`, and if left unchecked, these files can get huge and start impacting performance (disk I/O, CPU from constant writes, slower log reads). The quickest cleanup is `pm2 flush`, which clears all logs instantly. You can also manually inspect and remove files in `$HOME/.pm2/logs` if needed. As a habit, check log sizes periodically (`ls -lh ~/.pm2/logs`) and clean them before they become a problem. If your logs are growing fast, it’s usually a signal that your app is logging too frequently, not just a storage issue.
