## When only setting up the project

Use the existing migrations, then seed:

```bash
npx prisma generate
npx prisma migrate dev
npx prisma db seed
```

## When changing the schema

Edit `schema.prisma`, then create a migration:

```bash
npx prisma migrate dev --name describe_the_change
npx prisma db seed
```

## When deploying

Do not create migrations on the server.

Only apply migrations that already exist:

```bash
npx prisma migrate deploy
```
