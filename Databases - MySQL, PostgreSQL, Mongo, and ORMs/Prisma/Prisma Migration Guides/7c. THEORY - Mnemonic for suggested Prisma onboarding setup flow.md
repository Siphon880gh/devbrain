When there are Prisma migration files (`prisma/migrations/*`), your team member receiving the files would run things in this order:

Setup env variables
```
npm install
npx prisma generate
npx prisma migrate deploy
npx prisma db seed # or npm run seed
```

At this point you should understand why in this order. However, it helps to have a mnemonic to remember this order because it can be a quick reference for you:

GDS = generate, then deploy, then seed (if there's a seed script)

Prisma **gets database started** for EEs.

EEs are electrical engineers (Think College major electrical engineering which is often paired with computer science). The EEs if they didn't study programming may not be that good with the different databases so Prisma makes it easy for them to get databases started. The two E's remind you that the second word (database) gets the letter E and the third word (started) gets the letter E, so it bcoems:
GE DE SE

Or:
Generate Deploy Seed
