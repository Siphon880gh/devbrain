
These seed files are more than demo data. They often act as bootstrap data the application expects to exist.

## `seed-data.ts`

This file may define structured records such as:

- slug
    
- display name
    
- role title
    
- department
    
- status
    
- permission level
    
- category
    
- metadata
    
- configuration fields
    

That suggests the application may depend on seeded records for normal behavior.

## `seed.ts`

This file performs the actual writes to the database.

Typical responsibilities include:

- loading environment variables
    
- creating the Prisma client
    
- hashing a seed password
    
- creating or updating default users
    
- looping through seeded records
    
- using `upsert()` for safe re-runs
    

This is a strong onboarding pattern because a new developer can run one seed command and get a predictable local baseline.

## Practical takeaway

After applying migrations, run:

```bash
npx prisma db seed
```

That can provide:

- default users
    
- baseline roles
    
- system records
    
- predictable local test data
