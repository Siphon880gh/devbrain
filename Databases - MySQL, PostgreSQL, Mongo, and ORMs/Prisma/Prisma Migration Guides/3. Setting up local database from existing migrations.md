## Goal

The goal is to get a local database synced with the project’s current schema and baseline data.

## Step 1: Clone the project and install packages

```bash
npm install
```

## Step 2: Create the environment file

Example:

```env
DATABASE_URL="postgresql://user:password@localhost:5432/projectdb"
SEED_PASSWORD="changeme-local-only"
```

## Step 3: Generate Prisma Client

```bash
npx prisma generate
```

This ensures the local Prisma Client matches the current schema.

## Step 4: Apply the committed migrations

For a fresh local development database, the usual command is:

```bash
npx prisma migrate dev
```

Why use this locally?

- it is the normal development workflow
    
- it applies existing migrations to the local database
    
- it checks migration history and helps catch drift during development
    

In non-development environments, the simpler apply-only command is:

```bash
npx prisma migrate deploy
```

## Step 5: Seed the database

```bash
npx prisma db seed
```

If the seed script uses `upsert()`, rerunning the seed is usually safe and will not create duplicate records.

## Step 6: Start the app

```bash
npm run dev
```

Or use the project’s actual development command if it differs.
