One of the easiest ways to support both light mode and dark mode is to let the user choose with a button. A common pattern is to store the theme on the HTML element using an attribute such as:

```css
[data-theme="dark"]
```

Then your CSS can switch token values based on that selector.

## Step 1: Define your default theme

Start by defining your light theme in `:root`.

```css
:root {
  --color-page-bg: #ffffff;
  --color-surface: #f8fafc;
  --color-text-primary: #111827;
  --color-text-secondary: #6b7280;
  --color-border: #d1d5db;
  --color-action-primary: #2563eb;
}
```

## Step 2: Add a dark theme override

Now add a selector that applies when the HTML element has `data-theme="dark"`.

```css
@media (prefers-color-scheme: dark) {
  :root {
    --color-page-bg: #0f172a;
    --color-surface: #111827;
    --color-text-primary: #f9fafb;
    --color-text-secondary: #9ca3af;
    --color-border: #374151;
    --color-action-primary: #60a5fa;
  }
}
```

Then your components use the variables normally.

```css
body {
  background: var(--color-page-bg);
  color: var(--color-text-primary);
}

.card {
  background: var(--color-surface);
  border: 1px solid var(--color-border);
}
```

## Step 3: Add a button in HTML

Create a simple button users can click.

```html
<button id="theme-toggle" type="button">Toggle theme</button>
```

## Step 4: Use JavaScript to update the HTML attribute

When the user clicks the button, JavaScript can set or remove the `data-theme` attribute on the `<html>` element.

```html
<script>
  const toggleBtn = document.getElementById('theme-toggle');
  const root = document.documentElement;

  toggleBtn.addEventListener('click', () => {
    const currentTheme = root.getAttribute('data-theme');

    if (currentTheme === 'dark') {
      root.setAttribute('data-theme', 'light');
    } else {
      root.setAttribute('data-theme', 'dark');
    }
  });
</script>
```

This works because your CSS is listening for:

```css
html[data-theme="dark"]
```

and using that selector to swap the variables.

## Optional: remember the user’s choice

If you want the site to remember the user’s preference, save it in `localStorage`.

```html
<script>
  const toggleBtn = document.getElementById('theme-toggle');
  const root = document.documentElement;

  const savedTheme = localStorage.getItem('theme');
  if (savedTheme) {
    root.setAttribute('data-theme', savedTheme);
  }

  toggleBtn.addEventListener('click', () => {
    const currentTheme = root.getAttribute('data-theme');

    const nextTheme = currentTheme === 'dark' ? 'light' : 'dark';
    root.setAttribute('data-theme', nextTheme);
    localStorage.setItem('theme', nextTheme);
  });
</script>
```

## Why this pattern is useful

This pattern keeps theme logic simple:

- CSS controls the visual rules
    
- JavaScript only changes the HTML attribute
    
- components do not need separate hardcoded dark-mode styles everywhere
    

It also scales well. You can use the same approach for more themes later, such as `data-theme="sepia"` or `data-theme="high-contrast"`.

## Final takeaway

A button-based theme toggle works best when JavaScript updates the root HTML attribute, such as `data-theme="dark"`, and your CSS uses that selector to remap theme variables. This keeps your theme system simple, readable, and easy to maintain.
