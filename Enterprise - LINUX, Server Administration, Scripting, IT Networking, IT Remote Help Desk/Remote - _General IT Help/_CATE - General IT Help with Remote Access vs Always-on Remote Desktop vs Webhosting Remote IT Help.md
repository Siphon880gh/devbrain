There are two categories of Remote Help:
- **Web Hosting Remote IT Help**.
- **General Remote Access IT Help or Always On Remote Desktop**

Either category involves remote access, whether to control the computer remotely or because the team helping is remote.

----

With **web hosting or server support**, the technician usually helps with a **VPS or dedicated server** that you own. Because these servers are not automatically managed like shared hosting, the technician may need deeper access.

That usually happens in one of two ways:

- They give you access to a server console with more control over server settings.
- You give them the root password so they can log in and fix the server directly.

This type of help is focused on managing or troubleshooting the actual server, especially when you need control over things like services, firewall settings, software installs, or configuration files.

---

**General IT Help** is more like what you see in a corporate office environment.

For example, someone from IT may remotely connect to your desktop computer. You can usually see your screen, and you may see the mouse moving while they help fix the issue.

In this category, we also include similar remote-access setups where you intentionally keep a computer always on so you can access it from anywhere. For example, you might keep a machine running for **OpenClaw management** or other remote admin tasks.

So the difference is:

**Web Hosting / Server IT Help** is usually about controlling a VPS or dedicated server.

**General IT Help** is usually about remotely controlling a desktop computer or always-on machine.
